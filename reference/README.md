# reference/

> 参考素材目录。存放写作过程中需要的各类参考资料。**目录结构纳入版本管理，内容文件不纳入**（已在 `.gitignore` 中排除）。

## 目录结构

```
reference/
├── README.md              # 本文件
├── novels/                # 参考作品（完整小说文本）——风格分析+技法提取的源材料
├── manuscripts/           # 原稿（用户上传的原始稿件）——审阅流和改编流的输入
│   └── _analysis/         # 原作画像文档——改编流阶段0的输出
├── settings/              # 设定参考（世界观/角色设定书/规则书）——优先级高于其他来源
├── artwork/               # 美术设定图（角色立绘/场景概念/风格参考）
└── trpg_settings_logs/    # TRPG 跑团设定日志——世界观灵感来源

profiles/                  # （项目根目录，独立于 reference/）
└── authors/               # 作者风格档案——七维分析产出，写作/改编流风格对标用
    ├── _index.md          # 作者档案索引
    └── {作者名}.md         # 单个作者的完整风格档案
```

## 各类参考的使用方法

### novels/ — 参考作品

存放作为风格基准的完整小说文本。Agent 在以下场景使用：
- **作者档案建立**：「五更」执行分卷全量七维分析（叙述者风格/情节设计/关系设计/常用技法/世界观风格/大纲设计与篇幅结构/语言指纹）→ 写入 `profiles/authors/`
- **技法提取**：「五更」分析参考作品的情节结构/桥段设计/角色塑造 → 写入 `novel/technique-library.md`（标注来源作者）
- **风格分析**：「铃」读取参考作品，提取段落/句子/对话/幽默机制的统计特征 → 写入 `novel/author-voice.md`

**上传方式**：将 `.txt` 文件放入此目录。文件命名格式：`书名_作者.txt`。

**Agent 触发**：
- 放入文件后说"分析这个参考作品"→ 同时触发 `author-profiling` skill（建立作者档案）+ `reference-analysis` skill（技法入库）
- 写作/改编时指定对标某作者 → 读取 `profiles/authors/{作者名}.md` 作为风格约束

### manuscripts/ — 原稿

存放用户自己写的原始稿件（需要审阅或改写的版本），或其他已有文学作品（需要改编的素材）。Agent 在以下场景使用：
- **原稿审阅**：「五更」调度「铃」「琉璃」并行诊断 → 汇总报告 → 默认询问是否转入改编流 → 「墨」修订 → 输出到 `_drafts/`
- **原作改编**：「五更」+「琉璃」原作画像提取（六维分析）→ 用户选择改写层次（L1-L6）→ 「墨」在原作画像约束下执笔改编 → 输出到 `_drafts/`

**上传方式**：将原稿文件（`.txt` / `.md` / `.docx`）放入此目录。

**Agent 触发**：
- 放入文件后说"审阅这个原稿"→ 触发 `manuscript-review` skill
- 放入文件后说"改编/改写/润色这个作品"→ 触发 `adaptation-workflow` skill
- 审阅报告出来后，默认询问是否转入改编流进行更深层次改写

**`_analysis/` 子目录**：改编流阶段 0 生成的原作画像文档存放于此，作为后续所有改编阶段的持续对照材料。与上传文件同名，后缀 `.md`。

### settings/ — 设定参考

存放世界观设定书、规则书、角色设计文档等系统性设定资料。Agent 在以下场景使用：
- **设定提取**：从设定书中提取世界观规则/角色信息 → 写入 `novel/world/` 和 `novel/characters/`
- **一致性校验**：「琉璃」评审时以此目录中的设定书为权威依据

**优先级规则**：`settings/` 中的设定书 > 其他来源的设定信息。冲突时以设定书为准。

**上传方式**：将设定文件（`.docx` / `.xlsx` / `.md`）放入此目录。

### artwork/ — 美术设定图

存放角色立绘、场景概念图、风格参考图等视觉素材。Agent 在以下场景使用：
- **角色外貌描写**：「墨」写作时参考立绘确保外貌描写一致
- **插画生成**：「音」读取立绘建立角色视觉锚点 → 生成插画 prompt
- **风格参考**：「音」参考视觉风格图确定插画的整体美术方向

**上传方式**：将图片文件（`.png` / `.jpg`）放入此目录。

**Agent 触发**：放入文件后对 Agent 说"分析这些立绘"或"建立角色视觉档案"。

### trpg_settings_logs/ — TRPG 跑团设定日志

存放 TRPG 跑团过程中积累的世界观设定、角色设定、剧情日志等。作为世界观构建的灵感来源和参考素材。

## 关联目录：profiles/authors/

独立于 `reference/`，存放从参考作品中提取的作者风格档案。每个作者一个 `.md` 文件，由 `author-profiling` skill 生成和维护。

**档案内容**（七维分析 + 跨卷对比）：
1. 叙述者声音肖像
2. 情节设计模式
3. 关系设计模式
4. 常用技法清单
5. 世界观设计风格
6. 大纲设计与篇幅结构（章节字数分布/内容配比/节拍密度/章节骨架/弧线模板）
7. 语言指纹（量化数据 + 跨卷波动）

**使用场景**：
- 写作流启发式交谈：提供"如果是 XX 作者会如何写"视角
- 大纲规划：参照作者篇幅设计模式规划章节字数、场景数、内容配比
- 改编流 L1（文风移植）：作为目标风格约束
- 「铃」风格把关：将作者档案的语言指纹作为额外检查维度

## 方法论参考

`AI文学作品改写润色方法论.docx` — 文学作品改写润色的体系化方法论论文（六层次模型），已研究并整合到本项目的以下 Skill 中：
- `.claude/skills/adaptation-workflow.md` — 改编流六层次（L1 文风移植 → L6 完全改编）
- `.claude/skills/author-profiling.md` — 作者风格档案分析
- `.claude/skills/manuscript-review.md` — 原稿审阅流（可转入改编流）
- `.claude/skills/reference-analysis.md` — 参考作品技法提取

## 文件管理规则

| 规则 | 说明 |
|------|------|
| 目录结构 | ✅ 纳入版本管理（通过 `.gitkeep` 保持空目录） |

| README | ✅ 纳入版本管理 |
