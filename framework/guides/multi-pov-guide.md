# 多主角叙事指南

> **定位**：可选工作流。当项目确定为多主角结构时激活。本文档将《八方旅人》的多主角叙事机制转化为可操作的创作工具。
>
> **激活条件**：`project-config.md` 中叙事人称设置为多 POV，或用户在启发式交谈中确认使用多主角结构。
>
> **理论基础**：参考《多主角叙事的游戏与文学研究》（`reference/多主角叙事的游戏与文学研究.docx`）。

---

## 适用性判断

Agent 在阶段零（确定核心概念）做启发式交谈时，检查以下条件，任一满足则提出多主角选项：

- 故事有 ≥3 个动机/价值观/成长弧差异足够大的角色可承担 POV
- 世界观足够大，需要多视角拼合才能完整呈现
- 核心主题（如"旅行""成长""归宿""复仇"）能同时容纳多个独立故事线
- 各角色故事线在空间上可独立推进、时间上存在共享锚点

**不适用的情况**：
- 故事本质是单一主角的成长史，其他角色只是配角/辅助
- 世界观不需要多视角就能完整呈现
- 多条 POV 会稀释读者对核心主角的情感投入

## 一、轮转节奏设计

### 1.1 两种基本模式

| 模式 | 节奏 | 适用 | 实例 |
|------|------|------|------|
| **等距轮转** | A→B→C→A→B→C→... | 角色地位均等，无核心主角 | 《八方旅人》8 人每章轮转 |
| **焦点轮转** | A→B→A→C→A→B→... | 有核心主角 A，其他 POV 补充视角 | 核心主角每 2 章至少出场 1 次 |

### 1.2 节奏规则

- 同一 POV 角色连续不超过 2 章（避免其他线停滞感）
- 同一 POV 角色不超过 3 章不出场（避免读者遗忘）
- 每卷开头和结尾建议用同一核心视角（给读者稳定的进入和收束点）
- 重大共享事件发生时，建议用受影响最大的角色视角

### 1.3 轮转节奏表模板

在 `outline.md` L3（篇章级）中为每卷新增：

```
| 章位 | POV 角色 | 本章功能 | 共享事件锚点 | 信息边界（该 POV 未知的关键信息） |
|------|---------|----------|-------------|----------------------------------|
| 1 | A | 建立核心问题 | 事件 X 发生 | A 不知道 B 已暗中介入 |
| 2 | B | 揭示隐藏信息 | 事件 X 的另一面 | B 不知道 A 的真实身份 |
| 3 | C | 引入新威胁 | 事件 X 的后果 | C 不知道 A 和 B 的关联 |
```

## 二、角色专属章节设计

### 2.1 每章聚焦单一角色

借鉴《八方旅人》每章聚焦一个角色的核心动机：

- **章节开头**：锚定该角色当前的心理状态和核心动机
- **章节主体**：围绕该角色的个人目标推进，同时埋入共享世界线索
- **章节结尾**：该角色的个人弧推进一步 + 与共享事件的微弱共振

### 2.2 章节模板扩展

在现有 `framework/templates/chapters/_chapter-template.md` 基础上新增 POV 字段：

```
- **POV 角色**：（本章从谁的视角叙述）
- **本章个人弧推进**：（该角色在这一章中的成长/变化）
- **共享线索埋设**：（本章埋入的与其他 POV 线的关联线索）
- **信息边界**：（本章 POV 角色尚不知道、但读者可能已从其他 POV 知道的信息——用于设计 dramatic irony）
```

## 三、隐性线索与显性串联

### 3.1 线索类型

| 类型 | 说明 | 实例 |
|------|------|------|
| **共享物品** | 同一物品在不同 POV 中出现，赋予不同意义 | 《边狱之书》在不同学者章节中逐步揭示 |
| **共享地点** | 不同角色在不同时间访问同一地点 | 同一座城市在商人线（繁荣）和盗贼线（阴暗面）的不同呈现 |
| **共享历史事件** | 所有角色的故事都围绕同一事件展开 | "圣火神权阴谋"贯穿所有角色线 |
| **共享 NPC** | 同一配角在不同 POV 中扮演不同角色 | 某个商人 NPC 在 A 线是盟友，在 B 线是对手 |

### 3.2 埋设规范

每条线索在 `outline.md` 伏笔追踪表中标注：

```
| ID | 线索内容 | 类型 | 埋设章节(POV) | 计划揭示章节 | 当前状态 |
|----|---------|------|--------------|-------------|----------|
| H1 | 红色信笺的收件人 | 共享物品 | Ch1(A) | Ch5(B) | 已埋 |
```

### 3.3 显性串联时机

- **篇章末**（每 5-8 章）：两条 POV 线产生一次显性交汇（角色相遇/信息互通/事件交叉）
- **卷末**：所有 POV 线至少产生一次共振（共享事件的真相部分揭露）
- **全书终章**：所有独立故事线汇聚到同一场景/事件

## 四、小剧场设计

### 4.1 定义

"小剧场"是章节间或章节内的跨 POV 互动场景——当特定角色组合在特定地点相遇时触发的简短对话/场景。功能是弥补多线叙事的情感割裂，让读者感受到角色之间确实存在"羁绊网络"。

### 4.2 小剧场的四种功能类型

| 功能 | 说明 | 示例场景 |
|------|------|----------|
| **价值观碰撞** | 两个价值观不同的角色对同一事件的不同反应 | 神官主张宽恕 vs 盗贼主张以牙还牙 |
| **信息交换** | 不同 POV 角色分享各自掌握的信息碎片 | 学者分享古书线索，舞娘认出书中地点 |
| **情感羁绊** | 展现角色之间超越"任务关系"的情感连接 | 药师默默治疗受伤的剑士，不问缘由 |
| **伏笔呼应** | 让早期分散的隐性线索在小剧场中产生微弱共振 | 两个角色发现他们都在追踪同一枚纹章 |

### 4.3 设计规范

- **长度**：200-600 字（短，不喧宾夺主）
- **位置**：章节间过渡（作为"间章"）或章节内自然相遇场景
- **触发条件**：两个角色的路径在物理空间上产生交集
- **频率**：每 3-5 章至少一次（避免各线过于孤立）

### 4.4 小剧场规划表

在 `outline.md` 中新增：

```
| ID | 参与角色 | 触发地点 | 功能类型 | 所在章节 | 核心对话主题 |
|----|---------|---------|---------|---------|------------|
| T1 | A/B | 港口酒馆 | 信息交换 | Ch4 | 关于"红色信笺"的各自主线索 |
```

## 五、价值观系统性表达

### 5.1 原则

角色的价值观不通过对话直接说出（"我是个正义的人"），而是通过三种方式自然流露：

1. **行为模式**：角色在相同类型情境下的一贯选择
2. **语言风格**：词汇选择、句式偏好、对同一事物的不同描述方式
3. **情节选择**：在关键决策点的选择方向

### 5.2 价值观-行为映射表

为每个 POV 角色定义：

```
角色：[角色名]
- 核心价值观（1 个词）：
- 行为签名（3 个标志性行为——读者看到这些行为就知道是 ta 的章节）：
  1. 
  2. 
  3. 
- 语言签名（3 个词汇/句式特征）：
  1. 
  2. 
  3. 
- 道德底线（不可逾越的一条线）：
- 价值观盲区（ta 看不到但其他 POV 角色能看到的）：
```

此表写入角色档案。

## 六、非线性成长拼合

### 6.1 原则

多主角叙事中，每个角色的成长弧不是线性推进的——读者通过分散在不同章节中的碎片逐步拼合出完整的成长轨迹。

### 6.2 成长碎片设计

- 每个角色的成长弧在 `outline.md` 中预先规划完整路径
- 将路径拆分为 4-6 个"成长节点"
- 每个节点对应一章（该角色的 POV 章节）
- 节点之间，该角色可能在他人 POV 中以"中间状态"出现——这时的状态是合理的，因为读者知道还没到 ta 的下一章

### 6.3 角色成长追踪表

在 `outline.md` 中新增：

```
| 角色 | 成长节点 | 计划章节 | 实际章节 | 节点状态描述 |
|------|---------|---------|---------|------------|
| A | 1: 失去信仰 | Vol1-Ch1(A) | | 从坚定到动摇 |
| A | 2: 找到新目标 | Vol1-Ch4(A) | | 从动摇到重新出发 |
| A | 3: 第一次考验 | Vol1-Ch7(A) | | 新目标能否经受现实打击 |
```

## 七、跨 POV 一致性检查

### 7.1 信息边界检查（琉璃审阅新增项）

对每个章节检查：
- [ ] 本章 POV 角色已知的信息不超过其信息边界
- [ ] 本章 POV 角色没有"无意中"知道只有其他 POV 角色才知道的事
- [ ] 如果读者从其他 POV 已知某信息，本章的 dramatic irony 是否被有效利用

### 7.2 POV 声音区分度检查（铃审阅新增项）

- [ ] 不同 POV 角色的章节在语言风格上有可感知的差异
- [ ] 同一事件在不同 POV 中的描述方式不同（反映各自价值观和关注点）
- [ ] 角色 A 的章节读起来不像角色 B 的章节

### 7.3 节奏均衡检查（五更新增项）

- [ ] 各 POV 线的推进速度大致均衡
- [ ] 无 POV 角色连续 3 章以上完全不出场
- [ ] 共享事件的揭示节奏在各线中保持一致

## 八、激活流程

当用户确认使用多主角结构后，Agent 按以下顺序执行：

1. **更新 `project-config.md`**：叙事人称设为"多 POV"，标注 POV 角色数量和轮转模式
2. **创建 POV 角色档案**：每个 POV 角色填写价值观-行为映射表
3. **扩展 `outline.md`**：新增轮转节奏表、小剧场规划表、角色成长追踪表
4. **告知用户**：多主角工作流已激活，后续写作和审阅将自动启用跨 POV 检查
