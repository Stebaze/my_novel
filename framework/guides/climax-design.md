# 高潮设计方法论

> 被 `qing-novelist` Skill（D1 高潮章子对话）和 `mo-writer` Agent（简报 §3A 生成）读取。
> 核心原则：爆点不是"写得激烈"，而是"不可逆变化的集中交付"。

---

## 五阶段爆点桥段模型

任何一个完整的爆点桥段由五个阶段组成：

```
开端(Initiation) → 发展(Development) → 高潮(Climax) → 结局(Resolution) → 余震(Aftermath)
```

| 阶段 | 功能 | 核心问题 | 典型占比 |
|------|------|---------|---------|
| **开端** | 触发条件建立，初始状态确立 | "什么启动了这段剧情？" | ~15% |
| **发展** | 压力升级，退路封死 | "什么让事态变得更紧张？" | ~30% |
| **高潮** | 不可逆变化的集中爆发 | "什么被永久改变了？" | ~25% |
| **结局** | 即时后果处理，新常态建立 | "冲突如何解决？代价是什么？" | ~20% |
| **余震** | 情感消化，新种子埋设 | "角色如何消化冲击？下一步是什么？" | ~10% |

> 比例参考来自风月《天启预报》前中后期三个高潮弧线的统计分析（Ch280-290、Ch895-910、Ch1635-1646）。

每个阶段都有特定的写作目标，见「高潮段写作指南」。

### 索引维度

桥段库（`framework/templates/climax-patterns/_index.md`）提供四维检索：

1. **按作者**：风月 / 远瞳 / 言归正传 / ...
2. **按机制类型**：information_reveal / combat_decision / character_choice / multi_thread_convergence / emotional_breakthrough / power_breakthrough
3. **按光谱**：small / medium / large / super
4. **按小说阶段**：early / mid / late

### 匹配规则

D1 高潮章子对话中的桥段匹配逻辑：

```
1. 优先匹配卷级预设：thread-map §五 已指定参考桥段 → 直接加载
2. 无预设时基于上下文匹配：
   ├── 提取当前上下文特征：冲突类型 / 出场角色数 / 情感目标 / 小说阶段
   ├── 过滤 _index.md：
   │   ├── 小说阶段匹配 → +2 权重
   │   ├── 机制类型匹配 → +3 权重
   │   ├── 光谱匹配 → +1 权重
   │   └── 作者对标 → +2 权重（如果用户指定了对标作者）
   └── 呈现 Top-2 候选 + 各候选的 50 字阶段摘要 + 具体场景开头
3. 用户选定 → 记录到方向卡 §高潮设计
```

## 双层触发机制

### 卷级（D0b 子选项 f）

```
触发：新卷/新篇章第一章，作者关心爆点布局
流程：
  1. 扫描 thread-map §二 支线清单 + §四 线索交织点 + outline.md L2/L3
  2. 标注卷内各高潮点的章位、光谱、机制类型

```

### 章级（D1 高潮章子对话）

```
触发：D1 判定本章类型 = 高潮章
流程：
  1. 查 thread-map → 是否有预设？处于五阶段的哪个位置？
  2. 加载桥段模板 → 5 阶段映射到本章场景
  3. 支线高潮协调 → 节拍冲突/互补检查
  4. 余震设计 → 高潮后降落 + 新种子
```

## 支线高潮与主线高潮的关系

| 关系类型 | 描述 | 协调规则 |
|---------|------|---------|
| **并行** | 两条支线在同一章各自有独立高潮 | 检查情感色调是否冲突——一个悲壮一个搞笑 → 建议错开 |
| **交汇** | 多条线索在同一场景汇聚爆发 | 参考多线汇聚型桥段——每条线需要自己的 mini-payoff |
| **从属** | 支线高潮为主线高潮做铺垫/对照 | 支线高潮安排在主线高潮之前——作为"小爆点预热" |

协调检查清单：
- [ ] 本章的高潮节拍总数是否超过读者处理能力？（≤3 个主要节拍）
- [ ] 各高潮节拍之间是否有情感降落的间隙？（≥500 字缓冲）
- [ ] 支线高潮是否削弱了主线高潮的冲击力？（如果是，降级支线高潮为"推进"而非"高潮"）

## 全本阶段演变分析方法论

从一部完整作品中提取高潮桥段时，必须处理**阶段演变**：

### 三阶段划分

- **前期**（前20-30%）：世界观建立期，角色成长弧的奠基阶段
- **中期**（30-80%）：世界观展开期，冲突升级，角色从"发现者"变为"参与者"
- **后期**（80-100%）：世界观完全揭示，冲突内化，角色做出最终决断

### 提取步骤

1. 识别全书的卷/篇章结构，按章节百分比划分三期
2. 每期定位 2-3 个代表性高潮弧线
3. 对每个弧线执行五阶段分解 + 原文证据提取
4. 同类型跨期对比——标注"相同核心机制"和"阶段变体"
5. 产出：每期独立模板 + 阶段演变对比文件

### 风月《天启预报》的分析结论

见 `framework/templates/climax-patterns/fengyue/tqyb-phase-comparison.md`。

核心发现：**启示式高潮贯穿全书，但驱动力从"发现"（前期）→"理解"（中期）→"见证"（后期）逐层深化。**

## 高潮段写作指南

### 每阶段写作规则

**开端**：
- 事件密度：2-3 件/千字（常规密度）
- 进入方式：钩子回收式 / 事件中段切入式 / 氛围铺垫式
- 必须在第一阶段内给出至少 2 个锚点信息（谁/在哪/面临什么）

**发展**：
- 事件密度：3-4 件/千字（中密度）
- 升级机制：每次升级有独立的触发因素（新信息/退路封死/第三方介入）
- 可选支线/侧面描写：分散读者注意力的次级事件——延迟高潮预期

**高潮**：
- 事件密度：4-6 件/千字（高密度，远瞳+风月数据验证）
- 不可逆变化必须在此阶段发生——如果高潮过后一切照旧，这是伪高潮
- 断章方式：答案前一刻 / 关系临界点 / 情绪反转 / 认知翻转式

**结局**：
- 事件密度：3-4 件/千字（中密度）
- 即时后果的处理——不是解释"这意味着什么"，而是展示"现在是什么样"
- 状态重置 vs 状态跃迁：回到日常（但日常基线变了）还是进入全新状态

**余震**：
- 事件密度：1-2 件/千字（低密度——给读者呼吸空间）
- 角色情感反应：不是解决新问题，是消化已发生的事
- 新种子：至少埋一个"再读一章"的理由

### 阶段比例模板

| 光谱 | 开端:发展:高潮:结局:余震 | 参考来源 |
|------|------------------------|---------|
| 小 | 0.5 : 1 : 1 : 0.5 : 0.5 | 日常-异象-日常 |
| 中 | 1 : 2 : 1 : 1 : 0.5 | 童话破晓 |
| 大 | 1 : 3 : 2 : 2 : 1 | 永恒的终结 |
| 超 | 1 : 4 : 2 : 2 : 1 | 见证与新世 |

## 余震处理

### 设计原则

1. **情感降落梯度**：不能从高潮峰值直接跳回日常——需要 1-2 个中间情感状态
2. **冷却长度**：≈ 高潮强度的 1/2。一场 3 章的大高潮需要 1.5 章的余震
3. **新种子埋设**：余震不是"打扫战场"，而是"下一段旅程的起点"
4. **角色消化时间**：给角色（和读者）体会"刚才发生了什么"的空间

### 余震的三种模式

| 模式 | 描述 | 示例 |
|------|------|------|
| **日常沉降** | 回到日常场景，但日常基线已变 | 天启预报·Ch906 丹波群像——灵魂碎片散落后的温情人间 |
| **视角外移** | 从主角视角切换到旁观者/历史视角 | 天启预报·尾声——学者的考证困惑 |
| **时间跳跃** | 直接跳到"之后"——省略消化过程，呈现结果 | 天启预报·Ch1646 四十五万年压缩叙事 |

### 常见失败

- 余震过长 → 冲淡高潮（余震不应超过高潮本身长度）
- 余震过短 → 读者没有时间消化（至少 500 字）
- 跳过余震直接进入下一冲突 → 情感疲劳
- 余震中引入新主线冲突 → 喧宾夺主（余震只能埋种子，不能开新战场）

## 宏观节拍控制：自适应窗口节奏

> 基于风月《天启预报》全本1646章的定量分析。经变异系数(CV)优化，**55章**为该书的最佳窗口（CV=0.177）。
> 默认值为 **55章**，可在 `novel/project-config.md` →「节拍窗口大小」中按写作类型调整。

### 核心发现

**全书的三个宏观节拍规律**：

| 规律 | 数据 | 含义 |
|------|------|------|
| **章节膨胀律** | 前期均3,400字/章 → 中期3,600 → 后期4,500 → 终局9,600 | 越接近终局，单章承载量越大 |
| **决策梯度律** | 前期均26次/窗口 → 中期36 → 后期42 → 峰值83 | "事件驱动"→"决策驱动"的渐变 |
| **呼吸交替律** | 高事件窗口(300+)与低事件窗口(180-280)以约2:1比例交替 | 2窗紧张 → 1窗缓冲 → 循环 |

### 节拍窗口：卷级节奏设计的基本单位

**核心用途**：窗口用于**节拍设计**（每段剧情应该多"紧"）和**卷划分**（卷边界应对齐自然低谷），而非计数——窗口总数是计算结果，不是设计目标。

**默认值**：`55章`（经天启预报1646章CV验证，变异系数0.177）。可通过 `novel/project-config.md` →「节拍窗口大小」按写作类型调整。推荐值：

| 写作类型 | 推荐窗口 | 每卷建议窗数 |
|---------|:------:|:----------:|
| 长篇网文(800+章) | 55章 | 2-3窗/卷 |
| 中篇(300-800章) | 30章 | 3-4窗/卷 |
| 短篇/轻小说(<300章) | 10-15章 | 3-5窗/卷 |

> 每卷 2-3 窗是经天启预报数据验证的自然节奏——卷内完成一个完整的蓄势→升温→爆发周期。

### 四类窗口（用于节拍设计）

以事件密度均值（μ）为基准分类：

**排列模式（卷内标准节拍）**：

```
卷 = 蓄势窗 → 升温窗 → (爆发窗) → 冷却窗
         ↑___________ 2-3窗 = 完整呼吸周期 ___________↑
```

每卷至少 2 窗（蓄势→升温），至多 3 窗（+爆发），卷末应以冷却窗或蓄势窗收束——冷却窗是**自然的卷边界**。

**卷边界识别规则**：当某窗口的事件密度 < 全书均值×0.85 时，该窗口为自然卷边界候选。

### 卷划分：天启预报的实证

55章/窗下，天启预报1646章共30窗。按"每卷2-3窗 + 冷却窗为边界"划分，自然形成11卷：

| 卷 | 范围 | 窗数 | 节拍序列 |
|----|------|:---:|---------|
| 1 | Ch1-165 | 3 | 💤→💤→📈 |
| 2 | Ch166-330 | 3 | 💤→📈→💤 |
| 3 | Ch331-495 | 3 | 💤→💤→🧊 ←冷却窗收束 |
| 4 | Ch496-660 | 3 | 💤→📈→💤 |
| 5 | Ch661-825 | 3 | 🧊→💤→💤 |
| 6 | Ch826-935 | 2 | 📈→💤 |
| 7 | Ch936-1100 | 3 | 💤→💤→📈 |
| 8 | Ch1101-1265 | 3 | 📈→💤→💤 |
| 9 | Ch1266-1430 | 3 | 💤→📈→💤 |
| 10 | Ch1431-1595 | 3 | 🔥→🔥→📈 ←后期加速 |
| 11 | Ch1596-1646 | 2 | 💤(终局收束) |

**规律**：
- **卷=165章(3窗)是天启预报的标准卷长**，占全部11卷中的8卷
- 中期(卷3-7)冷却窗最密集——冲突高峰期需要更多缓冲
- 后期(卷10)出现罕见的🔥🔥双爆发窗——节奏加速的标志
- 终局卷(卷11)回溯密度爆发(282/窗)——世界观收束

### 节拍设计应用

**每类窗口的写作策略**：

| 窗口类型 | 事件密度 | 决策密度/窗 | 章节篇幅 | 策略 |
|---------|:--------:|:---------:|:--------:|------|
| 💤 蓄势 | μ×0.75-1.05 | 20-35 | 基准 | 建立新常态、铺设线索、角色定位 |
| 📈 升温 | μ×1.05-1.25 | 35-50 | 基准+10% | 冲突升级、支线展开、压力叠加 |
| 🔥 爆发 | >μ×1.25 | 45-60 | 基准+20% | 高潮集中、多线汇聚、不可逆变化 |
| 🧊 冷却 | <μ×0.85 | 20-30 | 基准 | 情感消化、角色反应、新种子埋设 |

**卷级节拍设计流程**：
1. 读取 `novel/project-config.md` →「节拍窗口大小」（默认55）
2. 将本卷章数划分为窗口
3. 为每个窗口分配类型，确保至少1个冷却窗在卷末
4. 决策密度应逐卷上升（前期20-30→中期30-45→后期40-60）
5. 章节篇幅应逐卷增长（后期应比前期长30-50%）

### 时间推进模式演变

| 阶段 | 次日标记 | 跳时(天/月) | 回溯(记忆) | 特征 |
|------|:-------:|:---------:|:---------:|------|
| 前期 | 均3.0/窗 | 均21/窗 | 均73/窗 | 线性推进为主，时间流动快 |
| 中期 | 均3.6/窗 | 均24/窗 | 均82/窗 | 跳时增多，时间节奏复杂化 |
| 后期 | 均2.3/窗 | 均22/窗 | 均82/窗 | 次日减少=单章覆盖更多时间 |
| 终局 | 均0.5/窗 | 均25/窗 | 均129/窗 | 时间几乎静止，记忆爆发 |

**时间密度规律**：前期时间流动最快，后期最慢（事件密集但时间几乎停滞——每一刻都被放大）。终局的时间完全凝固为记忆的堆积。

## 卷级时间推进模式

> 基于天启预报的定量分析，提取时间标记（次日/跳时/回溯）在全书的分布规律。

### 三种时间推进手法

| 手法 | 定义 | 效果 | 使用建议 |
|------|------|------|---------|
| **次日衔接** | 章节间以"第二天/次日"直接推进 | 保持叙事连续性，适合动作/战斗序列 | 战斗章/连续事件链使用，每天2-3章 |
| **跳时加速** | 以"几天后/几个月后"跳过非关键期 | 压缩过渡内容，加速到下一个关键节点 | 冷却窗内使用，跳1-2周为佳 |
| **回溯填充** | 在当前时间线中插入回忆/历史/设定 | 在不推进时间的前提下增加信息密度 | 后期和终局增加使用，前期克制 |

### 时间密度与章节类型的关系

```
战斗/动作章: 次日密度高(3-5/40窗) — 时间流动最快
日常/过渡章: 跳时密度高(25-35/40窗) — 时间被压缩  
高潮/启示章: 次日和跳时都低 — 时间被"拉长"，每一刻都在放大
终局章: 回溯密度极高(120+/40窗) — 时间变成记忆的堆叠
```

### 时间推进的卷级设计建议

1. **新卷开端**：跳时1-2次建立新常态 → 次日衔接启动事件
2. **冲突升级期**：次日密集(4-6/40窗)——保持紧张节奏
3. **高潮弧线**：次日+跳时都降到最低——让读者感觉"此刻永恒"
4. **冷却期**：跳时加速(30-35/40窗)——快速消化余波，迈向新常态
5. **终局**：允许回溯爆炸——让整个作品的记忆在此汇聚
