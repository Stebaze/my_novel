# 角色人设丰富指南

> **定位**：本文件是角色人设丰富（R8）的共享方法论。三个入口——bootstrap Phase 2b、plan-chapter Step 4（五更 Dimension 12）、独立触发（"加强角色X"）——都读取本文件执行参考角色引导式访谈。
>
> 本文件被 `qing-novelist` Skill（Dimension 12）和 `bootstrap-project` Skill（Phase 2b）直接引用。

---

## 一、五步访谈流程

### 核心原则

- 不是"你的角色太薄，加内容"——而是"借你喜欢的角色来启发你自己的角色"
- 每个步骤的提问都是开放式引导，AI 不替作者做决定
- 作者随时可以跳过任何步骤，不被阻塞
- 每个参考角色的 3-5 个核心特质最终必须有一个"差异化"答案

### 步骤 1：唤起

**目标**：让作者联想到能启发自己角色的参考角色。

**展示**：
```
你的角色 [name] 目前的设定比较薄：
  [1-2 句话概括当前已填内容，诚实但不贬低]

TA 让你想到哪些其他作品里的角色吗？
可以是小说、动漫、影视、游戏——任何让你觉得"有点像"或"希望 TA 像"的都可以。
```

**如果作者一时想不起**：
- 按角色类型提示："TA 是主角方的还是对手方的？TA 的性格偏冷还是偏热？TA 更接近智者型还是行动派？"
- 参考第三章的常见参考角色索引，提供 2-3 个可能的方向供作者选择
- 如果作者仍无法确定 → 跳过参考角色访谈，降级为直接逐维填充（跳转到步骤 4）

**如果作者给出多个参考角色**：全部记录，逐一执行步骤 2-5。

### 步骤 2：拆解

**目标**：让作者自己说出参考角色最打动他的地方——这是"映射"的原材料。

**提问**：
```
你觉得 [参考角色] 最让你印象深刻的 3-5 个特质/行为/时刻是什么？
不需要学术分析，就说说你为什么喜欢这个角色。
```

**引导细化**（如果作者的回答太抽象，如"他很酷"）：
- "他具体做了什么让你觉得酷？能回忆起一个具体场景吗？"
- "他和别的类似角色最大的不同在哪里？"
- "如果用三个形容词描述他，你会选哪三个？"

**记录**：将作者的描述整理为 3-5 个具体特质，每个特质附带作者提到的场景/行为证据。

### 步骤 3：映射

**目标**：将作者描述的特质映射到角色档案模板的 10 个维度，让作者看到"这些特质可以转化为档案中的哪些字段"。

**映射规则**：

| 作者说的 | 映射到的维度 | 引导问题 |
|---------|------------|---------|
| 性格相关的特质（"嘴硬心软"/"表面冷酷"） | 维度 3 性格剖面（表层/深层差异） | "[参考角色] 的表层和深层是怎么不同的？你的角色可以怎么改编这种反差？" |
| 行为模式（"总是牺牲自己"/"关键时刻站出来"） | 维度 4 心理动力结构（核心价值观/恐惧/防御机制） | "[参考角色] 为什么会这样做？TA 内心深处怕什么？你的角色怕的可以是什么？" |
| 说话方式（"毒舌但精准"/"沉默但一针见血"） | 维度 5 语言风格 + 维度 10 语言指纹 | "[参考角色] 的说话风格有什么特点？你的角色可以用类似的风格吗，还是需要调整？" |
| 成长轨迹（"从懦弱到勇敢"） | 维度 8 角色弧线 | "[参考角色] 的成长弧线是怎样的？你的角色的起点和终点可以是什么？" |
| 与他人的互动模式 | 维度 7 人际关系 | "[参考角色] 和身边人的关系模式是怎样的？这种模式能适配你的故事吗？" |
| 特殊能力或限制 | 维度 6 能力与限制 | "[参考角色] 的能力有什么独特的限制？你的角色可以有什么不同的限制？" |
| 身份背景 | 维度 1 基本信息 + 维度 2 角色原型 | "[参考角色] 的身份/职业对 TA 的性格有什么影响？你的角色的身份可以怎么设计？" |

**展示格式**：
```
我帮你把 [参考角色] 的特质映射到角色档案的维度：

| 参考角色的特质 | 对应档案维度 | 你的角色可以怎么改编 |
|-------------|------------|-----------------|
| "嘴硬心软" | 性格剖面（表层/深层） | [AI 建议：表层可以是什么态度，深层是什么真实想法] |
| ... | ... | ... |

你觉得这些映射对吗？有没有想调整的？
```

### 步骤 4：填充

**目标**：根据步骤 3 的映射结果，引导作者逐维填充角色档案。

**填充顺序**（从最容易到达的维度开始）：
1. 维度 2 角色原型 → 先选一个起点模板（A/B/C 系列）
2. 维度 1 基本信息 → 外表/标志动作/口头禅
3. 维度 3 性格剖面 → 表层/深层/核心价值观/恐惧/内在矛盾
4. 维度 5 语言风格 → 说话方式/句式特征/OOC 红线
5. 维度 7 人际关系 → 与关键角色的互动模式
6. 维度 6 能力与限制 → 擅长/不擅长/能力边界
7. 维度 8 角色弧线 → 起点/终点/每卷目标
8. 维度 4 心理动力结构 → 防御机制/情感基线（可选，深度角色才填）
9. 维度 9 名场面 → 暂空（写了才有）
10. 维度 10 语言指纹 → 暂空（写了才有，后续从正文中提取）

**每个维度的填充方式**：
1. 展示维度模板结构（从角色档案模板中引用）
2. 提供 1-2 个参考角色的做法作为启发
3. 提供 AI 建议措辞（标记"🤖 AI建议"）
4. 作者确认或自行修改

### 步骤 5：差异化检查

**目标**：确保作者的角色是对参考角色的"改编参考"而非"复制粘贴"。

**提问**：
```
最后一个问题——你的 [角色名] 和 [参考角色] 最大的不同是什么？

不需要很复杂，一两句话就可以。这个答案会写进角色档案里，以后每次写作时都会提醒你：这个角色不是 [参考角色] 的翻版。
```

**如果作者回答不上来**：逐维提示差异点——
- "TA 们面临的处境有什么不同？"
- "TA 们做选择的方式有什么不同？"
- "TA 们最在乎的东西有什么不同？"

**记录**：将差异化答案写入角色档案的「角色原型 → 此角色与参考原型的差异」字段。

## 四、差异化检查清单

完成角色丰富后，逐项确认：

- [ ] 角色和参考角色是否面临**不同的核心困境**？（处境不同）
- [ ] 角色的**行为驱动力**是否和参考角色有本质区别？（动机不同）
- [ ] 角色的**说话方式**是否和参考角色有明显的不同特征？（声音不同）
- [ ] 角色的**关系模式**是否适应你故事的角色阵容而非参考作品？（关系不同）
- [ ] 角色的**缺点/局限**是否和参考角色不同？（弱点不同）
- [ ] 至少有一个维度是"参考角色没有、但你的角色有"的特征？

**如果以上任意一条无法通过**：回到步骤 4，为对应维度重新设计改编方案。

## 五、降级行为

### 本指南文件缺失

如果 `framework/guides/character-enrichment-guide.md` 不存在：
1. ⚠️ 标注"角色丰富指南缺失，降级为内嵌四步流程"
2. 使用内嵌流程：唤起 → 拆解 → 填充 → 差异化（跳过映射步骤）
3. 访谈质量受影响：缺乏结构化的维度映射，填充可能不够系统

### 作者无法提供参考角色

如果步骤 1 唤起失败（作者想不起任何参考角色）：
1. 跳过步骤 2-3，直接进入步骤 4（逐维填充）
2. 每个维度提供 2 个通用的叙事原型作为参考（从第二章原型表中选）
3. 步骤 5 改为："你的角色和典型的 [原型名称] 最大的不同是什么？"

### 角色档案模板缺失

如果 `framework/templates/characters/_template.md` 不存在：
1. ⚠️ 标注"角色档案模板缺失，使用内嵌简化模板"
2. 内嵌模板：基本信息/性格剖面/语言风格/能力与限制/人际关系/角色弧线（6 维而非 10 维）

## 七、内在驱动五步推导

> 参考：梁元·月食 活人化系统。此流程帮助作者从角色的人生目标逐步推导到此刻的具体行动，填充角色档案的"内在驱动（八维）"节。

### 核心原则

- 推导方向是从大到小、从远到近：Life Goal → Long-term Project → Schedule → At Hand → Urge → Move
- 每一步都是前一步的自然细化，不跳步
- 最终产出的 Move 必须是读者能看到的具体物理行动
- 作者随时可以跳过，不被阻塞

### 步骤 1：确定人生终极目标（Life Goal）

**目标**：让角色有一个比"推进剧情"更大的存在理由。

**引导**：
```
[name] 最终想要什么？不是这一章的目标——是ta整个人生的方向。
用一句话说出来。具体到能拍出一个画面。
❌ "变强" → ✅ "成为能让师父说出'你出师了'的人"
❌ "被爱" → ✅ "有一个人，ta不用说什么理由，ta就在那儿"
```

**如果作者卡住**：提示从以下方向选——被认可 / 被理解 / 归属某处 / 超越某人 / 证明某事是错的 / 完成某个未竟之事 / 守护某个人或地方

### 步骤 2：确定近期具体目标（Long-term Project）

**目标**：建立 Life Goal 与当前剧情时间线之间的桥梁。

**引导**：
```
为了实现 [Life Goal]，[name] 在接下来几周/几个月内具体在做什么？
这是一个可以被"完成"或"失败"的、有明确终点的目标。
它不需要今天就完成——但ta每天都在朝它移动（哪怕很慢）。
```

### 步骤 3：确定今日客观日程（Schedule）

**目标**：锚定角色在"被剧情找上门之前"的日常状态。

**引导**：
```
今天，在 [剧情事件] 发生之前，[name] 本来打算做什么？
ta有固定的日程吗——起床→去哪→见谁→做什么？
如果没有日程，ta今天最可能在什么时间出现在什么地方？
```

### 步骤 4：确定此刻正在做什么（At Hand）

**目标**：场景开始时角色不是一张白纸——ta已经在做某件事。

**引导**：
```
场景开始的那一刻，[name] 正在做什么？
不是"等待剧情发生"——是ta自己的生活中正在进行的具体动作。
可以很小：整理书架、削铅笔、盯着窗外出神、反复擦同一个杯子。
```

### 步骤 5：确定此刻冲动与行动（Urge → Move）

**目标**：产生驱动场景中角色主动行为的核心欲望。

**引导**：
```
在这一刻，[name] 最自私的欲望是什么？不是"剧情需要ta做什么"——
是ta的身体/情绪/好奇心在催促ta做什么？

这个欲望必须：
- 是主动的（不是对别人的反应）
- 是具体的（不是"想变强"，是"想现在就试试新招"）
- 可能有破坏性（可以不考虑后果——那是 Evaluation 要检查的）

为了满足这个欲望，ta此刻采取了什么具体行动？
这个行动必须是读者能看到的外部行为。
```

**如果作者犹豫**：提示欲望可以从以下方向抽取——
- 身体驱动：想离开/想留下/想吃/想休息/想动一动
- 情感驱动：想被注意/想被忽略/想报复/想道歉/想确认
- 认知驱动：想弄清楚/想验证/想推翻/想比较
- 社交驱动：想拉近/想推远/想测试/想挑衅/想和解

### 步骤 5b：冲动评估（Evaluation）——可选

```
检查：这个 Urge 与 [name] 的性格/既有设定矛盾吗？
如果矛盾——哪个优先级更高？ta倾向于自己解决还是求助？
```

### 产出格式

完成后写入角色档案的"内在驱动（八维）"节。每次场景结束后，通过 `settings-manager` (record-character-state) 更新 At Hand 和 Urge。
