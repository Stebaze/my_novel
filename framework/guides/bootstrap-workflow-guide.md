# Bootstrap 工作流 — 逆向工程方法论

> **定位**：本文件是 `bootstrap-project` Skill Phase 1 的核心分析方法论。Skill 在执行全书综合分析（Phase 1a-1e）时读取本文件获取具体的分析规则、检测算法和置信度标准。
>
> 本文件被 `bootstrap-project` Skill 直接引用。

---

## 一、大纲结构反向工程

### L1 全书级分析

**目标**：从已有正文中推断核心主题、终点方向和不可违背规则。

**分析方法**：

1. **核心主题推断**：
   - 扫描所有章节，提取角色反复思考/追问的问题（如"我到底是谁""力量的意义是什么"）
   - 提取叙事中反复出现的意象和隐喻（如"镜子""门""光与暗"）
   - 提取角色在关键时刻做出的选择模式（牺牲什么/保护什么/放弃什么）
   - 将高频主题词聚类，提出 1-3 个候选核心主题

2. **终点方向推断**：
   - 提取主角最强烈的 2-3 个目标陈述（在想/在说/在为之行动的目标）
   - 提取叙事中的"承诺线索"——作者在开篇暗示的最终要解决的问题
   - 推断终点状态：主角现在缺什么 → 终点时应该获得/放下什么

3. **不可违背规则推断**：
   - 提取正文中明确陈述的能力边界（"XX 永远不能 YY"类的表述）
   - 提取叙事逻辑中的硬约束（如"灵格不可转移""只能通过入睡触发"）
   - 标注为"AI 推断"——这些规则最终需要作者确认

**输出格式**：
```markdown
### L1 全书级分析

- **候选核心主题**：[主题] — 置信度：[很高/高/中/低]
  - 原文证据：Ch{X} "[引用]"；Ch{Y} "[引用]"
- **终点方向建议**：[描述] — 置信度：[很高/高/中/低]
- **推断的不可违背规则**：
  1. [规则] — 📖 原文明确 / 🤖 AI推断
```

### L2 分卷级分析

**目标**：检测卷边界位置。

**叙事闭合信号检测**（满足 ≥2 个即标记为候选卷边界）：

| 信号类型 | 检测方法 | 权重 |
|---------|---------|:--:|
| 时间跳跃 | 章节间时间跨度 ≥ 3 天 或 出现"三个月后"类明确标记 | 高 |
| 状态总结 | 章末尾有角色对当前阶段的总结性思考（"这一路走来""从那天起"） | 高 |
| 情感收束 | 章末尾情感弧线完整闭合——问题解决 + 情绪释放 + 过渡到新状态 | 高 |
| 场景密度骤降 | 连续 2 章事件密度明显低于前序章（从"每章 3-4 场景"降到"1-2 场景"） | 中 |
| 新角色/新设定的批量引入 | 2 章内引入 ≥3 个新角色或新设定，暗示新阶段开始 | 低 |

**卷主题推断**：对每个候选卷，提取该卷内重复出现的关键词和核心冲突类型，形成卷主题建议。

**输出格式**：
```markdown
### L2 卷边界建议

| 卷 | 建议范围 | 闭合信号 | 置信度 | 建议主题 |
|:--:|---------|---------|:-----:|---------|
| 1 | Ch1-Ch{N} | 时间跳跃 + 情感收束 | 高 | [主题建议] |
```

### L3 篇章级分析

**目标**：在每卷内检测 5-7 章的篇章分组。

**篇章边界信号**：
- 弧线闭合：一个小型情节线从头到尾完整闭合（问题提出→尝试→解决）
- 情感拐点：连续 2 章的情感曲线从上升转为下降（或反之）
- 关系里程碑：关键角色关系发生质变（建立/升级/转折/断裂）
- 场景节奏变化：场景切换频率明显变化

**输出格式**：
```markdown
### 第一卷篇章分组

| 篇章 | 章位 | 篇章功能 | 核心问题 | 置信度 |
|:--:|------|---------|---------|:-----:|
| 1 | Ch1-Ch{N} | [开端/发展/转折/高潮/收束] | [一句话] | |
```

## 二、节拍映射与高潮诊断

### 五阶段节拍映射

**映射规则**：对每章判断其在五阶段序列中的位置。

| 阶段 | 特征 | 检测信号 |
|------|------|---------|
| 开端 | 建立场景、引入冲突、信息铺垫 | 新信息密度高、行动密度低、对话以解释为主 |
| 发展 | 冲突升级、压力增加、退路封死 | 事件密度上升、角色选择空间缩小、新障碍出现 |
| 高潮 | 冲突到达峰值、不可逆变化发生 | 事件密度最高（≥3 件有意义事件/千字）、节奏最快、关键信息被揭示 |
| 结局 | 核心冲突解决、即时后果显现 | 事件密度骤降、总结性语言增加、"解决"类动词出现 |
| 余震 | 情感消化、新种子埋设 | 反思段落、情感描写密集、新线索/新问题被暗示 |

**检测算法**：
1. 逐章统计事件密度（有意义事件数 / 千字）
2. 找到事件密度峰值位置 → 标记为高潮区域
3. 向前追溯压力累积起点 → 标记为开端
4. 向后追踪事件密度下降至基线 → 标记为结局+余震

### 高潮峰值检测

**检测指标**：
- **事件密度**：高潮段 > 3 件有意义事件/千字（日常段基线 ~1-1.5）
- **句子长度变化率**：高潮段平均句长比全章均值短 30%+（短句加速）
- **信息揭示密度**：高潮段内新信息揭示 ≥ 2 条
- **不可逆变化标记**：高潮段后至少有 1 个永久状态变化

### 机制类型匹配

对检测到的每个高潮桥段，匹配到 6 种机制类型：

| 机制类型 | 识别特征 |
|---------|---------|
| information_reveal | 高潮峰值是一个关键信息的揭示——"原来是这样"类的认知颠覆 |
| combat_decision | 高潮峰值是战斗决胜——以武力/策略击败对手 |
| emotional_breakthrough | 高潮峰值是情感突破——角色终于承认/接受/放下某件事 |
| multi_thread_convergence | 高潮峰值是多条线索同时收束——≥3 条支线在同一场景交汇 |
| character_choice | 高潮峰值是角色的重大抉择——在两种价值观/两条路径之间做决定 |
| power_breakthrough | 高潮峰值是能力突破——角色突破自身极限获得新能力 |

### 质量诊断维度

对每个高潮桥段，对照指导者模板（如有）诊断以下维度：

| 诊断维度 | 计算方式 | 健康范围 |
|---------|---------|---------|
| 蓄势充分度 | 发展段章节数 / 高潮段章节数 | small 桥段 ≥1:1 / medium ≥3:1 / large ≥4:1 |
| 高潮峰值高度 | 高潮段事件密度 / 全卷平均事件密度 | ≥1.5× |
| 余震到位度 | 余震段是否有 ≥1 个情感消化段落 | 必须有 |
| 升级机制多样性 | 发展段中使用了多少种不同类型的压力叠加 | ≥2 种 |
| 句长加速比 | 高潮段平均句长 / 全章平均句长 | ≤0.7（句子明显缩短） |

## 三、设定提取与分类

### 三层来源分类标准

| 来源 | 标记 | 判定标准 |
|------|:--:|---------|
| 正文明确陈述 | `📖 原文明确` | 原文中直接陈述的事实——有明确的陈述句（"XX 是 YY"、"XX 不能 YY"），无歧义 |
| AI 从正文推断 | `🤖 AI推断` | 根据上下文推断的信息——需要结合多处文本才能得出结论，或原文暗示但未明说 |
| 作者脑内补充 | `💭 作者补充` | 作者在访谈中主动提供的、尚未在正文中出现的设定 |

### 关键设定判定规则

满足以下任一条件的设定为"关键设定"，需主动向作者求证：

1. **能力边界**：角色的能力限制、触发条件、代价/副作用
2. **世界规则**：世界观的基本运行规则（灵气/魔法/科技的运作方式）
3. **势力关系**：组织/阵营之间的同盟/敌对/中立关系
4. **核心谜团**：驱动主线的情节谜团（古镜来历/灵格觉醒条件等）
5. **角色上限**：角色的最终能力天花板
6. **规则例外**：正文中出现过"唯一例外"或"特例"表述的规则

### 差异检测流程

```
对每个设定条目：
  1. 提取"正文中实际呈现的内容"（从 Phase 1d 扫描结果）
  2. 对比"作者确认的意图"（从 Phase 2a 访谈结果）
  3. 标注差异类型：
     - 一致 ✅
     - 补充：作者意图比正文更丰富 → 标记为"待补充到正文"
     - 矛盾：正文和作者意图冲突 → 🟡 提醒作者选择最终版本
     - 未呈现：作者意图未在正文中出现 → 标记为"待引入章节"
```

## 四、指导模式差距诊断

### 对比维度

| 维度 | 对比什么 | 数据来源 |
|------|---------|---------|
| 高潮布局 | 作者的节拍占比 vs 指导者模板的同类型占比 | Phase 1b + climax-patterns |
| 升级机制 | 作者的压力叠加方式种类 vs 指导者模板的多样性 | Phase 1b + climax-patterns |
| 余震处理 | 作者的余震段长度和内容 vs 指导者模板 | Phase 1b + climax-patterns |
| 角色塑造 | 作者的角色完整度 vs 指导者作品中的角色塑造方法 | Phase 1d + author profile |
| 叙述声音 | 作者的叙述者人格一致性 vs 指导者的叙述声音特征 | Phase 1c + author profile |

### 差距分类规则

| 类别 | 判定标准 | 处理方式 |
|------|---------|---------|
| 设计问题 | 差距体现在结构层面（蓄势比/事件密度/升级机制）——可以通过调整写作策略改善 | 提供指导者技法建议，作者决定是否采纳 |
| 声音问题 | 差距体现在叙述者人格/语言指纹/情感表达方式——触及作者的创作身份 | 标注但不强制修改，提示"可以考虑" |

### 严重度标准

| 级别 | 条件 | 示例 |
|:--:|------|------|
| 🔴 结构性 | 可能影响读者留存的问题 | 连续多章无钩子、高潮后无余震直接跳日常、所有高潮同一机制类型 |
| 🟡 技法可优化 | 影响阅读体验但不致命 | 升级机制单一、信息释放节奏平、余震段太短 |
| ⚪ 风格偏好 | 不影响功能，纯审美选择 | 高潮偏好情感突破型 vs 战斗决胜型、日常比例偏好 |

## 五、置信度标注方案

所有 Phase 1 分析产出统一使用以下置信度体系：

| 置信度 | 条件 | 原文证据要求 |
|:----:|------|-----------|
| 很高 | ≥3 处独立原文证据支持同一结论，且来自不同章节 | ≥3 条引用 |
| 高 | ≥2 处原文证据支持，或单处证据非常明确（如"XX 是 YY"的直接陈述） | ≥2 条引用 |
| 中 | 1 处原文证据 + 合理的叙事逻辑推断，但证据不够充分 | 1 条引用 + 推断逻辑说明 |
| 低 | 仅基于叙事惯例推断，无直接原文证据 | 推断逻辑说明 |

**标注格式**：
```
[结论描述] — 置信度：高
  原文证据：
  - Ch{X}："[引用原文]"
  - Ch{Y}："[引用原文]"
```

## 六、降级行为

### 数据不足降级

| 条件 | 影响维度 | 降级行为 |
|------|---------|---------|
| < 3 章 | 全部 | 标注"⚠️ 样本不足（N={N}章），分析结果仅供初步参考。建议 ≥10 章后重新 bootstrap。" |
| 3-9 章 | 全部 | 所有置信度上限降一级（最高为"高"）。标注"⚠️ 样本偏少，置信度受限。" |
| 10-29 章 | L3 篇章级 | L3 置信度上限降为"中"。标注"⚠️ 篇章级分析在样本中等时可靠性有限。" |
| ≥30 章 | 无降级 | 全维度正常执行 |

### 组件不可用降级

| 缺失组件 | 影响 | 降级行为 |
|---------|------|---------|
| 指导者档案 | Phase 1e | 跳过 1e，标注"指导者未指定或无档案，跳过技法差距诊断" |
| climax-patterns/ | Phase 1b (d: 模板对比) | 仅做机制类型识别+节拍映射，不做模板对比诊断 |
| author-profiling Skill | Phase 1c | 使用内嵌简化提取（仅基于 Read 的统计分析） |
| technique-selector Skill | Phase 1b (技法推荐) | 仅输出差距列表，不提供具体技法建议 |
