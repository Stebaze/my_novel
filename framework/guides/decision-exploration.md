# 情节决策探索方法

> **定位**：当用户对情节方向不确定时，Agent 使用本文档中的方法帮助用户探索可能性、
> 理解每种选择的后果、并做出有根据的决策。
> 本文档是 Agent 的"思维工具箱"——不是给用户看的，是给 Agent 用的。

---

## 核心原则

1. **不做选择，而是展示选择的后果。** Agent 永远不说"你应该选 A"。Agent 说"如果选 A，接下来 5 章会是这样。如果选 B，会是这样。"
2. **先发散，后收敛。** 先帮用户看到所有可能性（包括用户没想到的），再帮用户排除不合适的。
3. **用故事的方式探索故事。** 不要写优缺点列表。写 100 字的"如果选了这条路，下一章的开头会是什么样"。

---

## 方法一：后果涟漪（Consequence Ripple）

### 适用场景
用户在 A 和 B 两个（或多个）情节走向之间犹豫。

### 执行步骤

1. **画出每个选项的"涟漪图"**（用文字，不用图）：
   - 第一层涟漪：选择后对下一章的直接影响（谁知道了什么/谁和谁的关系变了/什么事件被触发了）
   - 第二层涟漪：对接下来 3 章的影响（新的冲突出现了吗/哪个角色因此被推到前台/什么伏笔可以被触发）
   - 第三层涟漪：对这一卷结局的影响（这个选择会把故事带向哪个方向的终点）

2. **写一个 100 字的"选择 A 的第一章开头"**——用具体的场景画面，不是抽象描述。

3. **重复步骤 2 给选项 B。**

4. **追问**："这两个 100 字，哪个让你更想继续写下去？"

### Agent 输出格式

```
## 选项 A：[一句话描述]

**涟漪效应：**
- 下一章 → [具体影响]
- 接下来 3 章 → [连锁反应]
- 卷结局 → [最终方向]

**下一章的第一段**（如果选 A）：
> [100字场景开头]

## 选项 B：[一句话描述]

（同上结构）

## 追问
这两个开头段落，哪个让你更想写下去？为什么？
```

## 方法二：角色完整性检查（Character Integrity Check）

### 适用场景
用户不确定"这个角色在这种情况下会怎么做"。

### 执行步骤

1. **回溯角色档案中的决策依据**：
   - 核心价值观是什么？→ 这个情境触碰了哪条价值观？
   - 恐惧/弱点是什么？→ 这个情境是否触发了恐惧？
   - 内在矛盾是什么？→ 这个情境让矛盾的两端如何拉扯？

2. **角色声音模拟**：用角色的第一人称写 3-5 句内心独白——角色自己在做这个决定时脑子里在想什么？

3. **"如果选 A，角色会怎么评价自己的选择？如果选 B 呢？"** ——角色对自己的选择的评价，往往比选择本身更能揭示什么是对的。

### Agent 输出格式

```
## 角色 [名字] 的决策剖面

**被触碰的价值观**：[哪一条，怎么被触碰的]
**被触发的恐惧**：[哪一条，程度如何]
**内在矛盾的拉扯**：[两端分别是什么]

**角色自己的声音**（第一人称内心独白）：
> "[3-5句]"

## 追问
听完她/他自己的声音——你觉得她会怎么选？
```

## 方法三：类型期待地图（Genre Expectation Map）

### 适用场景
用户在"满足读者期待"和"反套路"之间犹豫。

### 执行步骤

1. **标注当前位置的类型期待**：在这个类型的这个位置，读者通常预期发生什么？
2. **定义"满足期待"的版本**：如果按读者预期走——会是什么场景？读者的情感回报是什么？
3. **定义"反套路"的版本**：如果打破预期——扭转的方向是什么？读者会感到惊喜还是困惑？
4. **定义"第三种路"**：既不完全满足也不完全打破——微调期待中的某一个元素，让熟悉的东西产生新的味道。

### Agent 输出格式

```
## 当前位置的类型期待

读者在这个位置通常会预期：[描述]

## 路径 A：满足期待
- 场景：[简述]
- 读者得到：[情感回报]
- 风险：可预测

## 路径 B：反套路
- 场景：[简述]
- 读者感到：[惊喜/困惑/新鲜]
- 风险：如果处理不好 → 读者觉得"不合理"

## 路径 C：熟悉的陌生感
- 保留 [X 元素]，但改变 [Y 元素]
- 读者感到：熟悉但新鲜

## 追问
你现在更需要"稳"（A）还是"破"（B），还是"稳中求变"（C）？
```

## 方法四：主题对齐检验（Theme Alignment）

### 适用场景
用户面对多个"都可以"的选择时，用主题作为最终裁决。

### 执行步骤

1. 回溯 L1 全书级的**核心主题**。
2. 对每个选项追问：**"这个选择让主角更接近核心主题的实现，还是更远离？"**
3. 有时候，让主角"远离"主题的选择反而是正确的——因为戏剧张力来自偏离和回归。

### Agent 输出格式

```
## 主题对齐检查

核心主题：[L1 主题]

选项 A → [更接近/更远离] 主题，因为 ____
选项 B → [更接近/更远离] 主题，因为 ____

**注意**：如果两个选项都接近主题，选择让主角"付出更大代价"的那个——代价越大，读者的情感投入越深。
如果两个选项都远离主题，追问自己"我是在写一个必要的偏离（为了后面的回归更有力），还是在跑题？"
```

## 方法五：消失的场景（The Missing Scene）

### 适用场景
用户觉得"这段剧情不对，但说不出来哪里不对"。

### 执行步骤

1. **问用户**："如果这一章必须删掉一个场景，你会删哪个？"
2. 用户指出来后，**问**："这个场景本来想达成什么？实际上达成了吗？"
3. **然后问**："如果在这个位置加一个现在不存在的场景——你最想看到什么？"
4. 这个"想看到但不存在"的场景，往往就是问题所在。

### Agent 输出格式

```
## 场景诊断

**如果必须删一个场景**：[用户指出的场景]

**这个场景想达成什么？**
**实际达成了吗？**

**如果在这里加一个新场景——你最想看到什么？**
→ [这个"想看到但不存在"的场景，就是缺失的那块拼图]

## 建议
考虑用 [新场景思路] 替代 [旧场景]，或者插入在 [位置]。
```

## 方法六：参考三角定位（Reference Triangulation）

### 适用场景
用户需要一个外部参考来打破思考惯性。

### 执行步骤

1. 找到当前情境在三部参考作品中最接近的对应场景。
2. 分析每部作品的处理方式。
3. 指出用户的角色/故事与哪部参考作品的 DNA 最接近——但不等于要复制那个处理方式。

### Agent 输出格式

```
## 参考三角定位

**当前情境**：[描述]

| 参考作品 | 类似情境 | 处理方式 | 为什么这样处理 |
|----------|----------|----------|---------------|
| 桌游咖 | [场景] | [处理] | [角色性格驱动] |
| 败犬女主 | [场景] | [处理] | [叙事基调驱动] |
| 福尔摩斯小姐 | [场景] | [处理] | [类型期待驱动] |

**你的故事 DNA 最接近**：[参考作品]，因为 [理由]。
但你的不同之处在于 [差异]——这意味着你可以 [调整方向]。

## 追问
参考给了你思路——但你的故事不是参考的复制。你想往哪个方向微调？
```

## 方法七：双写采样（Dual-Write Sample）

### 适用场景
用户在两个选项之间完全无法判断，需要"手感"来做决定。

### 执行步骤

1. 用**高纹理模式**快速生成两个版本的 200 字场景开头。
2. 不分析。直接问用户："哪个读起来更顺？更让你想接着写？"
3. 通常——写作"手感"会在 200 字内显现。写起来更顺畅的那条路，往往是更接近角色声音的那条路。

### Agent 输出格式

```
## 选项 A 的 200 字开头

> [200字场景]

## 选项 B 的 200 字开头

> [200字场景]

## 追问
不分析。就凭感觉——哪个让你更想接着写下去？
```

## 方法选择指南

Agent 在遇到用户不确定时，按以下逻辑选择方法：

```
用户说"我不确定这里该怎么发展"
    ↓
问自己：用户不确定的是什么？
    ↓
├─ "两个选项之间不知道怎么选" → 方法一（后果涟漪）+ 方法七（双写采样）
├─ "不知道这个角色会怎么做" → 方法二（角色完整性检查）
├─ "怕读者不喜欢" → 方法三（类型期待地图）
├─ "两个选项都行" → 方法四（主题对齐检验）
├─ "感觉不对但说不出来" → 方法五（消失的场景）
├─ "想不出新的可能性" → 方法六（参考三角定位）
└─ 还是不确定 → 方法七（双写采样）——写出来就知道了
```
