# 中国网文轻小说写作指南

> 基于「福尔摩斯小姐必不可能成为败犬！」的系统分析，结合中国网文平台的出版与更新机制，提炼的实战写作要求。与 `novel/style-guide.md` 互补——前者关注"写得好看"，本指南关注"写得能活"。

## 一、平台生存法则

中国网文平台（起点、刺猬猫、SF轻小说等）的核心机制决定了写作策略：

### 1.1 前三章定生死

读者给你三章的时间说服他留下来。这三章必须依次完成：

| 章节 | 必须完成的任务 | 具体标准 |
|------|---------------|----------|
| **第一章** | 建立世界 + 主角 + 核心钩子 | 读者读完后知道"这是个什么世界"、"主角是什么人"、"为什么我要继续读" |
| **第二章** | 展开系统/能力 + 第一个冲突 | 展示主角的独特之处（系统/金手指/特殊能力）、引入第一个让读者关心的矛盾 |
| **第三章** | 引出主要女性角色 + 确立作品基调 | 至少一个女性角色要有"记忆点"（反差/名场面/独特互动）、读者能判断"这书是什么味" |

**实战检验标准**：把前三章给一个没听过这本书的人看，如果他在第三章结束时不问"然后呢？"——前三章不合格。

### 1.2 日更节奏与章节设计

网文平台以日更为基本节奏。每章 2000-4000 字是标准区间，**硬上限 4500 字**——超过此限制的章节必须在写作阶段或初稿完成后拆分。

**分章规则**：

当一章超过 4500 字时，按以下优先级寻找拆分点：

1. **自然场景转折**：时间跳转（"第二天早晨"）、地点切换（"回到棚户区"）、POV 切换
2. **情感节拍断点**：一个完整的情绪弧线结束后——读者刚经历了一次完整的情感起伏，此时断开不会破坏沉浸感
3. **信息揭示后**：角色/读者刚知道了一个关键信息，下一章将以这个信息为起点展开
4. **`---` 分隔线**：如果正文中已经用 `---` 分隔了场景，它通常是天然的分章点

**拆分后处理**：
- 新章开头必须有一个 1-2 句的微型钩子，让接续阅读的读者立刻回到场景中
- 如果拆分点在情感高潮处，新章开头先用半句话回收上一章的情绪，再推进
- 拆分后的两章各自必须有独立的爽点/甜点——不能出现"前半章有爽点，后半章没有"的情况

**一章 = 一个完整的"小呼吸"**：
- 开头有一个小钩子（可以接续上一章的悬念，也可以新抛一个问题）
- 中间有至少一个"爽点"或"甜点"（战斗胜利/智力碾压/角色互动糖分）
- 结尾必须**卡高潮**（断章）——让读者想点下一章

**断章技法**：
- 在答案即将揭晓的前一句断开
- 在角色即将说出关键台词时断开
- 在战斗/对抗的胜负即将分晓时断开
- 在两个角色关系即将发生质变时断开
- 注意：断章不是耍赖——你必须在下一章真正给出答案，否则会消耗读者信任

### 1.3 推荐票/月票的周期性

网文的曝光机制（推荐位、榜单）以周为单位运作。这意味着：
- 每周应有一个**小高潮**（刺激读者投票和追读）
- 每卷（约 30-50 章）应有一个**大高潮**
- 低谷期不宜持续超过 3 章（日更三天 = 现实时间三天，读者已经开始不耐烦）
- 周末和周一是流量高峰，重要章节安排在此时发布效果更好

## 二、标题与简介

### 2.1 书名设计

网文书名承担着"标签化"功能——要在 10 个字以内让读者知道：
1. 这是什么类型（轻小说/同人/原创/系统/恋爱）
2. 核心卖点是什么（败犬/反派/逆袭/修罗场/）
3. 有什么独特之处

**成功模式**：
- **反差型**：「福尔摩斯小姐必不可能成为败犬！」——经典角色 + 意外属性 + 否定句制造悬念
- **宣言型**：「败犬女主太多了！」——直接宣告核心概念
- **身份型**：「关于我转生变成史莱姆这档事」——一句话概括全书设定

### 2.2 简介（ Synopsis）结构

参考《福尔摩斯小姐》的简介，典型的"三段式简介"：

```
第一段（引子）：用一段有冲击力的引语/场景/对话抓住注意力
    "我不明白！为什么大家都在讨论福尔摩斯小姐是败犬！"

第二段（核心设定）：简明扼要地说明本书的独特设定
    穿越 + 演绎系统 + 多重身份 + 多位女主角

第三段（钩子）：暗示核心冲突和情感张力，留白
    "作为侦探的绝对财产，由此产生的压抑……"
```

**简介禁忌**：
- ❌ 不要用"且看主角如何……"的老套句式
- ❌ 不要在简介里堆设定（设定应该在三章内展示，不是介绍）
- ❌ 不要剧透关键剧情
- ✅ 用角色对话/场景片段代替抽象介绍
- ✅ 在简介末尾留一个疑问或张力点

## 三、人物模板化与快速导入

### 3.1 为什么需要模板化

网文读者每天阅读大量作品。模板化不是贬义，而是**效率工具**——让读者在看到角色的 30 秒内就能建立初步印象，然后通过后续描写让角色"破模板而出"。

### 3.2 核心角色模板

**男主模板：身份转换型**（参考：卢希安）
- **表层身份**：普通人/学生/不起眼的角色（"文学院小透明"）
- **隐藏身份**：拥有系统/特殊能力的强者（"犯罪界的拿破仑·莫里亚蒂"）
- **身份切换触发器**：一个具体的视觉/动作标志（戴眼镜/摘眼镜、穿围裙/脱围裙），像开关一样即时切换两种状态
- **魅力来源**：反差——日常模式越是平凡普通，暗面模式越要保持优雅和强大
- **读者代入路径**："我也想有一个开关，在平凡日常和非凡冒险之间自由切换"

**女主模板 A：反差型**（参考：玛丽·摩斯坦）
- **表层**：柔弱/天真/需要保护（"跪在地毯上颤抖的金发少女"）
- **深层**：腹黑/控制狂/一切都在计算中（"哪里还有半点泪痕？翠绿色的眼眸里平静得令人毛骨悚然"）
- **反差揭露时机**：在第一卷中段揭露，不要太早（失去冲击力）也不要太晚（读者已经定型印象）
- **对男主的态度**：看似被保护，实则在操纵；但操纵的目的不是害男主，而是把男主拉入她的故事
- **读者情感路径**：怜悯 → 震惊 → 着迷

**女主模板 B：冷娇型**（参考：夏洛特·福尔摩斯）
- **表层**：理性至上/情感无能/智商碾压（"情感是失败者身上的一种化学缺陷"）
- **深层**：比她自己承认的更需要某人（"棒棒糖转得慢了半拍"）
- **表达情感的独特方式**：不是说我爱你，而是"你可以帮我记录通风管道位置"
- **读者情感路径**：敬畏 → 觉得可爱 → 心疼

**配角快速建立法**：
- 一个视觉特征 + 一个性格标签 = 五秒内建立印象
- 杰基尔 = 厚底眼镜 + "像受惊兔子般的谨慎"
- 莫兰 = 一本小册子 + "表面管家实则全能父亲角色"
- 不要怕配角模板化——配角首先要功能清晰，然后才谈深度

### 3.3 模板化的进阶：破模板

模板让人记住角色，但让人爱上角色的，是破模板的时刻。
- 腹黑女主在某一刻真的感到害怕/动摇
- 冷娇侦探在某一刻笑了一下
- 弱气室友在某一刻展现了惊人的勇气

**破模板法则**：每卷给主要角色至少一次"超出读者预期"的时刻。但破模板必须在角色档案的"内在一致性"范围内——不能为了破而破。

## 四、情节节奏与爆点管理

### 4.1 爆点密度要求

网文读者的耐心以"页"为单位。参考密度：

| 节奏要素 | 最低频率 | 最优频率 |
|----------|----------|----------|
| 小爆点（令人满意的台词/小反转/糖分互动） | 每章 1 次 | 每章 2-3 次 |
| 中爆点（能力展示/智力碾压/关系推进场景） | 每 3 章 1 次 | 每 2 章 1 次 |
| 大爆点（身份揭露/重大反转/情感突破） | 每卷 2-3 次 | 每卷 3-4 次 |

### 4.2 爆点类型库

1. **智力碾压型**：主角用推理/计谋击败远超自己的对手（侦探类核心爽点）
2. **身份暴露型**：主角的隐藏身份（强大那一面）被人发现——或者差点被发现
3. **反差萌型**：强势角色露出脆弱一面 / 弱势角色展现惊人实力
4. **修罗场型**：两个以上女性角色在同一场景中与男主互动，产生化学反应
5. **名场面型**：一句特别精彩的台词，或一幕特别有画面感的场景，让读者想截图分享
6. **糖分爆发型**：男女主之间突然的亲密接触/真心话/独处时刻

### 4.3 情绪曲线管理

一卷（约 30-50 章）的情绪曲线应该是波浪形的：

```
紧张        ──╮         ╭──         ╭──╮
              ╰──╮    ╭──╯    ╰──╮  │  ╰──╮
日常  ──╮       ╰────╯            ╰──╯     ╰────
        ╰──────────────────────────────────────
        开篇  展开    中段高潮   过渡   终章高潮
```

**关键规则**：
- 连续紧张不能超过 3 章（读者需要呼吸）
- 连续日常（无推进）不能超过 2 章（读者会觉得水）
- 高潮之后的"冷却章"长度 = 高潮强度的 1/2（大高潮后可以缓一章，小爆点后缓半章）

## 五、类型融合与标签策略

### 5.1 成功作品的类型公式

《福尔摩斯小姐》的类型公式 = 福尔摩斯同人 × 系统流 × 恋爱喜剧 × 悬疑推理 × 多重身份

这意味着它同时在五个类型标签下吸引读者。你的作品也应该有 3-5 个类型的交叉。

**常用类型组合**：
- **恋爱喜剧 + 系统** → 系统提供目标和量化成长，日常提供互动和糖分
- **同人 + 原创女主角** → 借用原著世界观降低理解门槛，原创角色提供新鲜感
- **悬疑 + 恋爱** → 破案过程展示男主能力，案件间隙发展感情
- **校园 + 非日常** → 日常场景提供亲切感，隐藏的非日常身份提供张力

### 5.2 标签的写作学意义

标签不只是给读者搜索用的——每个标签对应着特定的读者期待和爽点模式：

| 标签 | 读者期待 | 你必须在早期章节交付的 |
|------|----------|------------------------|
| 系统流 | 看到系统面板、量化成长、技能升级 | 第一章出现系统面板，前五章内完成第一次升级/抽卡 |
| 扮猪吃老虎 | 看到主角在公共场合被低估，然后反击 | 前五章内必须有一次"反转装逼" |
| 修罗场 | 看到多个女性角色因男主产生互动/冲突 | 前两卷内必须有至少一次三方同框的修罗场景 |
| 傲娇 | 看到傲的角色在娇的瞬间动摇 | 第一卷内必须有至少一次明显的"娇"的泄露 |
| 腹黑 | 看到角色在表象之下的算计 | 第一卷前中段揭露反差 |

## 六、对话与场景的网文优化

### 6.1 移动端阅读适配

大多数网文读者用手机阅读。这意味着：
- 段落尽量不超过 4 行（手机上约 120 字）
- 对话密集的场景天然适合手机阅读
- 大段景物描写和心理独白在手机上是"劝退墙"
- 适当用空行分隔场景内的节奏变化

### 6.2 场景切换的频率

网文场景的"镜头"切换应该比传统小说更快：
- 一个场景内不宜超过 1500 字（除非是全书高潮）
- 场景之间用简洁的过渡（"三分钟后"、"与此同时"、"贝克街的早晨"）
- 可以在同一章内切 2-3 个场景，保持信息流动
- 多线叙事时，每条线在一章内至少出现一次

### 6.3 网文特有的对话技法

**信息嵌入法**：在对话中嵌入世界观信息，而不是用旁白介绍。
```
❌ 旁白：伦敦的煤气灯在1881年还是主流照明方式。
✅ "这该死的煤气灯又灭了。"雷斯垂德骂了一句。
```

**弹幕思维**：想象读者读到这一行会发什么弹幕/评论，在文中预先回应。
- 男主做出帅气行为 → 下一句立刻让女主吐槽，替读者"破功"
- 角色说了一句容易引发误解的话 → 立即让另一个角色误解，制造喜剧效果
- 这种"弹幕预判"会让读者觉得作者和他们在同一频道

## 七、写作质量检查清单（网文版）

写完每一章后对照：

### 钩子检查
- [ ] 这一章的第一段能让人想继续读下去吗？
- [ ] 这一章的结尾是一个有效的断章吗？（卡高潮/悬念/情感揭示）

### 爽点检查
- [ ] 这一章至少有一个"爽点"或"甜点"吗？
- [ ] 如果没有大爽点，有没有小爽点（精彩的台词/反转/互动）？

### 节奏检查
- [ ] 场景切换是否频繁到不沉闷？
- [ ] 有没有超过 500 字的连续纯描写/纯独白？
- [ ] 对话比例是否超过 50%？

### 人物检查
- [ ] 新角色出场时是否有"记忆点"？（一个视觉特征 + 一个性格标签 + 一个有辨识度的台词）
- [ ] 主要角色这一章有没有展现出"超出预期"的瞬间？

### 追读检查
- [ ] 假想你是读者，读完这一章你会点"下一章"吗？
- [ ] 如果不会，缺的是什么？补充它再发。
