# AVG 剧本格式规格

> **定位**：本文档定义引擎无关的 AVG 剧本写作格式。剧本是人类可读的标记文本，类似舞台/影视剧本，不绑定任何特定引擎。聚焦于文字剧本层面的设计。

---

## 一、设计原则

1. **人类可写可读**：格式应像舞台剧本一样自然，不需要编程知识即可编写
2. **包含演出信息**：标注立绘变化、BGM、转场等演出指令，但不涉及引擎具体实现
3. **支持分支**：选择肢、条件跳转、flag 修改——剧本的文字层面必须覆盖
4. **AI 友好**：结构化标签方便 AI 理解和生成，同时保持足够的自然语言密度

---

## 二、块类型

AVG 剧本由以下块类型组成：

| 类型 | 语法 | 功能 |
|------|------|------|
| 场景头 | `【场景：地点·时间】` | 标记场景开始，确立空间和时间 |
| 环境指令 | `【BGM：曲名】` / `【bg：背景ID】` | 设定听觉/视觉环境 |
| 立绘指令 | `【立绘：角色 表情 位置 [入场方式]】` | 控制角色立绘的显示/切换 |
| 效果指令 | `【bgm 静默 N秒】` / `【特效：效果名】` | 时间控制/特殊效果 |
| 对话行 | `角色名 [表情]：「对话内容」 ⏱ N.Ns` | 角色对话（含立绘表情标注+配音时长估算） |
| 演出注 | `（注：动作/语气/心理描述）` | 对话下的表演指导，括号内 |
| 旁白块 | `【旁白】...【/旁白】` | 叙述性文字（场景描述/内心独白） |
| 检定块 | `【检定：技能 | 难度：NN | 结果】` | 跑团 Replay VN：骰子检定演出节点 |
| CG块 | `【CG：名称 | 触发时机】` + 生图提示词 | 剧情关键帧——全屏插画，用于名场面定格 |
| 选择块 | `【选择】` + 选项列表 | 玩家决策点（跑团 Replay VN 不使用） |
| 跳转指令 | `→ 场景_ID` | 剧情跳转目标（跑团 Replay VN 不使用） |
| Flag 标记 | `[flag名 操作]` | 条件/效果标记（跑团 Replay VN 不使用） |

## 三、详细语法

### 3.1 场景头

```
【场景：地点名称·时间段】
```

**规则**：
- 每个场景文件以场景头开始
- 地点名称应具体（"放课后的教室"而非"教室"）
- 时间段可选：清晨/午前/午后/黄昏/夜晚/深夜

**示例**：
```
【场景：校舍天台上·黄昏】
【场景：文化祭中庭·午后】
【场景：真由的房间·夜晚】
```

### 3.2 环境指令

```
【BGM：曲目标识】
【bg：背景标识】
```

**BGM 命名建议**：
- `日常轻松` / `日常温暖` / `日常慵懒`
- `喜剧诙谐` / `尴尬沉默`
- `感伤钢琴` / `紧张弦乐` / `温馨结尾`
- `静默`（无 BGM）

**背景命名建议**：
- 地点 + 时间段：`教室_黄昏`、`天台_午后`、`走廊_夜晚`
- 特殊场景：`文化祭_中庭`、`海边_傍晚`

**背景生图提示词**（本分支强制要求）：
- 每个 `【bg：背景ID】` 必须在章末「背景生图提示词索引」中提供对应的中文提示词
- 提示词可直接交由 GPT 图像生成模型使用
- 风格统一：日式动画/游戏背景（视觉小说风格），16:9 横版
- 提示词应包含：场景描述 + 光影氛围 + 关键视觉元素 + 构图/视角
- 已存在于其他章节的背景 ID 可直接复用，无需重复编写

### 3.3 立绘指令

```
【立绘：角色名 表情 位置 [入场方式] [退场方式]】
```

**表情列表**（通用表情集，每个角色可自定义扩展）：
| 通用表情 | 说明 | 败犬女主典型使用 |
|----------|------|-----------------|
| `默认` | 中性表情 | 普通对话 |
| `微笑` | 温和的笑 | 日常互动 |
| `苦笑` | 无奈的笑 | 吐槽/自嘲 |
| `开心` | 明显的喜悦 | 好事发生 |
| `生气` | 愤怒/不满 | 被惹到 |
| `害羞` | 脸红/回避视线 | 被戳中 |
| `悲伤` | 低沉/失落 | 谈及创伤 |
| `眼眶泛红` | 忍住不哭 | 苦涩瞬间 |
| `惊讶` | 瞪大眼睛 | 意外信息 |
| `无语` | 面无表情/死鱼眼 | 吐槽反应 |
| `坏笑` | 恶作剧表情 | 捉弄别人 |
| `认真` | 严肃/专注 | 重要宣言 |
| `逞强` | 假装没事但看得出 | 败犬女主高频使用 |

**位置**：`左侧` / `中央` / `右侧`

**入场/退场**（可选）：
- `滑入` / `淡入` / `弹出`
- `滑出` / `淡出`

**示例**：
```
【立绘：真由 苦笑 右侧】
【立绘：主角 无语 左侧】
【立绘：真由 眼眶泛红 右侧】
【立绘：千夏 开心 中央 滑入】
【立绘：真由 淡出】
```

### 3.4 效果指令

```
【bgm 静默 N秒】
【特效：效果名】
【停顿 N秒】
```

**使用场景**：
- `【bgm 静默 2秒】`——对话中情绪急转前的空白，制造"空气凝固"感
- `【停顿 1秒】`——角色说出重要台词前的蓄力
- **时长计入**：`【bgm 静默 N秒】`和`【停顿 N秒】`的 N 秒直接计入本章配音总时长

### 3.5 对话行

```
角色名 [表情]：「对话内容。」 ⏱ N.Ns
  （演出注：动作/语气/心理描述）
```

**规则**：
- 对话使用 `「」` 引号（中文剧本风格）
- **每句台词必须标注立绘表情**：角色名后紧跟 `[表情]`，标注当前立绘使用的表情差分。表情名与 §3.3 立绘指令的表情列表一致
- 演出注紧跟对话行下方，缩进两个空格，用括号包裹
- 演出注描述：语气、动作细节、心理活动、肢体语言
- 演出注不应复述对话内容——应补充对话之外的视觉/听觉信息
- **配音时长标注**：每行对话末尾追加 `⏱ N.Ns`，标注该句配音预估秒数（详见 §七）

**表情标注规范**：
- 同一角色连续多句台词使用相同表情时，每句仍需标注（确保视觉团队逐句可查）
- 表情变化时，需同时使用 `【立绘：角色 新表情】` 指令块 + 下一句对白的 `[新表情]` 标注
- 无立绘的纯语音角色（如电话/回想），标注 `[语音]`
- 表情列表见 §3.3 立绘指令

**时长估算公式**：
- **默认语速**：3.5 字/秒（中文配音自然语速，含换气停顿）
- **计算公式**：`对话内容字数（不含标点）÷ 3.5`，四舍五入到 0.1 秒
- **句末标点附加停顿**：`。！？` +0.3s / `……` +0.5s / `，、` +0.15s
- **最短时长**：任何对话行不得短于 0.5s
- **无配音行**：仅标点/省略号 `「……」` 不计时长，标注 `⏱ —`

**好的演出注**：
```
真由 [苦笑]：「我没事啦。」 ⏱ 1.4s
  （嘴上说没事，手指却在绞裙子。目光刻意避开主角）

主角 [无语]：「嗯，看起来确实像没事。」 ⏱ 2.1s
  （死鱼眼，棒读。把手插进口袋里望向窗外）
```

**不好的演出注**：
```
真由：「我没事啦。」 ⏱ 1.4s
  （真由说自己没事）——❌ 复述对话，无新信息

主角：「嗯，看起来确实像没事。」 ⏱ 2.1s
  （主角表示同意）——❌ 同上
```

**演出注的质量标准**（来自败犬女主）：
1. 标注肉眼可见的肢体动作（绞裙子、看天花板、手指敲桌）
2. 标注与"嘴上说的"矛盾的微表情（笑但眼眶红了）
3. 标注环境互动（望向窗外、摆弄饮料吸管）
4. 不解释动机——让读者/玩家自己感受

### 3.6 旁白块

```
【旁白】
叙述性文字。用于：
- 场景环境描写（给玩家建立空间感）
- 主角内心独白（非对话的心理活动）
- 时间流逝说明
【/旁白】
```

**示例**：
```
【旁白】
放课后的教室总是这样。夕阳把课桌的影子拉得很长，空气里飘着谁的值日生忘记擦的黑板粉笔灰味。真由趴在自己的桌子上，脸埋在手臂里，只看得到后脑勺的马尾。
【/旁白】
```

### 3.7 选择块

```
【选择】
  - 「选项文本」 → 目标场景ID
    [flag操作]
    [flag操作]
```

**规则**：
- 每个选项一行，`-` 开头
- `「选项文本」`是玩家看到的文字
- `→ 目标场景ID` 是跳转目标（场景文件 ID）
- `[flag操作]` 紧跟选项下方，缩进 4 空格

**Flag 操作语法**：
```
[flag名 = 值]          — 设置布尔/枚举
[flag名 +N]            — 整数增减
[flag名 -N]
```

**示例**：
```
【选择】
  - 「说吧，反正也没别的事」 → 场景_mayu_02_倾诉
    [好感度_真由 +2]
    [flag_倾听 = true]
  - 「今天不行，先回去吧」 → 场景_mayu_02_回避
    [flag_回避 += 1]
```

### 3.8 条件块

```
【条件：flag名 == 值】
  条件满足时显示的对话/内容
【否则】
  条件不满足时显示的内容
【/条件】
```

**使用场景**：同一场景内根据玩家之前的选择展示不同对话。

### 3.9 检定块（跑团 Replay VN 专用）

```
【检定：技能名 | 难度：NN | 结果】
  （检定演出注：视觉呈现说明）
```

**规则**：
- 用于跑团 Replay 视觉小说中，在关键技能/能力判定节点插入骰子检定演出
- 检定块打断正常对话流，以视觉特效（骰子滚动、数值跳动等）呈现判定过程
- 检定块之后紧跟判定结果带来的叙事后果（成功/失败的不同描写）

**字段说明**：

| 字段 | 必填 | 说明 |
|------|------|------|
| `技能名` | 是 | 检定技能/属性名称，如：观察、交涉、隐匿、战斗、魔力感知、学识等 |
| `难度` | 是 | 目标难度等级（DC），如：10（简单）/ 15（普通）/ 20（困难）/ 25+（极难） |
| `结果` | 否 | 成功 / 失败 / 大成功 / 大失败。写前未确定时可留空——正文根据跑团记录填充 |

**简化格式**（快速标注，不打断阅读流）：
```
【检定：技能名 | 难度：NN】
```
正文紧接着描写检定结果——适合检定与叙事融合的场景。

**完整格式**（检定作为独立演出节拍）：
```
【检定：技能名 | 难度：NN | 结果】
  （检定演出注：骰子视觉/音效/情绪。例：一颗二十面骰在黑暗中旋转，数字从模糊到清晰——）
```

**检定演出注**：描述这一幕在视觉上的呈现方式——骰子类型（D20/D100）、画面色调、音效风格、角色在检定时的表情。

**示例**：

```
【检定：观察 | 难度：15 | 成功】
  （D20 检定。画面切为黑底，一颗金色二十面骰在中央旋转。清脆的碰撞声。骰子停住——17。画面切回：姬雅眯起眼，在走廊转角处的阴影中捕捉到了微弱的魔力残光）

姬雅：「那里。有人来过——不超过十分钟。」 ⏱ 2.5s
  （指尖轻触下颌。目光锁定走廊尽头那幅肖像画下方的地板缝）
```

```
【检定：交涉 | 难度：20 | 失败】
  （D20 检定。深蓝底色，骰子旋转——停住：8。骰子化为碎片，画面切回酒吧昏暗的灯光。羽蝶脸上没有表情变化，但她的手指从酒杯上移开了）

羽蝶：「我说过了。不接受'朋友'。」 ⏱ 2.0s
  （语气比刚才更冷。把杯底最后一口烈酒饮尽）
```

```
【检定：隐匿 | 难度：12】
```
（后文描写克蕾尔压低身体，屏住呼吸——她融入了柱子投下的阴影中，巡逻魔偶从她身前不到三步处走过，没有停顿。）

### 3.10 CG块（剧情关键帧）

```
【CG：CG名称 | 触发时机】
  🎨 生图提示词：[完整的GPT中文生图提示词]
```

**规则**：
- CG 是剧情关键帧——用于名场面定格。在情感高潮、重要初遇、关键抉择、章节结尾等位置插入
- CG 打断正常对话流，以全屏插画呈现，通常配合 BGM 切换或静默
- 每章 3-8 个 CG（根据章节长度调整）

**生图提示词规则**（本分支强制要求）：
- **角色称谓**：出场角色仅写姓名（姬雅、克蕾尔、白 等），**不描述样貌**——样貌描述从角色档案中获取
- **必须描述**：角色的动作、表情、服装类型（校服/常服/礼装/睡衣/轻甲/战斗装束 等）
- **必须包含**：场景环境、光影氛围、构图/视角
- **风格统一**：日式动画/游戏CG风格，16:9横版

**示例**：

```
【CG：真由在夕阳教室 | 对话高潮·告白后】
  🎨 生图提示词：日式动画CG风格，16:9横版。放学后的教室，黄昏橙光从窗外洒入。真由站在窗前，背对镜头，身穿校服（深蓝色西装外套+百褶裙）。她微微低着头，右手抬起——像是在擦眼角。窗外远处是操场和暮色天空。课桌上散落着书包和文具。暖橙色调，略带感伤的氛围。

【bgm 静默 3秒】

真由：「第三次。同一个男生。」
  （声音突然变小，像被抽掉了什么）
```

## 四、完整场景示例（标准 AVG）

```
【场景：放课后的教室·黄昏】
【BGM：懒散的日常】
【bg：教室_黄昏】

【旁白】
社团活动结束后，教室里只剩两个人。真由趴在自己的桌子上，从刚才起已经沉默了五分钟。
【/旁白】

【立绘：主角 无语 左侧】
【立绘：真由 默认 右侧】

主角 [无语]：「差不多该说了吧。你把我留下来，不是让我看你睡觉的吧。」 ⏱ 2.9s
  （靠在椅背上。语气不重，但话本身很直）

【立绘：真由 苦笑】

真由 [苦笑]：「真失礼啊。我可是在做心理准备。」 ⏱ 2.2s
  （抬起头，捋了捋刘海。努力做出轻松的样子但失败了）

主角 [无语]：「什么心理准备。」 ⏱ 1.3s
  （已经隐约猜到了）

【立绘：真由 逞强】

真由 [逞强]：「跟你说，我又被甩了。」 ⏱ 1.7s
  （笑着说。笑得很用力）

【bgm 静默 2秒】

主角 [无语]：「……」 ⏱ —

真由 [逞强]：「第三次。同一个男生。」 ⏱ 1.8s
  （声音突然变小，像被抽掉了什么）

真由 [眼眶泛红]：「我是不是……真的不行啊。」 ⏱ 2.5s

【bgm 静默】

主角 [无奈]：「你问错人了。我又不是恋爱大师。」 ⏱ 2.3s
  （声音比平时轻。不知道该看哪里，最后选择了天花板）

真由 [眼眶泛红]：「那你至少听我说完啊。」 ⏱ 1.8s
  （用手背揉了揉眼睛）

真由 [苦笑]：「这可是你唯一的功能。」 ⏱ 1.9s

【立绘：主角 无奈】

【立绘：真由 苦笑】

【选择】
  - 「说吧，反正也没别的事」 → 场景_mayu_02_倾诉
    [好感度_真由 +2]
    [flag_倾听 = true]
  - 「说完了请你喝东西」 → 场景_mayu_02_茶饮

    [flag_请客 = true]
  - 「今天先送你回去吧」 → 场景_mayu_02_回避
    [flag_回避 += 1]
```

## 七、配音时长标注规范

> **定位**：因为每句话需要做配音（Voice-Over），剧本中的每一行对白和每一段静默/停顿都需要预估时长，确保配音制作时可以准确排期。

### 7.1 对话行时长标注

每条对话行末尾追加 `⏱ N.Ns`，N.N 为该句配音预估秒数。

**格式**：
```
角色名：「对话内容。」 ⏱ N.Ns
  （演出注）
```

**无配音行**：
```
角色名：「……」 ⏱ —
角色名：「……嗯。」 ⏱ 1.0s  （仅「嗯」计入）
```

### 7.2 估算公式

| 参数 | 值 | 说明 |
|------|-----|------|
| **基准语速** | 3.5 字/秒 | 中文配音自然语速，涵盖正常换气停顿 |
| **字数计算** | 仅计中文字符 | 去除标点符号后的纯文字数。引号「」不计入 |
| **句末标点附加** | 。！？+0.3s / ……+0.5s / ，、+0.15s | 句末标点夹带自然停顿 |
| **最短时长** | 0.5s | 任何有实质对话的行不得短于此值 |
| **单字/短词** | 1.0s 起 | "嗯""好""不"等单字回应至少 1.0s |

**计算公式**：
```
预估秒数 = max(0.5, 纯文字数 / 3.5 + 标点附加秒数)
```

**估算示例**：

| 对话内容 | 纯字数 | 计算 | 标注 |
|----------|--------|------|------|
| 「我没事啦。」 | 4 | 4/3.5 + 0.3 = 1.44 | ⏱ 1.4s |
| 「真失礼啊。我可是在做心理准备。」 | 14 | 14/3.5 + 0.3 = 4.3 | ⏱ 4.3s |
| 「嗯。」 | 1 | max(0.5, 1/3.5) → 取最短 0.5s | ⏱ 0.5s |
| 「我是不是……真的不行啊。」 | 9 | 9/3.5 + 0.5(……) + 0.3 = 3.37 | ⏱ 3.4s |
| 「（沉默）」 | — | 无配音 | ⏱ — |

### 7.3 效果指令时长

`【停顿 N秒】` 和 `【bgm 静默 N秒】` 中的 N 秒**直接计入**场景总时长。

```
【bgm 静默 2秒】  → 计入 2.0s
【停顿 1秒】      → 计入 1.0s
```

### 7.4 旁白与演出注

- **旁白块**：默认不计配音时长（旁白在 AVG 中通常为文字阅读，非配音）。如需配音，在旁白块后标注 `⏱ N.Ns`（同对话行公式）
- **演出注**：不计配音时长（演出注是给导演/画师看的，不配音）

### 7.5 场景头时长汇总

每个场景文件在剧本正文结束后，追加时长汇总块：

```
---
## 配音时长汇总
- **对白句数**：XX 句
- **对白时长**：XX.Xs（纯对话）
- **停顿/静默**：XX.Xs
- **场景总时长**：XX.Xs（约 X 分 Y 秒）
```

### 7.6 章节头时长汇总

如果场景文件对应一个完整章节（小说改编场景），在**场景头**下方追加本章时长预估：

```
【场景：放课后的教室·黄昏】
> 本章配音总时长：约 X 分 Y 秒（XX 句对白 + X 处停顿）
【BGM：懒散的日常】
```

### 7.7 ⏱ 标记视觉规范

- 使用 Unicode 字符 `⏱` (U+23F1, STOPWATCH)
- 紧跟对话行末尾的 `」` 之后，空格分隔
- 秒数保留 1 位小数
- 无配音行标注 `⏱ —`（em dash）
- 该标记同时作为机读标识——脚本解析工具可通过 `⏱` 字符提取配音时长数据

## 六、AI 生成时的格式约束

Agent 墨在生成 AVG 剧本时必须遵守：

1. **每句话配上演出注**：对话行的演出注不可省略（这是区别于散文小说的关键——让"视觉呈现"落地）
2. **每句话标注配音时长**：所有对话行末尾追加 `⏱ N.Ns`，按 3.5 字/秒估算（详见 §七）。静默/停顿指令同样计入总时长
3. **立绘变化不可遗漏**：角色情绪变了就要更新立绘指令
4. **BGM 静默作为武器**：关键情绪转折点用【bgm 静默】制造留白
5. **演出注写"看得见的"**：不写抽象心理，写具体的动作和表情
6. **场景头标注总时长**：每个场景文件开头汇总本章总配音时长（格式见 §七）
7. **项目类型决定块类型可用性**：见 §八——标准 AVG 使用选择块+flag；跑团 Replay VN 使用检定块，不使用选择/flag/跳转

## 八、项目类型与块类型适用性

本格式同时支持两种项目类型。Agent 需根据项目配置（`project-config.md`）判断当前项目类型，选择正确的块类型组合。

### 8.1 标准 AVG（分支型）

玩家通过选择肢影响剧情走向。使用全部块类型：

| 使用 | 块类型 |
|------|--------|
| ✅ 使用 | 场景头、环境指令、立绘指令、效果指令、对话行、演出注、旁白块、**选择块**、**条件块**、**跳转指令**、**Flag 标记** |
| ❌ 不使用 | 检定块 |

### 8.2 跑团 Replay VN（线性型）

基于 TRPG 跑团记录改编的线性视觉小说。**不存在玩家选择与剧情分歧**——剧情按跑团记录线性推进。

| 使用 | 块类型 |
|------|--------|
| ✅ 使用 | 场景头、环境指令、立绘指令、效果指令、对话行、演出注、旁白块、**检定块** |
| ❌ 不使用 | 选择块、条件块、跳转指令、Flag 标记 |

**跑团 Replay VN 额外规则**：

1. **线性叙事**：只有一个剧情入口和一个出口，场景之间按固定顺序衔接。场景文件命名使用线性编号而非路线标识（`场景_01_xxx.md`）
2. **检定密度**：每 5-10 句对白之间应至少有 1 次检定——对应跑团中大约每 5-10 分钟一次的关键判定。过于密集会打断叙事流，过于稀疏会失去 TRPG 感
3. **检定 ≠ 数据展示**：检定块是叙事工具而非数据展示——检定演出注必须包含画面/音效/情绪描写。不要让骰子检定变成"数据弹窗"——它应该是一个微型的、有紧张感的演出节拍
4. **成功/失败都有意义**：检定成功≠一帆风顺，检定失败≠叙事卡住。检定结果之后的描写应展示角色如何应对——成功时的自信、失败时的应变。TRPG 的魅力在于"即使失败故事也在推进"
5. **检定与配音**：检定块本身不计配音时长（它是视觉演出）。检定演出注中的括号内文字不计时长
6. **GM 旁白风格**：旁白块在跑团 Replay VN 中可承担"虚拟 GM"的叙事角色——使用第一人称口吻描述场景/规则/检定结果。非必需，但可作为风格选择
