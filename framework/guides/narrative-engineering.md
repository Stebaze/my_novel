# 叙事工程方法论

> **定位**：本文档定义了从宏观到微观的五级叙事组织方法。Agent 在写作和审阅时必须能在这个框架中定位"当前在做什么"。

---

## 多视角设计原则

本文档的方法论综合了四个视角：

| 视角 | 关心什么 | 对方法论的贡献 |
|------|----------|---------------|
| **作者** | 怎么可持续地产出 200 章不崩 | 五级大纲的递进设计、事件因果链的自动校验 |
| **编辑** | 节奏对不对、场景有没有浪费、角色弧是否完整 | 场景经济性检查、角色出场率监控、张力曲线校验 |
| **老读者** | 伏笔有没有收、角色有没有崩、套路有没有新意 | 伏笔揭示的回响验证、OOC 边界动态更新、套路→反套路转化 |
| **新读者** | 能不能随时入坑、信息会不会过载、角色记不记得住 | 信息释放梯度控制、角色首次出场记忆点强制检查 |

---

## 一、五级大纲体系

### L1：全书级 —— 只用一段话说清楚

这是整个系列的"宪法"。一切后续规划都从此推导。

**必填字段**：
- **核心主题**：（一个词或一句话，如"少数与多数""败者也能成为主角"）
- **终点画面**：最后一卷的最后一章，读者合上书时看到什么画面、感受到什么情绪
- **主角终点状态**：主角从什么状态出发，到达什么状态（不仅是身份变化，更是内心状态）
- **不可违背的规则**：这个世界/这个故事里绝对不可以发生的事（2-3 条）——这是防止连载膨胀的刹车

**Agent 使用规则**：
- L1 一旦确定，后续所有审阅都要追问"这个场景/章节/卷是在接近终点还是偏离终点？"
- L1 可以被修改——但修改必须记录到 `notes.md` 决策表，并同步更新所有下级大纲

### L2：分卷级 —— 每卷是一个独立的"情感产品"

每卷必须满足：**新读者从这一卷开始读，也能享受**。同时每卷在全书主线中承担不可替代的功能。

**每卷必填字段**：

| 字段 | 说明 | Agent 检查 |
|------|------|-----------|
| 卷主题 | 这一卷的核心情感/议题 | 本卷所有章节是否扣题 |
| 情感目标 | 读者读完这卷应该感受到什么 | 最后一章的结尾情感是否匹配 |
| 剧情目标 | 在主线中不可替代的推进 | 删掉这卷主线会断裂吗 |
| 大高潮 | 本卷最大的爆点 | 高潮前的铺垫是否充分 |
| 卷末状态 | 主角/女主/关键配角在本卷结束时的位置 | 和下一卷起点是否衔接 |
| 本卷新角色 | 本卷引入的角色及其功能 | 每个新角色是否至少有一个记忆点场景 |

**卷间衔接规则**：
- 每卷结尾必须同时做到"本卷故事闭合"和"全书悬念延续"
- 闭合不够 → 读者觉得烂尾；悬念不够 → 读者不会追下一卷
- 闭合:悬念 ≈ 7:3 （本卷主线要收，全书主线留一个钩子）

### L3：篇章级 —— 5-8 章的"迷你电影"

一篇 = 一个有头有尾的完整故事单元，通常 5-8 章（1.5-3 万字）。

**篇章结构模板**：

| 章位 | 功能 | 必须交付 |
|------|------|----------|
| 第 1 章（开端） | 建立篇章的核心问题/冲突 | 读者知道"这篇要解决什么" |
| 第 2-3 章（展开） | 问题升级、初次尝试失败、信息收集 | 至少一次反转或新信息揭示 |
| 第 4-6 章（高潮前） | 最紧张/最接近答案的阶段 | 情感和情节的双重张力高峰 |
| 第 7 章（高潮） | 核心冲突的爆发和解决 | 不可逆的变化发生 |
| 第 8 章（余波/钩子） | 处理高潮后果、埋下一篇章的钩子 | 角色对变化的反应 + 新问题的影子 |

**篇章间衔接规则**：
- 每篇结束后角色状态必须与上一篇不同（哪怕是微小的变化）
- 如果某篇章结束后所有角色状态完全不变 → 这篇可以删

### L4：章节级 —— 一个完整的呼吸循环

见 `framework/templates/chapters/_chapter-template.md`。关键约束：
- 一章 = 一个情感变化（进入情感 ≠ 退出情感）
- 一章 = 至少一个小爆点 + 一个有效断章
- 一章内场景数 2-5 个，单场景不超过 1500 字

### L5：场景级 —— 叙事的原子单位

每个场景必须同时满足：

| 维度 | 要求 | 检验问题 |
|------|------|----------|
| **功能** | 至少承担一种功能（推进剧情/推进关系/揭示信息/建立氛围） | 删掉这个场景，前后文能直接衔接吗？能 = 该删 |
| **情感变化** | 进入情感 ≠ 退出情感 | 角色在这个场景中经历了什么情感变化？ |
| **信息增量** | 读者/主角知道了之前不知道的事 | 这个场景告诉了我们什么新信息？ |
| **角色在场** | 每个在场角色都有存在的理由 | 这个角色在这个场景里除了"刚好在"还有别的功能吗？ |

**场景组合规则**：
- 同一章内相邻场景的情感基调不能完全相同
- 喜剧场景和严肃场景应交替（但不能是机械的 1:1）
- 单人场景（男主独白/独处）在每章中至少出现 1 次——这是读者"呼吸"的空间

## 二、事件编排法

### 2.1 因果链原则

**核心规则**：每一个事件必须是前一个事件的**直接后果**，或者有角色**主动选择**面对它。

```
❌ 事件 A → 然后事件 B 发生了（巧合/被动）
✅ 事件 A → 因为 A，主角选择做 X → 导致了事件 B（因果/主动）
```

**Agent 自检**：画出本章/本篇的事件链。每个箭头问"是因为（because）还是然后（and then）？"如果连续 2 个"and then" → 标记为被动叙事，需要修改。

### 2.2 张力波浪模型

故事张力不是一条线，而是一组**振幅递增的波浪**：

```
张力
 ^
 |        ╭─╮          ╭───╮
 |   ╭─╮  │ │  ╭─╮    │   │     ╭────╮
 |  ╱   ╲╱ ╲╱  ╲╱  ╲  │   ╲   ╱    ╲
 | ╱                ╲╱    ╲╱        ╲___
 |╱                                       ╲___
 +────────────────────────────────────────────→ 时间
   篇章1    篇章2    篇章3      篇章4       篇章5
```

**规则**：
- 每个篇章的波峰应**略高于**上一篇章的波峰（整体趋势向上）
- 波谷不能回到零——读者在任何时候都应该有"想知道接下来怎样"的最低张力
- 篇章 1 的波谷 > 篇章 1 的起点（读者读完后比开始前更投入）
- **大高潮（卷高潮）前的篇章**：波谷可以不降——允许连续紧张 5-8 章

### 2.3 多线并行编排

如果有多条情节线（如：主线悬疑 + 恋爱线 + 配角支线），编排规则：

**线间切换原则**：
- 在一条线的**波谷**切换到另一条线——不要在波峰切走（读者会摔书）
- 每条线在每 3 章内至少出现一次——消失超过 3 章读者会忘记那条线在干什么
- 切换时用一个"钩子"——上一线的波谷留下一个问题，让读者愿意暂时跟另一条线走

**线的优先级**：
- 主线（A 线）：每章都推进，至少每章有一场景
- 恋爱线（B 线）：每 1-2 章有一次互动/推进
- 配角线（C 线）：每 2-3 章推进一次，不喧宾夺主

### 2.4 事件密度曲线

不同阶段的事件密度要求不同：

| 阶段 | 章节范围 | 事件密度 | 说明 |
|------|----------|----------|------|
| 开局 | Vol1 Ch1-10 | 中等偏高 | 快速建立世界观和角色，但不能信息过载 |
| 展开 | Vol1 Ch11-30 | 中等 | 节奏稍缓，深入角色关系和世界观 |
| 高潮前 | Vol1 Ch31-40 | 逐渐升高 | 紧张感累积，事件频率增加 |
| 高潮 | Vol1 Ch41-45 | 最高 | 连续高密度，可以打破"连续紧张不超过 3 章"规则 |
| 余波/过渡 | Vol1 Ch46-50 | 低 | 处理后果，为下一卷铺垫 |

## 三、人物活动编排

### 3.1 角色出场节奏

**核心原则**：没有任何主要角色应该在连续 3 章以上完全不出场。如果某个角色"暂时没戏"——创造一个理由让他/她出现。

**出场率参考**（以每 10 章为单位）：

| 角色类型 | 最低出场章数 | 说明 |
|----------|------------|------|
| 男主 | 10/10 | 每章都在（POV 角色） |
| 女主 | 8-10/10 | 几乎每章都在或有提及 |
| 主要配角 | 4-6/10 | 每 2-3 章出现一次 |
| 次要配角 | 1-3/10 | 出现时要有功能 |

**出场率监控**：Agent 在每 10 章后自动统计角色出场章数。发现某个角色出场率低于最低线 → 标记为"角色失踪风险"。

### 3.2 场景角色组合矩阵

不同角色组合产生不同的化学类型。规划场景时参考此矩阵：

| 组合 | 化学类型 | 适用场景 | 使用频率建议 |
|------|----------|----------|------------|
| 男主 × 女主（双人） | 关系推进/斗嘴/糖分/试探 | 日常、关键情感转折 | 每 1-2 章必有一次 |
| 男主 × 女配（双人） | 暧昧/误会/情报交换/展示男主另一面 | 修罗场铺垫、支线推进 | 每 3-4 章 |
| 男主 × 男配（双人） | 友情/竞争/情报/"男人间的对话" | 男主获取情报/建议 | 每 5-8 章 |
| 女主 × 女配（双人） | 修罗场/同盟/"女人间的对话" | 展现女主另一面 | 每 5-8 章 |
| 全主角团（3+人） | 群像互动、信息汇总、气氛调节 | 篇章开端/结尾、重大事件 | 每 5-8 章 |
| 男主独处 | 内心独白/反思/读者呼吸空间 | 每章至少一次 | 每章 |
| 配角线（无男主） | 支线推进/伏笔埋设 | 偶尔 | 每 5-10 章 1 次 |

### 3.3 关系里程碑的编排节奏

**核心原则**：关系不是匀速推进的。它是"跳跃—巩固—再跳跃"的节奏。

**关系推进的黄金比例**：
- 推进（关系发生质变）：~15% 的章节
- 巩固（在现有关系阶段内互动，深化默契）：~60% 的章节
- 倒退/误会/矛盾（制造张力）：~25% 的章节

**里程碑规划表**（在 `outline.md` 中按卷填写）：

| 里程碑 | 类型 | 计划章节 | 触发事件 | 关系从 X → Y |
|--------|------|----------|----------|-------------|
| M1 | 初次注意 | | | 路人 → 有点在意 |
| M2 | 被迫靠近 | | | 有点在意 → 被迫共处 |
| M3 | 第一次心动 | | | 被迫共处 → 意识到喜欢 |
| ... | | | | |

**Agent 检查**：每完成一个关系里程碑后，检查：
1. 里程碑之前的铺垫是否充分（至少 2 次"差点发生但没发生"）
2. 里程碑之后的巩固章数是否足够（至少 2-3 章消化期）
3. 里程碑本身是否"不可逆"——发生之后角色关系确实变了，不会下一章恢复原状

### 3.4 角色信息释放的梯度控制

> 从"新读者"视角出发：读者对每个角色的了解应该是渐进的。

**角色信息释放的三阶段**：

| 阶段 | 读者知道的 | 写作原则 | 章节范围 |
|------|-----------|----------|----------|
| 初识 | 外貌特征 + 一个性格标签 + 一个记忆点场景 | 只给"认知钩子"，不要展开背景 | 出场后 1-3 章 |
| 深入 | 表层性格下的矛盾 + 一个小秘密/弱点 + 和男主的关系质感 | 通过事件自然揭示，不要独白式交代 | 出场后 5-20 章 |
| 完整 | 深层动机 + 完整背景 + 角色弧方向 | 作为篇章高潮或卷高潮的一部分揭示 | 每卷揭示一个层次 |

**新读者友好检查**：
- 任意打开一章——一个不认识角色的人能从这一章的角色互动中大致推断出他们的关系吗？
- 如果不能 → 这一章缺少"关系展示"场景，需要加入能体现关系现状的互动

### 3.5 对话目的性设计

> 参考：梁元·月食 对话设计原则。每句对话必须有可追溯的欲求来源——不为"让读者知道信息"而说话。

#### 对话六目的

| 目的 | 定义 | 示例 |
|------|------|------|
| **抒情** | 表达情感、寻求共鸣 | "你知不知道我等了多久" |
| **社交** | 维护/调整关系距离 | "今天天气不错"（拉近）/ "随你便"（推远） |
| **求知** | 获取信息、理解某事 | "那一刀，你看到了什么" |
| **表达** | 陈述观点、立场、判断 | "我觉得这不对" |
| **工作** | 完成任务、协调行动 | "你去左边，我守右边" |
| **娱乐** | 消遣、调侃、打发时间 | 斗嘴、开玩笑 |

每句台词可追溯到至少一个目的（可混合，如求知+社交）。无欲求则沉默——角色不为了"让读者知道信息"而说话。不同角色的目的偏好不同：有人惯用娱乐来掩饰抒情，有人只用工作目的与人交流。

#### 对话设计检查

**写作时（mo-writer 简报 §1）**：为关键对话标注每个角色的对话欲求。连续 5 句以上无目的标注的对话 → 警示：可能是在"为解释而说"。

**评审时（ping-critic Step 3a）**：检查：
- 大段对话中每个角色的欲求链是否清晰？
- 有无"为读者解释设定"的科普式台词？（最常见违规）
- 角色的对话风格是否与其语言纹理一致？

### 3.6 语言纹理差异化

> 不同角色的词汇量、句式复杂度、口癖必须与其背景一致。不是所有角色都"会说话"——有些人不善表达，有些人话太多，有些人只会用一种方式说话。

**差异化维度**：

| 维度 | 决定因素 | 示例 |
|------|----------|------|
| **词汇量** | 生理年龄 + 教育背景 | 少年角色不用成语，学者角色术语自然流出 |
| **句式复杂度** | 性格 + 谈吐素养 | 急性子用碎片句，沉稳者用完整复句 |
| **口癖/高频词** | 个人习惯 + 地域/文化背景 | 口头禅不是装饰——是角色身份的听觉标记 |
| **对话占比** | 性格（外向/内向）+ 当前情绪 | 沉默的角色在关键时刻开口才有重量 |
| **目的偏好** | 性格 + 人际关系策略 | 有人只用娱乐目的开始对话，有人永远从求知切入 |

**Agent 使用**：
- `voice-sculptor`：将语言纹理维度纳入角色声音特征提取
- `mo-writer`：在简报 §2 的出场角色设计中标注语言纹理要点
- `ping-critic`：评审时检查不同角色的台词是否可以互换（可以互换 = 纹理差异化不足）

## 四、多视角一致性检查

### 4.1 作者视角：可持续性检查

每写完一卷，Agent 自问：
- [ ] 下一卷的篇章结构已经规划了吗？
- [ ] 有没有角色已经超过 10 章没有实质进展？
- [ ] 有没有伏笔埋了超过 2 卷还没碰过？
- [ ] 主线事件是否在按 L2 分卷目标推进？
- [ ] 有没有偏离 L1 设定的终点方向？

### 4.2 编辑视角：经济性检查

每写完一章，Agent 检查：
- [ ] 每个场景都有不可删除的理由？
- [ ] 每个出场角色都有场景功能？
- [ ] 本章事件是前一章事件的因果延续？（检查"and then"数量）
- [ ] 情感节拍是否如计划交付？

### 4.3 老读者视角：深度一致性检查

每写完一卷：
- [ ] 伏笔揭示时是否有回响感？（对比原始埋入语境）
- [ ] 角色有没有出现"第一卷做了但第五卷忘了"的能力/经历？
- [ ] 世界观规则是否有被悄悄违反的地方？
- [ ] 角色的成长是渐进的还是跳跃的？
- [ ] 有没有哪个老套路在本卷中被赋予了新意？

### 4.4 新读者视角：可进入性检查

每写完一卷：
- [ ] 从本卷第一章开始读，能理解角色关系吗？
- [ ] 前情信息是通过对话/场景自然带出的，还是通过旁白交代的？
- [ ] 本卷有没有至少 1-2 个场景是"即使没读过前文也能享受"的？
- [ ] 新角色出场时有没有记忆点？

## 六、网文节奏引擎（来自《没钱修什么仙》）

> 本章总结了从《没钱修什么仙》中提取的事件组织、章节推进、剧情升级技法。该书在"用考试/竞赛驱动长线叙事"方面达到了极高的工业化水准。

### 6.1 考试/竞赛驱动的章节组织法

**核心发现**：在这部小说中，"考试"不是偶尔出现的情节事件——它是整个叙事机器的核心引擎。从高中月考到联赛到高考到筑基考到大学期末考，考试串联了 500+ 章。

**考试作为叙事引擎的五层功能**：

| 层次 | 功能 | 实例 |
|------|------|------|
| **推进剧情** | 考试结果改变角色处境（排名=资源=生存） | 月考排名决定分班；竞赛名次决定奖金 |
| **展示成长** | 考前 vs 考后的数据对比，可视化成长 | 肉体强度 0.82→9.94，排名 19→4 |
| **引入对手** | 每次竞赛引入新的竞争对手（不同学校/层级） | 何大有→宋海龙→乐沐岚→夜凌霄 |
| **升级舞台** | 每次竞赛的舞台更大（校内→市内→全昆墟） | 嵩阳月考→市法赛→筑基考→十大联赛 |
| **情感释放** | 排名揭晓 = 情绪高潮 | "第一名，张羽，综合分2553" |

**章节块结构**（一个完整竞赛弧 ≈ 10-20 章）：

```
[赛前准备 3-5章]          [竞赛过程 4-8章]          [赛后后果 3-5章]
                                                                
压力积累 + 资源获取        淘汰赛/擂台战             排名揭晓 + 奖励结算  
新功法/新药/新装备         对手实力展示              新目标设定
对手摸底 + 战术讨论        主角极限发挥              更高层级邀请/威胁
赛前意外/劣势              关键逆转时刻              关系变化+ 下一竞赛预告
```

**要点**：每个竞赛弧 = 一个完整的"情感产品"。读完 10-20 章，读者会觉得"这一轮结束了，但我还想看下一轮"。

### 6.2 日常→竞赛 呼吸节奏

**核心发现**：这部小说不会连续高潮。它在"高压日常"和"竞赛爆发"之间建立了精确的交替节奏。

**节奏模型**：

```
高压日常（2-4章）  →  小爆点/月考（1-2章）  →  高压日常（2-4章）  →  大爆点/市赛（5-10章）
                                                                
修炼/打工/还债          排名揭晓                新功法/升级           多轮淘汰+决赛
量化微升级              社交关系变化            新对手新目标           排名+奖金+新弧开启
```

**日常章的功能**（看似"平淡"但必须存在）：
- 展示主角的"高压状态"——24/7修炼的代价（没有日常章的苦，竞赛章的爽就没有对比）
- 量化微升级——经验值从 12/80→25/80，读者享受积累过程
- 关系互动——白真真转500块、周天翊八卦、赵天行被拉着打工
- 经济压力——账单、催收、打工——提醒读者世界的残酷
- 信息收集——对手情报、功法信息、竞赛规则

**"呼吸"规则**：
- 连续高潮不超过 5 章（竞赛决赛阶段）
- 连续日常不超过 4 章（需要小爆点破局）
- 日常章中必须有至少一个"微爽点"（数字升级/关系互动/幽默场景）

### 6.3 排名驱动的剧情升级机制

**核心发现**：排名是这部小说中最高效的"剧情升级触发器"——主角每上升一个排名段位，自然解锁下一个层次的世界和冲突。

**排名段的剧情功能**：

| 排名段 | 解锁内容 | 典型冲突 |
|--------|----------|----------|
| 年级 19-10 | 校内竞争，示范班边缘 | 被老师放弃/同学看不起 |
| 年级 10-5 | 校内顶级圈子，参加市赛资格 | 校内学霸竞争，资源争夺 |
| 年级 5-1 | 全市竞赛种子，跨校竞争 | 三大名校精英，阶级碰撞 |
| 全市前 10 | 大学招生关注，筑基资格 | 有钱人买名次，制度不公 |
| 全市前 3 | 十大顶尖大学视线，筑基证争夺 | 和"人类巅峰"级对手决战 |
| 大学筑基班第 1 | 导师争夺，王牌专业入场券 | 大学派系斗争，更高层博弈 |

**关键规则**：每次排名跃升，(1) 对手升级——不是主角变强了敌人就弱了，而是更强的人开始把你当对手；(2) 舞台升级——校内→市内→全层→跨层→全昆墟；(3) 问题升级——从"还不起贷款"到"被大宗门盯上"；(4) 但核心矛盾不变——穷 vs 富、被迫 vs 自由。

### 6.4 三维压力编织法

**核心发现**：主角同时承受三种压力——金钱、时间、成绩——它们在每一章中交替收紧和放松，形成"压力的复调音乐"。

**三维压力的动态调配**：

```
金钱压力：欠债70万 → 打工赚钱 → 奖金到账(释放) → 新功法的花费(新压力) → 贷款(更大压力)
时间压力：仪式24/7逼迫 → 极限修炼 → 竞赛结束(暂时喘息) → 新目标(压力恢复)
成绩压力：排名下滑 → 突破升级 → 排名上升(释放) → 更高排名目标(新压力)
```

**编织规则**：
- 任何一个维度都不能"彻底解决"——只能"暂时缓解"
- 三个维度的压力峰值应该错开——金钱压力最大时成绩压力可以稍低，反之亦然
- 当一个维度压力骤增时（欠款暴雷），另一个维度可以给甜头（新功法学成）——读者不会感到绝望
- 三个维度同时收紧 = 读者感到的压迫感 ≥ 主角的压迫感（会弃书）→ 只在卷高潮前使用

### 6.5 金字塔式世界展开法

**核心发现**：世界不是一次性介绍完的，而是随着主角的排名上升，一层一层解锁的。

**展开节奏**：

| 阶段 | 揭示的世界层级 | 信息传递方式 |
|------|---------------|-------------|
| 开局（Ch1-10） | 嵩阳高中、昆墟一层基本规则 | 主角的面试/上学/打工日常 |
| 展开（Ch11-50） | 三大名校、竞赛体系、法宝/法骸 | 月考和市赛的对手和场景 |
| 深化（Ch51-150） | 筑基考试体系、各层高中差距 | 筑基考场上遇到全层精英 |
| 升级（Ch151+） | 大学体系、十大、昆墟上层 | 考入大学后直接进入新世界 |

**信息释放的"80/20 法则"**：
- 80% 的世界信息通过角色的行为和对话自然带出（不是旁白交代）
- 20% 通过主角的内心吐槽或回忆补充
- 关键规则（如"灵界"的本质、"法骸"的原理）在读者已经通过场景感受多次后，才用课堂/对话做正式解释

### 6.6 章尾钩子类型频率分布

基于对本书的分析，一个成功的网文章节结尾应该使用多样化的钩子类型。推荐的频率分布：

| 钩子类型 | 使用频率 | 说明 |
|----------|----------|------|
| 答案前一刻 | ~20% | 问题即将被回答时断开 |
| 名次/数字揭晓 | ~15% | 排名、分数、金额揭晓 |
| 能力升级宣告 | ~15% | 新等级/新功法即将展示 |
| 关系临界点 | ~15% | 角色关系即将质变 |
| 敌人/对手出现 | ~10% | 新威胁的引入 |
| 金钱压力 | ~10% | 财务数字变化（赚钱/欠债/花光） |
| 情绪反转 | ~10% | 先营造一种情绪再打破 |
| 平静中的不祥 | ~5% | 日常中的异常暗示 |

**关键规则**：不要连续 3 章使用同一种钩子类型。名次揭晓和金钱压力是本书特有的高效钩子——它们的优势在于"数字变化 = 客观事实 = 不可逆"。

## 八、衔接设计方法论

> 衔接（Transition/Coupling）不是写完场景后补的"过渡段"——它是场景 A 的最后一句与场景 B 的前两句之间的**桥接单元**。好的衔接让读者"无意识滑入新场景"，差的衔接让读者"被推出后需要重新定位"。本章从三层模型的视角系统化衔接设计。

### 8.1 三层衔接模型

衔接发生在三个层级，每个层级有不同的设计要求和失败后果：

| 层级 | 边界 | 设计窗口 | 失败后果 | 技法来源 |
|------|------|---------|---------|---------|
| **场景间** | 章内 `---` 切换点 | 场景 A 最后 1-2 句 + 场景 B 前 1-2 句 | 读者在章内"卡住"，需要回头重读 | `technique-library.md` §7.1 (S1-S14) |
| **章间** | 第 N 章结尾 → 第 N+1 章开头 | 上章最后 2 段 + 下章前 2 段 | 读者放下书后不再拿起；或拿起后需要重新定位 | `technique-library.md` §7.2 (C1-C8) |
| **卷间** | 第 N 卷结尾 → 第 N+1 卷开头 | 上卷最后 1 章 + 新卷前 1 章 | 读者觉得"故事讲完了，没必要追下一卷"；或新读者无法从新卷进入 | `technique-library.md` §7.3 (V1-V6) |

**层级关系**：
- 场景间衔接是**微观节奏**——影响读者的"秒级"阅读体验
- 章间衔接是**中观节奏**——影响读者的"章级"追读决策
- 卷间衔接是**宏观节奏**——影响读者的"卷级"是否继续投资

### 8.2 衔接技法选择矩阵

#### 场景间衔接选择

| 场景 A→B 的关系 | 推荐衔接技法 | 次选 | 说明 |
|-----------------|-------------|------|------|
| 同天，不同地点 | S2 地点锚定, S14 媒介物转场 | S4 门之通道 | 空间移动是可追踪的 |
| 不同天，同地点 | S1 时间锚定 | S11 软性开场 | 时间词在第一句 |
| 不同天，不同地点 | S1 + S2 组合 | S10→S11 | 先时间后地点 |
| 同时间，不同地点（多线） | S3 同时反打 | — | "与此同时"是触发词 |
| 战斗→日常 | S8 日常沉降, S5 对比转场 | S11 软性开场 | 需要"降落"感 |
| 严肃→喜剧 | S7 荒诞破局, S5 对比转场 | — | 破局必须角色驱动 |
| 日常→战斗 | S12 外部闯入 | S4 门之通道 | 闯入必须有后果 |
| 需要诗意/情感连贯 | S6 主题呼应, S10 无声收尾 | C4 情感延续 | 共享意象要轻 |
| 史诗感→个体感 | S13 宏微反差 | S8 日常沉降 | 反差两端都要充分 |
| 视角切换（同时间线） | S9 梦醒锚点, S3 同时反打 | C5 视角接力 | 新 POV 角色名首句出现 |

#### 章间衔接选择

| 章节关系 | 推荐 | 不推荐 | 说明 |
|---------|------|--------|------|
| 场景延续（同一场景跨章） | C1 浸入式跨章 | C3 锚点重启 | 在对话/动作中间断开 |
| 有悬念需要回收 | C2 钩子回收式 | C6 话说体 | 网文默认选项 |
| 有明确的时/空跳跃 | C3 锚点重启式 | C1 浸入式 | 新章首句完成定向 |
| 情感/氛围需要连续 | C4 情感延续式 | C3 锚点重启 | 情感不中断 |
| 多 POV 轮转 | C5 视角接力式 | C1 浸入式 | 新视角角色名首句出现 |
| 更新间隔长，需要提醒读者 | C6 话说体 | C1 浸入式 | 一句话回溯 |
| 强悬念 + 对话驱动 | C7 对话钩子开章 | C3 锚点重启 | 首句即对话 |

#### 卷间衔接选择

| 卷间关系 | 推荐 | 关键动作 |
|---------|------|---------|
| 力量/舞台有提升 | V2 地位重置+升级 | 新卷日常必须含有"升级痕迹" |
| 时间跨越大 | V3 时间跳跃 | 跳跃摘要 ≤3 段 |
| 地点/世界转移 | V4 舞台转移 | 新舞台在旧卷必须有伏笔或提及 |
| 需要重新建立日常 | V5 序章重启 | 序必须是"有趣的日常" |
| 策略型主角、多势力 | V6 预案→意外→再预案 | 上卷结尾明确写出主角的预期 |

### 8.3 衔接与心流的关系

每个衔接类型对心流五维有不同影响。设计衔接时需考虑目标心流效果：

| 衔接方式 | 挑战-技能平衡 | 目标清晰 | 即时反馈 | 专注保护 | 行动-意识融合 |
|---------|:---:|:---:|:---:|:---:|:---:|
| S1 时间锚定 | → | ↑↑ | → | → | → |
| S3 同时反打 | ↑ | ↓ (需快速重定位) | ↑↑ | ↓ (切换打断沉浸) | → |
| S5 对比转场 | ↑↑ | → | ↑ | ↓ (刻意中断) | ↑ (反差激发情感) |
| S6 主题呼应 | → | ↑ | → | ↑↑ | ↑ (诗意联系) |
| S7 荒诞破局 | ↑↑ | → | ↑↑ | ↓ (刻意中断) | ↑ (角色声音强化) |
| S10 无声收尾 | ↓ (低密度) | → | ↓ | ↑↑ | ↑↑ (情感延续) |
| S13 宏微反差 | ↑↑ | → | ↑ | ↓ | ↑↑ (尺度震撼) |
| C1 浸入式跨章 | → | ↑↑ (零重定位成本) | ↑ | ↑↑ | ↑ |
| C2 钩子回收式 | ↑ | ↑ | ↑↑ | ↓ (悬念=元认知提醒) | → |
| C7 对话钩子开章 | ↑ | ↓ (首句无定向) | ↑↑ | → | ↑ |

> ↑ = 该衔接方式在此维度上提升心流，↓ = 可能干扰心流，→ = 中性。

**核心原则**：
- 定向性最强的技法（S1/S2/C3）保护维度二（目标清晰），但连续使用导致维度四（专注保护）的模式疲劳
- 反差类技法（S5/S7/S13）提升维度三（即时反馈），但干扰维度四——需要交替使用
- 情感类技法（S6/S10/C4）提升维度五（行动-意识融合），但降低维度三——适合情感向作品
- 章节边界的衔接方式直接影响读者"放下书后是否还回来"——优先 C2/C7（强钩子），其次 C3/C4

### 8.4 Agent 使用衔接技法的方式

#### 规划阶段（plan）
1. 启发式交谈维度 4b：为每个场景转换点 + 章边界选择衔接技法
2. 调用 technique-selector 推荐，参考 §7.4 选择矩阵
3. 如果指定对标作者：默认采用该作者的核心衔接技法（见 §7.4 风格对标速查）
4. 产出"场景衔接计划"表，写入心流设计卡

#### 执行阶段（execute）
1. 墨在写作前仪式步骤 3.6 预加载衔接计划
2. 写每个场景转换时：不是先写"---"再想衔接——而是场景 A 的最后 2 句就是衔接的开始
3. 桥接单元的写作顺序：场景 A 倒数第 2 句（收束准备）→ 场景 A 最后 1 句（桥接发射）→ "---" → 场景 B 前 2 句（锚点定向 + 展开）
4. 写完后在技法追踪表中标注衔接使用（计划内/自行选择/未按计划）

#### 评估阶段（evaluate）
1. 铃的全章心流评估维度 2.1c：衔接技法多样性
2. 铃的全章心流评估维度 2.1d：章间衔接质量
3. 反馈写回 notes.md（衔接模式记录），供下章 2.1c 跨章对比使用

#### 循环改进
- 每完成一章，notes.md 记录本章使用的衔接技法类型
- 连续 3 章使用同一衔接技法 → 铃标记 ❌ → 下章启发式交谈中五更强制推荐不同技法
- 每卷结束后回顾：本卷高频衔接技法 vs 低频衔接技法 → 在下卷规划时调整

### 8.5 衔接失败的常见症状与修复

| 症状 | 根因 | 修复方向 |
|------|------|---------|
| 读者说"场景切换太突兀" | 缺少定向（2.1a）或桥接（2.1b）| 在新场景前 2 句加入时间/地点/情绪锚点 |
| 读者说"怎么突然到这里了" | 硬切换——无任何衔接标记 | 从 §7.1 中选取至少 1 种衔接技法 |
| 读者说"节奏很奇怪" | 衔接技法选择与场景关系不匹配 | 对照 §7.4 选择矩阵：战斗→日常该用 S8 而非 S1 |
| 连续多章衔接方式单调 | 衔接多样性不足（2.1c）| 从 §7.1 中选取与当前技法类型不同的替代技法 |
| 读者放下书后不再拿起 | 章间衔接太弱（2.1d 硬切换）| 确保下章开头至少在 1 种衔接标记（C1-C8 之一）|
| 读完一卷觉得"故事讲完了" | 闭合:悬念比例失衡（V1 偏闭合）| 卷尾增加新悬念线——从本卷事件中自然生长 |

## 九、Agent 使用此方法论的方式

### 规划阶段

1. 用户确定主题 → Agent 协助建立 L1 全书级
2. Agent 以 L1 为"宪法"推导 L2 分卷规划
3. 每卷开始前，Agent 协助规划 L3 篇章结构
4. 每章开始前，Agent 读取 L3 确认本章位置和任务

### 写作阶段

1. Agent 根据场景规划卡生成内容
2. 生成后执行 AI 自校正
3. 完成后回写章节状态和相关追踪表

### 审阅阶段

1. 批评家 Agent 从四个视角交叉审阅
2. 反馈回写，下次生成时 Agent 自动关注问题维度
3. 定期（每 10 章/每卷）执行多视角一致性检查
