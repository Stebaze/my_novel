# 情节线程地图

> **定位**：追踪全书的情节线程（主线/支线/伏笔/线索交织）。与 `outline.md`（大纲规划，相对静态）分离——本文件是**动态追踪**，随着写作推进持续更新。
>
> **使用方式**：Agent 在写作前读取本文件获取当前态势，写作后更新对应追踪表。`plan-chapter` 启发式交谈的"全书态势扫描"和"伏笔操作清单"维度依赖本文件。

>

> - ⏰ 临近揭示 — 计划揭示章节在 ±3 章内

## 六、卷级节拍设计（自适应窗口）

> **用途**：节拍设计（每段剧情的节奏强度）和卷划分（卷边界对齐自然低谷）。
> **窗口总数是计算结果，不是设计目标。**
>
> 默认 **55章**（经天启预报1646章CV验证，CV=0.177）。实际值从 `novel/project-config.md` →「节拍窗口大小」读取。
> 每卷 2-3 窗完成一个完整的 💤蓄势→📈升温→🔥爆发(→🧊冷却) 呼吸周期。

### 本卷窗口规划

> 窗口 = **{W}章/窗**（来自 project-config.md）

| 窗口 | 章位 | 预计字数 | 类型 | 事件密度 | 决策密度 | 收束方式 |
|------|------|---------|------|:------:|:------:|---------|
| W{N} | Ch{X}-{Y} | ~{Z} | 💤/📈/🔥/🧊 | /窗 | /窗 | 次日/跳时/回溯 |

> **窗口类型**：💤蓄势(μ×0.75-1.05) / 📈升温(μ×1.05-1.25) / 🔥爆发(>μ×1.25) / 🧊冷却(<μ×0.85)
> **决策密度参考**：前期20-30/中期30-45/后期40-60/终局50-80+（每窗）
> **收束方式**：次日衔接(连续推进) / 跳时加速(压缩过渡) / 回溯填充(记忆收束)

### 卷节拍健康检查

- [ ] 每卷是否至少包含 💤→📈 两个阶段？（不足→节奏过于平直）
- [ ] 卷末是否为 🧊 或 💤 类型？（以🔥收尾→读者无消化空间）
- [ ] 🔥爆发窗之间是否间距 ≥2 窗？（过密→高潮贬值）
- [ ] 决策密度是否逐卷上升？（停滞→角色成长感缺失）
- [ ] 章节篇幅是否后期比前期长 30-50%？（持平→场景深度不足）

### 天启预报参考（55章/窗，30窗→11卷）

```
卷1-2(前期): 💤💤📈 | 💤📈💤          ← 每3窗=1卷，建立节奏
卷3-5(中期): 💤💤🧊 | 💤📈💤 | 🧊💤💤  ← 冷却窗密集=冲突高峰期
卷6-9(中期): 📈💤 | 💤💤📈 | 📈💤💤 | 💤📈💤
卷10(后期):  🔥🔥📈                    ← 双爆发窗=节奏加速
卷11(终局):  💤                        ← 终局收束(回溯密度282/窗)
```

## 附录：使用说明

### Agent 读取时机
- `plan-chapter` 阶段 3.0「视角准备」：读取主线状态 + 支线清单 + 伏笔追踪 + 卷级高潮节拍地图
- `plan-chapter` D1 高潮章子对话：读取卷级高潮节拍地图 + 支线高潮列 → 匹配桥段模板
- `chapter-review` 阶段 2：检查本章是否遗漏计划揭示的伏笔

### Agent 写入时机
- 写作后：新埋/回收/推进的伏笔 → 更新 §三 伏笔追踪
- 写作后：推进/新建立/收束的支线 → 更新 §二 支线清单
- D0b 高潮节拍规划后：高潮点 → 写入 §五 卷级高潮节拍地图
- 卷末：主线阶段推进 → 更新 §一 主线推进状态
- 导入章节后：`import-chapter` AI 扫描结果经作者确认后写入

### 与 outline.md 的分工
- `outline.md`：大纲规划（L1-L5），回答"计划写什么"
- `thread-map.md`：线程追踪，回答"实际推进到哪里了、还有什么在运行"
