# 角色弧光追踪

> **定位**：追踪每个主要角色的弧光进展、关系里程碑和出场频率。与 `outline.md`（大纲规划，相对静态）分离——本文件是**动态追踪**，随着写作推进持续更新。
>
> **使用方式**：Agent 在写作前读取本文件获取角色弧光当前位置，写作后更新弧光进展和关系变化。`plan-chapter` 启发式交谈的"全书态势扫描"依赖本文件。

## 附录：使用说明

### Agent 读取时机
- `plan-chapter` 阶段 3.0「视角准备」：读取所有主要角色的弧光位置 + 关系里程碑
- `chapter-review` 阶段 3（角色与设定层）：检查角色行为是否与弧光位置一致
- 作者问"这个角色接下来该怎样"时：读取弧光详情，给出基于弧光阶段的建议

### Agent 写入时机
- 写作后：角色弧光有推进 → 更新 §二 对应角色的"当前弧光位置"和"最近弧光事件"
- 写作后：关系有变化 → 更新 §三 关系里程碑进度
- 每 10 章：统计出场率 → 更新 §四 角色出场率追踪
- 卷末：弧光阶段推进 → 更新 §一 弧光总览 + §二 阶段状态
- 导入章节后：`import-chapter` AI 扫描结果经作者确认后写入

### 与角色档案的分工
- `characters/[角色名].md`：角色的静态设定（性格/背景/能力/语言指纹）
- `character-arcs.md`：角色的动态追踪（弧光推进/关系变化/出场频率）
