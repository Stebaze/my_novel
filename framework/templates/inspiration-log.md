# 灵感日志

> **定位**：Agent 在对话过程中实时捕获的灵感、想法、问题、临时决策。
> 这不是"写完之后的总结"，而是"对话中随时冒出随时记录"的活文档。
> Agent 有责任确保好想法不被遗忘。

---

## 记录原则

- **即时性**：灵感出现时立即记录，不等到对话结束
- **原始性**：记录时保留最初的语言，不美化、不整理
- **可追溯**：每条记录标注日期和来源上下文
- **可执行**：Agent 在后续对话中主动回顾未处理的灵感

## Agent 行为规范

### 关于灵感日志，Agent 必须：

1. **对话中发现用户说出一个好想法时**——主动问"这个要记到灵感日志吗？"或者直接记录（如果用户已授权）
2. **每次会话开始时**——浏览灵感日志，看有没有可以推进的开放问题或未处理的灵感
3. **在用户卡文时**——翻阅灵感池和名场面构想，看有没有可以激活的旧灵感
4. **灵感被采用后**——更新状态为 ✅，并记录采用了在哪一章

### 关于灵感日志，Agent 不得：

1. 不要评判灵感的"质量"——记录，不筛选
2. 不要在对话中突然列出整个灵感日志（除非用户问）
3. 不要让灵感日志变成另一个"待办清单压力源"——它的氛围应该是"宝库"，不是"欠债"
