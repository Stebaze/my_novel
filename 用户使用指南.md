# 用户使用指南：如何用本项目辅助写作

> **定位**：面向作者的操作手册。按"你说什么 → 系统做什么 → 你做什么"组织，覆盖从项目初始化到日常写作的完整流程。详细规则见 [`CLAUDE.md`](CLAUDE.md)；方法论见 [`framework/guides/`](framework/guides/)。

---

## 核心理念

**AI 不写正文。** 这个项目的角色是：

- **写之前**：帮你理清方向，产出"写作简报"——告诉你可以怎么写、用什么技法、注意什么
- **写之后**：帮你发现问题，产出"修改报告"——指出哪里卡、哪里崩、哪里角色行为不一致

**笔始终在你手里。** 简报不是必须照做的剧本，是可以偏离的起点。评审报告不是批评，是帮你看自己盲区的工具。

---

## 完整写作循环

日常写作遵循 4 session 循环：

```
SESSION 1 — 规划（plan）              SESSION 2 — 生成（generate）
  /plan-chapter {N}                      /generate-chapter {N}
  → 方向卡 + handoff                     → 简报 + 章节 + 评审 + Fix ≤ 2 轮

[可选] SESSION 3 — 人工复审           SESSION 4 — 发布
  /chapter-review {N}（writing）         /publish-chapter {N}
  → 评审 + 修订循环 + 定稿               → 设定合并 + 琉璃校验
```

详细 4 session 工作流图见 [`CLAUDE.md`](CLAUDE.md) 与 [`framework/_specs/interaction-spec.md`](framework/_specs/interaction-spec.md)。

## 写第X章——plan-chapter 完整 walkthrough

### 触发

- 「写第 X 章」/「规划第 X 章」/「续写」/「继续写」/「下一章」

### Step 1：系统管道（自动，10-30 秒）

系统在后台完成：

1. **前置就绪检查**（`pre-flight-check` C0-C8）：8 项门禁
   - 🚫 硬阻断（如缺少 author-voice.md）→ 系统停下来，等你补齐
   - 🟡 软阻断（如某角色声音未建立）→ 警告 + 你确认
2. **设定快照**（`settings-manager`）：冻结截止前一章的所有设定
3. **草稿初始化**（`file-manager`）：确保写作空间就绪

### Step 2：五更启发式交谈（核心交互，5-15 轮对话）

`qing-novelist` 加载五更人格与你对话。

**开场白**：

> "关于这一章，你有没有已经想写的内容？某个具体的场景、某段对话、某个情绪转折？"

**两种路径**：

| 你有想法 | 你没想法 |
|---------|---------|
| 五更围绕你的想法展开，帮你嵌入章节结构、衔接伏笔和角色弧光 | 五更基于大纲/伏笔/角色状态给你几个方向建议，你从中选择或组合 |

**交谈风格**：从不让你在抽象标签间选择。需要你做决定时，会写两段具体的文字（如两种不同风格的开头），让你读到感觉后说"这个更对"。

**可能聊到的话题**：

| 话题 | 什么时候聊 |
|------|-----------|
| 本章目的——铺垫/高潮/过渡/日常？ | 每章必确认 |
| 场景选择——写哪几个场景？ | 你不确定时 |
| 技法推荐——用什么断章/衔接/叙事技法？ | 每章推荐 |
| 风格锚点——冷峻/温热/抒情？ | 风格切换时 |
| 对标作者——参考某作家的写法？ | 你提及时 |
| 伏笔操作——本章埋/回收哪些伏笔？ | 线程地图存在时 |
| 角色声音——某角色还没确立说话风格？ | 声音缺失时 |
| 全书态势——各支线/伏笔/弧光健康吗？ | 新卷/新篇章开始时 |

**产出**：方向卡（`_briefs/chapter-{N}-direction.md`）。你确认"方向可以"后进入 Step 3。

### Step 3：简报生成（自动，10-30 秒）

调 `mo-writer` 生成 7 层简报（`_briefs/chapter-{N}-brief.md`），包含：
- 平台节奏提醒、场景结构骨架、角色出场与任务
- 心流曲线设计、技法执行指南

调 `yin-illustrator` 为每个场景设计文字视觉（氛围/色调/关键画面）。

简报出现在 `_briefs/chapter-{N}-brief.md`，系统会呈现概要。

### Step 4：你写正文

**这一阶段没有交互。** 打开简报，自己写正文。怎么写、写到什么程度偏离简报，完全由你决定。

### Step 5：进入下一 session

开新对话 → `/generate-chapter {N}`（基于 handoff 文件）。

## /generate-chapter — AI 生成整章 walkthrough

> **适用场景**：规划已完成（handoff 已落盘），你想让 AI 生成整章。

### 触发

新会话中输入 `/generate-chapter {N}`。

### Step 0：handoff 验证（自动）

`pre-flight-check` C8 验证 `_briefs/chapter-{N}-handoff.md` 存在。
- 🚫 硬阻断：缺失则不能继续

### Step 1：写作简报生成（10-30 秒）

调 `mo-writer`（基于 handoff 输入）生成 7 层简报。

### Step 2：场景生成 + 200 字摘要 + 自动拼装（1-3 分钟）

调 `sensory-writer`（per-scene 模式）：
- 逐场景生成正文（方法一/二/三 感官锚定）
- 每场景产出 200 字结构化摘要 → `_exchanges/scene-summaries.json`
- 自动拼装为完整章节 → `chapters/chapter-{N}.md`

### Step 3：自动评审（ai-content 模式）

调 `chapter-review`（`mode="ai-content"`，`auto=true`）：
- 调 `ping-critic` 一次完成指纹 + 校对 + 心流 + 三维评审
- AI 专项校验：事件落地 / 场景连贯 / 突兀收束 / POV 连续 / 伏笔核对
- 产出 `_reviews/chapter-{N}-review.md`

### Step 4：[条件] Fix 循环（最多 2 轮）

- 无 🔴 项 → ✅ 通过，跳过
- 🔴 ≤ 2 且定位明确 → 调 `sensory-writer` 重写受影响场景
- 🔴 > 2 或定位模糊 → ⚠️ 警告 + 用户选择"进入 Fix" / "人工修订"

### Step 5：落盘

`chapters/chapter-{N}.md` + `_reviews/chapter-{N}-review.md` + `_reviews/chapter-{N}-fix-log.md`

### 后续

- 想人工复审 → `/chapter-review {N}`（writing mode）
- 没问题 → `/publish-chapter {N}`

## 评审第X章——chapter-review 完整 walkthrough

### 触发

- 「评审第 X 章」/「第 X 章写完了」/「帮我看看第 X 章」

### 3 mode 自动选择

| mode | 何时用 |
|------|------|
| **writing**（默认）| 人工执笔写的章节 |
| **ai-content** | AI 生成的章节（自动评审已被 generate-chapter 跳过/重做时用）|
| **adaptation** | 改编已有作品 |

### Step 1：自动评审（20-60 秒）

调 `ping-critic` 一次性完成：
- **心流五维 18 项**：读起来会不会卡？认知负荷合理？情绪递进自然？
- **AI 指纹 + 校对**：错别字/语病/标点/分段 + AI 痕迹扫描
- **三维评审**：结构（场景分布/节拍/断章）+ 角色设定（一致性/关系进度）+ 文本质地

产出 `_reviews/chapter-{N}-review.md`。

### Step 2：呈现结果

```
心流评级：顺畅 / 有摩擦 / 断裂 / 无法进入
🔴 必须修：X 处（直接矛盾、心流断裂点）
🟡 应该修：X 处（节奏问题、细节不一致）
⚠️ 建议：X 处
校对统计：语病 X / 错别字 X / 标点 X / 分段 X
```

### Step 3：修订循环

你修改 → 说「改好了」→ 重新评审 → 出新报告。**不限轮次，你决定何时定稿。**

**自助选项**：

| 你想做什么 | 说什么 |
|-----------|--------|
| 只看某个场景的问题 | 「只查场景三」 |
| 某个问题看不明白 | 「问题 3 展开说说」 |
| 某个问题不想改 | 「这个问题不改了」 |
| 直接定稿 | 「可以了，定稿」 |

### Step 4：定稿

你说「定稿」后，系统自动：
1. 更新设定变更追踪（`_changes.md`）
2. 更新角色连续状态（9 维，位置/身体/心理/关系等）
3. 更新伏笔/支线推进标记
4. 提示：「设定已更新。可以发布了吗？」

## 发布第X章——publish-chapter walkthrough

### 触发

- 「发布第 X 章」/「第 X 章已发布」

### 3 步骤

1. **设定合并**（`settings-manager` `merge`）：草稿 `_changes.md` 变更回写到正式 `novel/`
2. **草稿正文同步**：草稿 `chapters/chapter-N.md` 复制到正式 `novel/chapters/`
3. **发布前琉璃校验**（`ping-critic` `publish-verify`）：校验设定一致性和文本质量

完成后提示：已发布到 `novel/chapters/chapter-{N}.md`。

## 草稿三层体系

| 层 | 目录 | 性质 | 修改权限 |
|----|------|------|---------|
| **原稿** | `novel/_reference/` | 用户上传原始素材 | 🚫 禁止修改 |
| **草稿** | `novel/_drafts/{日期}/` | 多会话实验空间，独立设定副本 | ✅ 自由修改 |
| **正式稿** | `novel/chapters/` | 已发布版本 | ⚠️ 谨慎修改（锁定后不可回退）|

**迁移路径**：原稿 → 草稿（import-chapter 导入 / bootstrap-project 冷启动）→ 正式稿（publish-chapter 发布）

**草稿管理**：初始化（`file-manager` `ensure-draft`）/ 写作期（独立设定副本互不干扰）/ 接受合并（`settings-manager` `merge`）/ 废弃（保留作为历史）

## 设定时间线管理

所有设定以**章节为版本标记**，每条信息标注"引入章节"。

- **读取**：评审/修改第 N 章时，只参考引入章节 ≤ N 的设定条目
- **修改**：修改设定时向前扫描——检查章节号 > 旧引入章节的已有章节是否因此冲突
- **冲突分级**：🔴 必须修（直接矛盾/能力越界）/ 🟡 应修（关系跳跃/细节不一致）/ 🟢 提示（风格微调/可选）
- **角色状态追踪**：每章结束时记录出场角色的 9 维状态（时间/空间/身体/心理/关系/目标/情绪/持续行动/关键物品）

详细操作（读取合并优先级、向前扫描执行步骤、差异对比）见 `settings-manager` Skill。

## 降级策略与阻断等级

所有 Skill 在发现所需文件缺失时，遵循统一顺序：

1. **先补齐**：调 `file-manager` 尝试从上层补齐
2. **补齐失败才降级**：明确标注"⚠️ 文件 X 缺失：能力 Z 已降级为 Y"
3. **降级不静默**：每次降级在阶段输出中体现

**阻断等级**：

| 等级 | 含义 | 例子 |
|------|------|------|
| 🚫 硬阻断 | 不能继续，必须解决 | author-voice.md 缺失 → 禁止进入简报生成 |
| 🟡 软阻断 | 警告 + 用户确认后放行 | 出场角色档案大面积缺失 |
| ⚠️ 提醒 | 标注影响范围，不阻断 | 参考指南缺失 |

**强制规则**：Skill 调用失败必须 🚫 硬阻断，禁止 ⚠️ 降级后继续。仅文件/Guide 缺失可选 ⚠️。8 条强制规则字面见 [`CLAUDE.md`](CLAUDE.md)。

## 常见问题 FAQ

### 我该选哪条路径开始项目？

| 你的情况 | 选择 |
|---------|------|
| 已有成稿（哪怕只有几章）| 冷启动（"初始化项目"）— 系统从你的正文反推大纲和设定 |
| 完全从零开始 | 新书项目（"我要写一本小说"）|
| 有原作要改写 | 改编（"改编这个作品"）|

### 冷启动（bootstrap）和导入章节（import）有什么区别？

**冷启动** = 完整项目初始化——批量导入 → 全文逆向分析 → 作者确认 → 工件生成。适合"第一次把稿件纳入系统"。

**导入章节** = 轻量操作——导入单个/少量章节到已有项目的草稿，自动扫描设定，注册评审队列。适合"项目已在运行，只想补几章"。

### 我可以跳过规划直接写吗？

不可以。写前规划是强制管道——前置检查确保设定就绪，五更交谈确保方向清晰，简报确保你有足够的信息支撑写作。

### 我可以跳过评审直接发布吗？

不可以。评审确保设定一致性和文本质量——跳过评审可能导致后续章节基于有问题的设定继续写，越错越远。

### 简报中的参考示例我必须照着写吗？

不。参考示例是"如果你不确定怎么写，可以这样写"的参考，不是模板。你完全可以偏离它，用自己的方式写。偏离的部分会在评审中标注，但不代表错误。

### 评审说的问题我必须全改吗？

不。🔴 强烈建议修，🟡 你可以选择修或不修，⚠️ 完全由你决定。使用「这个问题不改了」可以标记为"作者确认不修改"，不阻塞定稿。

### 我可以只评审部分内容吗？

可以。说「只查场景三」即可指定评审范围。

### AI 生成章节和人工写章节的评审有什么区别？

`chapter-review` 提供 3 mode：
- `writing`（默认）— 人工写章节的标准评审
- `ai-content` — AI 生成章节，含场景级机器化校验（事件落地/场景连贯/突兀收束/POV 连续/伏笔核对）
- `adaptation` — 改编章节，含原作对齐检查（严格/平衡/宽松）
