# C7：角色状态连续性检查

**条件**：N > 1，强制执行，🚫 **硬阻断**

> 状态字段定义参考 `framework/guides/jung-character-framework.md` §6（自性化阶段）§7（内在驱动更新）§10.4（settings-manager 追踪字段）。

## 执行

```
effective_prev = C3.6 定位的 target

读取 _character-state.md：
├── 不存在或空模板 → 🚫 阻断 → 回补（从 Ch{effective_prev} 正文提取）
├── 存在且含 Ch{effective_prev} 记录 → 对每个出场角色检查状态记录
│   └── 关键维度硬约束警告（已死亡/未愈伤/跨章持续活动中断）
└── 存在但不含 Ch{effective_prev} 记录 → 🚫 阻断：
    "Ch{effective_prev} 已写作但无角色状态记录——请调 settings-manager
    (record-character-state) 补齐"
```

## 标注

如 `effective_prev < N-1` → 报告中注明 "C7 以 Ch{effective_prev} 为准（Ch{N-1} 未写）"。
