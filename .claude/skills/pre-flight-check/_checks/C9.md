# C9：outline 实质填充检查

**条件**：scope="writing" 且 N 在 outline L3 已规划篇章范围内，🟡 **软阻断**

> 书级大纲门禁——尊重网文边写边定工作模式，不硬阻断。检测 `outline.md` L1-L3 是否实质填充（防止 `（待定）` 占位溜过门禁）。`outline-tingle`（正向）/ `bootstrap-project`（逆向）产出会被本检查检测为已填实。

## 步骤 1：触发判定

```
读取 {draft_dir}/outline.md（无草稿降级 novel/outline.md）
├── 文件不存在 → 跳过 C9（C3 已报缺失，本检查不重复）
├── frontmatter mode = "adaptation"：
│   ├── workflow_position = "outline-tingle-step2-done" → ✅ 直接通过
│   │   （改编大纲已由 adaptation-workflow 阶段 0.5 调 outline-tingle mode="adaptation" 产，
│   │    L1-L3 实质填充已由 outline-tingle 门禁 3 保证，C9 放行不报 🟡）
│   └── workflow_position ≠ "outline-tingle-step2-done" → 进入步骤 2 检测 L1-L3 填充，
│       修复路径走步骤 3 改编模式特例
└── frontmatter mode ≠ "adaptation"（原创模式或未设）→ 解析 L3 段（## L3 到 ## L4 之间；无 L4 则到文件末尾）：
    ├── 找出所有 `### 第{n}篇：{title}` 块及其章位表
    ├── 篇章"实质规划"判定：title ≠ `（待定）`
    ├── 无任何实质规划篇章 → 跳过 C9（避免空 outline 全报）
    └── 有至少一个实质规划篇章 → 取这些篇章章位表中章节列最大值 max_planned_chapter
        ├── N > max_planned_chapter → 跳过 C9（超纲章节，避免边写边定误报）
        └── N ≤ max_planned_chapter → 进入步骤 2
```

## 步骤 2：L1-L3 实质填充检测

```
扫描 outline.md 的 L1、L2、L3 段（## L1 到 ## L4 之间；无 L4 则到文件末尾）：
├── 任一字段值含 `（待定）` 占位 → 标记问题字段
└── 解析 frontmatter `pending_confirm` 数组：
    └── 数组含 L1 字段名（核心主题/终点画面/主角起点状态→终点状态/不可违背的规则/
        一句话概括/核心隐喻/类型标签 等）→ 标记问题字段

判定：
├── 无问题字段 → ✅ 通过（report "outline L1-L3 已实质填充"）
└── 有问题字段 → 🟡 软阻断，报告问题字段清单
```

## 步骤 3：修复路径

### 原创模式（frontmatter `mode: original` 或未设）

| 路径 | 调用 |
|------|------|
| 正向（从 premise 起） | `/outline-tingle`（Session 1 divergent）→ `/outline-tingle continue`（Session 2 grilling） |
| 逆向（已有成稿/素材） | `/bootstrap-project`（Phase 1a 大纲逆向） |
| 手动 | 直接编辑 `outline.md` L1-L3 字段，清除 `（待定）`，frontmatter `pending_confirm` 清空 |

### 改编模式（frontmatter `mode: adaptation`）

C9 改编模式修复路径按 `workflow_position` 当前状态分流——避免 mid-Session-2 状态被误导向 `/adaptation-workflow` 阶段 0.5（后者会从头触发 Session 1 divergent，覆盖已选定的改编主题方向）：

| workflow_position 当前值 | 缺失环节 | 修复入口 |
|--------------------------|---------|---------|
| 空/缺失/非 outline-tingle-* | Session 1 未跑 | `/adaptation-workflow` 阶段 0.5（从头编排 outline-tingle mode="adaptation" Session 1 → Session 2） |
| outline-tingle-step1-done | Session 2 未完成 | **`/outline-tingle continue`**（直接续 Session 2，不重跑 Session 1——避免覆盖已选定改编主题方向） |
| outline-tingle-l1-confirmed | L2/L3 未完成 | **`/outline-tingle continue`**（断点续传 L2/L3） |
| outline-tingle-step2-done | （步骤 1 已判通过，不应进入此表） | — |

| 路径（逆向/手动） | 调用 |
|------------------|------|
| 逆向（从原作画像/原作章节提取） | `/bootstrap-project` |
| 手动 | 直接编辑 `outline.md` L1-L3 字段 + 补 B9 三态标注 |

> 改编大纲以原作画像为素材源，原作画像缺失时先跑 `/adaptation-workflow` 阶段 0。`workflow_position` 达 `outline-tingle-step2-done` 后 C9 直接放行（步骤 1 已判定）。mid-Session-2 状态（step1-done / l1-confirmed）必须走 `/outline-tingle continue` 续传，**不**重跑 `/adaptation-workflow` 阶段 0.5——后者会触发 outline-tingle Session 1 重新 divergent，覆盖作者已选定的改编主题方向。

## 降级

- `outline.md` 文件缺失 → 跳过 C9（C3 已报 🚫/⚠️，本检查不重复）
- frontmatter YAML 解析失败 → ⚠️ 提示"outline.md frontmatter 损坏，无法判读 mode/pending_confirm，C9 仅基于 `（待定）` 文本扫描"
- frontmatter 缺 `pending_confirm` 字段（v1 outline） → 仅基于 `（待定）` 文本扫描；提示"outline.md 为 v1 格式，建议调 `migration-keeper` (migrate-project) 迁移到 v2"
- L3 段格式异常（无 `### 第{n}篇` 标题） → ⚠️ 提示并跳过步骤 1 篇章范围判定，回退为"只要 L1-L3 含 `（待定）` 即 🟡"
