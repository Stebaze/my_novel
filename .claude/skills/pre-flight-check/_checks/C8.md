# C8：Handoff 文件验证

**条件**：scope="writing"，🚫 **硬阻断入口**

> 4-Skill 对称架构的入口硬检查——`generate-chapter` 启动时必须验证 plan→generate 跨 Session 状态交接文件存在且契约完整。完整契约见 `framework/_specs/interaction-spec.md` §2.4。

## 步骤 1：文件存在性

```
目标文件：{draft_dir}/_briefs/chapter-{N}-handoff.md
├── 不存在 → 🚫 硬阻断：
│   "Ch{N} handoff 缺失——必须先调 plan-chapter（阶段 0-5），
│    plan-chapter 完成后会自动写 handoff。或用户在新 session 输入
│    /generate-chapter {N} 触发后续生成（前置条件：plan-chapter 已完成）"
└── 存在 → 进入步骤 2（字段契约验证）
```

## 步骤 2：8 字段契约验证

按 `framework/_specs/interaction-spec.md` §2.4 定义的 8 字段逐一验证：

| 字段 | 必填 | 校验 |
|------|:---:|------|
| `chapter` (int) | ✅ | 缺失或值 ≠ N → 🚫 |
| `direction` (path) | ✅ | 缺失或文件不存在 → 🚫 |
| `brief` (path) | 🟡 | workflow_position 前缀为 `generate-*` 时必填；缺失或文件不存在 → 🚫 |
| `chapter_file` (path) | ✅ | 缺失或路径不以 `chapters/` 开头 → 🚫 |
| `character_state` (path) | ✅ | 缺失或文件不存在 → 🚫 |
| `style_profile` (path) | ✅ | 缺失或文件不存在 → 🚫 |
| `workflow_position` (string) | ✅ | 缺失或格式不符（不含三段式）→ 🚫；前缀非 `plan-` / `generate-` / `review-` → 🟡 |
| `resume_command` (string) | ✅ | 缺失或格式不符（不以 `/` 开头）→ 🚫 |

`path` 字段以 `{draft_dir}` 为根的相对路径。`workflow_position` 用 `<skill>-<step>-<artifact>` 三段式。

## 步骤 3：输出

- 全部 ✅ 通过 → 报告 "Handoff 验证通过：{N} 字段全填入 + 简报状态={have_brief}"
- 任一 🚫 → 报告 + 修复路径：
  - "调 `plan-chapter {N}` 重跑阶段 5（settings-manager record-handoff）"
  - "或手动：复制 `framework/templates/chapter-handoff-template.md` → 填写 8 字段"

## 降级

- `{draft_dir}` 不可用（无草稿）→ 跳过 C8（无草稿即无 handoff 概念，scope="writing" 应在草稿初始化后执行）
- frontmatter 解析失败（YAML 损坏）→ 🚫 "handoff 文件 frontmatter 解析失败，请校验 YAML 语法"
