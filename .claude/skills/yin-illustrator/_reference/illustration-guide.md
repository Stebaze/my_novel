# 插画叙事策略指南

> **定位**：本文档将论文「基于心流理论的小说插画使用方法研究」的核心框架转化为可操作的插画策略工具。Agent「音」在规划视觉元素、生成插画 prompt、审阅图文一致性时参考此文档。
>
> **理论基础**：契克森米哈赖心流理论、四种图文互动模式、四阶段AI辅助创作法、多平台提示词工程。

## 一、图文互动模式系统

### 1.1 四种互动模式

插画不是文字的"配图"——它是叙事的有机组成部分。根据论文框架，插画与文本的互动关系分为四种模式：

| 模式 | 定义 | 插画做什么 | 文字做什么 | 读者体验 |
|------|------|-----------|-----------|----------|
| **补充式** | 插画补充文字难以描述的细节 | 呈现角色表情、场景氛围、服饰纹理、环境色调——那些文字写出来太啰嗦但读者需要的具象信息 | 推进剧情、写对话、写内心活动——文字擅长的事 | "看到了文字没写但应该有的东西" |
| **隐喻式** | 插画通过象征元素传达未明言的主题/情感 | 用色彩、构图、象征物（破碎的镜子/孤灯/风暴中的树）传达角色的内心状态或章节主题 | 文字聚焦于外部事件和对话，情感藏在水面下 | "感觉到了一些文字没有说出口的东西" |
| **重构式** | 插画以视觉语言重新诠释文本内容 | 从另一个角度/层次呈现同一事件——文字写的是男主视角的战斗，插画展示的是远处旁观者看到的景象 | 文字提供事件的某一层叙述 | "原来这个场景还可以这样被看见" |
| **冲突式** | 插画与文本形成张力 | 画面与文字描述存在微妙差异——文字说"她笑了"，画面中的笑却让读者感到不安；文字说"一切正常"，画面角落有不该存在的东西 | 文字提供表面叙述 | "文字说的是这个，但画面在暗示别的……到底哪个是真的？" |

**四种模式的共同原则**：插画不能说文字已经说过的东西。如果文字已经描述了"她穿着白色的连衣裙站在海边"，插画就不该只是把这个句子画出来——要么补充细节（裙摆被风吹起的弧度、脚边退潮留下的泡沫），要么换一个模式（隐喻：她的影子在沙滩上拉得很长，像一个被钉在原地的人）。

### 1.2 模式选择策略

按场景类型和情感目标选择互动模式：

| 场景类型 | 推荐模式 | 原因 |
|----------|----------|------|
| 日常温馨 | 补充式 | 读者需要看到角色在"日常"中的样子——环境的具体质感、角色放松时的微表情 |
| 情感爆发（吵架/告白/离别） | 隐喻式 | 情感太浓时直接画出来容易过火，象征物更能让读者自己体会 |
| 战斗/动作高潮 | 补充式 | 动作场面的空间关系和动态需要视觉锚定 |
| 伏笔揭露/真相揭晓 | 冲突式 | 让画面暗示比文字先一步，"泄露"不该泄露的信息 |
| 角色首次出场 | 补充式 | 第一印象必须是视觉的——读者需要立刻"看到"这个人 |
| 章节结尾/断章 | 重构式或冲突式 | 给读者一个文字之外的视角，让断章的余韵拉长 |
| 世界观震撼场景（进入新世界/巨型建筑/奇观） | 补充式 | 文字写奇观太费字数，一张图让读者瞬间"站在那里" |
| 角色内心挣扎（独处/回忆/噩梦） | 隐喻式 | 内心戏不适合直白配图，象征物更有力量 |

**多插画章节的模式搭配规则**：
- 2张：一张补充式（锚定空间/角色）+ 一张隐喻式或冲突式（锚定情感/主题）
- 3张：补充式（建立场景）+ 隐喻式（情感深化）+ 重构式或冲突式（结尾余韵）

### 1.3 模式误用警告

| 错误 | 后果 | 正确做法 |
|------|------|----------|
| 全程补充式 | 插画变成"文字插图化"，失去独立叙事价值 | 至少每3张插画中有一张非补充式 |
| 冲突式用在日常场景 | 读者以为有阴谋/伏笔，结果没有 → 信任透支 | 冲突式只在真有"表里不一"信息时使用 |
| 隐喻式象征物太晦涩 | 读者看不懂，插画变成装饰 | 象征物必须在章内有文字锚点（出现过/被提及过） |

## 三、角色视觉档案

### 3.1 角色视觉档案模板

每个主要角色建立视觉档案，与 `novel/characters/` 中文字档案联动：

```
## [角色名] 视觉档案

### 基础外观（锁死——跨 prompt 不可变）
- 发型/发色：
- 瞳色：
- 脸型特征：
- 身高/体型：
- 标志性特征（痣/疤痕/配饰/特殊印记）：

### 风格化锚点词
> 1-3个短语，用于在 prompt 中锁定角色外观。不同平台翻译时保持语义等价。
- 锚点词1：
- 锚点词2：
- 锚点词3：

### 服装系统
- 日常服装：
- 战斗/特殊场景服装：
- 正式场合服装：
- 不可变元素（跨服装始终保留——如"左耳的银色耳钉"）：

### 视觉演变轨迹
| 章节 | 外观变化 | 原因 |
|------|----------|------|
| Ch1 | 初始外观 | - |

### 角色专属色调
- 主色：
- 辅色：
- 禁忌色：
```

### 3.2 角色视觉锚点词策略

**原理**：不同平台对同一外观描述的理解有差异。锚点词的设计目标是——即使用不同平台、不同模型版本，核心外观特征仍能被准确理解。

**锚点词设计原则**：
1. **用具体事物而非抽象描述**："杏眼" > "眼睛好看"；"狼尾发型" > "很酷的发型"
2. **组合锁定而非单一依赖**：发型+瞳色+配饰三个独立特征组合，即使某个平台对其中一个理解偏差，整体辨识度仍在
3. **避免平台特有术语**：不说"像XX画师风格的五官"（不同平台对画师风格理解差异大），说"高鼻梁/薄唇/尖下巴"（通用面部特征）
4. **锚点词跨平台翻译时保持"视觉等价"而非"字面等价"**：中文"丹凤眼"在英文 prompt 中宜译为 "almond-shaped eyes with an upward tilt at the outer corners"

### 3.3 跨 prompt 角色一致性策略

**可变 vs 不可变维度矩阵**：

| 维度 | 可变性 | 说明 |
|------|--------|------|
| 发型/发色 | 🔒 锁死 | 除非剧情需要（如剪发/染发），否则不变 |
| 瞳色 | 🔒 锁死 | 角色最稳定的辨识特征 |
| 脸型/五官比例 | 🔒 锁死 | 这是"像不像这个角色"的核心 |
| 身高/体型 | 🔒 锁死 | 影响角色在画面中的存在感 |
| 标志性配饰 | 🔒 锁死 | 如耳钉/项链/发饰——角色的视觉签名 |
| 表情 | 🔓 可变 | 随场景情绪变化 |
| 动作/姿态 | 🔓 可变 | 随场景行为变化 |
| 服装 | 🔓 可变 | 随场景切换，但保留不可变元素 |
| 光影/色调 | 🔓 可变 | 随场景氛围调整 |

**一致性校验规则**：
- 同一角色在不同章节的 prompt 中，🔒 维度的描述短语必须完全一致（逐字对照）
- 「音」生成新 prompt 时，自动从角色视觉档案中拉取🔒维度短语，不重写
- 审阅时发现同角色外观偏差 → 回溯检查是否某次 prompt 中🔒维度被改动

## 四、场景-插画映射策略

### 4.1 哪些场景需要插画

**判断标准（满足任意2条即触发）**：

1. **视觉新奇度**：读者从未"见过"这个场景类型（新地图/新建筑/新生物）
2. **情感峰值**：本章情感弧线的最高点或转折点
3. **角色关键帧**：角色做出决定性行动/表情的瞬间
4. **空间复杂度**：动作发生在复杂空间中（战斗/追逐/多人互动），纯文字难以让读者建立空间感
5. **断章钩子**：章节结尾需要视觉余韵延长读者期待

**不需要插画的场景**：
- 纯对话/信息交代场景（两个角色坐着说话）
- 过渡性场景（从A地点到B地点的路程，除非路途本身有视觉奇观）
- 重复性场景（与前章高度相似的环境/构图）

### 4.2 插画位置策略

| 位置 | 功能 | 适合模式 | 画面特征 |
|------|------|----------|----------|
| **章首** | 氛围定调——让读者一打开章节就进入情绪 | 补充式/隐喻式 | 环境/氛围为主，角色可以不在画面中 |
| **章中** | 情感强化——在关键场景处给读者一个视觉"重音" | 补充式/隐喻式 | 角色+关键动作，画面有明确的情绪焦点 |
| **章尾** | 钩子预告——让读者对下一章产生期待 | 重构式/冲突式 | 暗示性的、不完全的信息——看了画面反而更想读下一章 |

### 4.3 插画密度控制

- **每章1-3张**，常规章节1张，高潮章节2-3张
- **连续3章无插画** → 读者视觉饥饿 → 需要在第4章补至少1张
- **连续2章有3张插画** → 视觉过载 → 下一章减至1张
- **全书插画密度**：平均每章1.5-2张为健康范围

## 五、心流增强的插画设计

### 5.1 创作者心流：分阶段AI辅助创作法

论文提出的四阶段方法论，适配到本工程：

**阶段一：准备（文本 → 视觉概念）**
1. 「音」读取已完成场景的文本
2. 提取关键视觉元素：地点、时间（光线条件）、出场角色、关键动作、情感基调
3. 确定插画风格锚点（引用第二章风格圣经）
4. 选择互动模式（引用第一章模式选择表）
5. 输出：一篇 100 字内的主视觉描述草稿

**阶段二：设计（快速迭代）**
1. 将主视觉描述翻译为各平台 prompt
2. 每个平台生成 1-2 个版本作为概念验证
3. 对比各平台效果 → 选择最佳平台作为本章主力
4. 调整 prompt 参数（比例/风格预设/负向词）直到效果满意
5. 输出：各平台最终可用 prompt

**阶段三：深化（人机协作）**
1. 选择最佳生成结果作为基础
2. 标注需要保留的元素（角色表情/关键细节）和需要调整的元素
3. 微调 prompt → 重新生成 → 对比 → 选择最终版
4. 输出：定稿插画 + 定稿 prompt

**阶段四：完成（评估与存档）**
1. 用 IEQ 心流体验问卷框架自评（插画是否增强沉浸感？是否与文字形成有效互动？）
2. 将定稿 prompt 写入章节模板视觉锚点区
3. 更新角色视觉档案（如有外观变化）
4. 输出：归档到 `_drafts/` 或 `chapters/`

### 5.2 读者心流：三层增强机制

论文数据显示图文结合使读者心流体验从51%→73%。三个增强层：

**第一层：多感官锚定**
- 插画中的环境细节（潮湿的石板路/窗上的雾气/飘落的灰烬）触发读者的感官记忆
- 这些细节文字可以提但画出来更直接——读者"感受到"而不只是"读到"

**第二层：叙事锚点**
- 插画为抽象的情节节点提供具象参照——读者脑中有了"这个场景长这样"的参照系
- 后续文字提到同一地点/物品时，读者自动调用视觉记忆，认知负荷降低

**第三层：情感锚定**
- 颜色和光影直接作用于情绪——暖金色=安全感，冷蓝灰=疏离，暗红=危险
- 插画预设了读者的"情绪频道"，文字在此基础上调谐而非从零建立

### 5.3 心流断裂检测（插画维度）

审阅时检查以下断裂信号：

| 断裂信号 | 表现 | 修复方式 |
|----------|------|----------|
| **风格跳变** | 连续两章插画风格明显不同（如同一个角色从二次元变成半写实） | 检查风格锚点词是否被改动；恢复一致 |
| **角色漂移** | 同角色在不同章中面部特征不一致 | 回溯🔒维度锚点词，逐字对齐 |
| **情绪错位** | 插画的情绪基调与文字情感目标冲突（文字是悲伤场景，插画光线明亮温暖） | 调整光影/色调参数，重新生成 |
| **密度失衡** | 连续5章无插画，或剧情平淡章有3张插画而高潮章只有1张 | 调整插画分配计划 |
| **模式单调** | 连续4张以上使用同一互动模式（全是补充式） | 至少换一张为隐喻式或冲突式 |
| **视觉过载** | 单张插画信息量过大（>3个焦点），读者不知道该看哪里 | 简化构图，单一焦点 |

## 六、多平台AI插画提示词工程

### 6.1 主视觉描述（Master Visual Spec）

Master Visual Spec 是所有平台 prompt 的**单一真相来源**。它用平台无关的语言描述画面，各平台 prompt 从它翻译。

**Master Spec 模板**：

```
【构图类型】：（半身特写/全身/远景/鸟瞰/特写局部/双人构图/多人构图）
【角色】：（谁在画面中——位置/姿态/朝向）
【动作/表情】：（在做什么——具体动作+面部表情，用动词而非形容词）
【场景环境】：（在哪里——地点+时间+天气/光线条件+关键环境细节）
【光影色调】：（光源方向+主色调+对比度+氛围关键词）
【情绪氛围】：（画面传递的情绪——用1-2个情绪词）
【视觉焦点】：（读者视线应该首先落在哪里——用构图语言描述）
```

**Master Spec 实例**：

```
【构图类型】：半身特写，角色居中偏右，视线向左
【角色】：少女，侧身而立，右手抬起似要触碰什么，左臂自然下垂
【动作/表情】：嘴角微扬但眉头轻蹙——笑里有犹豫；右手手指微曲，停在半空，没有真的触碰
【场景环境】：傍晚教室，夕阳从左侧窗户斜射入室，空桌椅排列整齐，空气中可见微尘在光柱中浮动
【光影色调】：暖金色侧光，暗部偏冷蓝灰，整体对比度中等偏低，柔光氛围
【情绪氛围】：不舍与释然交织
【视觉焦点】：右手停在半空的手势——"想触碰却收回"
```

### 6.4 角色一致性 Prompt 策略

#### 角色锚点词模板

每个角色在视觉档案中定义一组**锁死的外观短语**。每次生成该角色的插画时，这组短语**原样复制**到 prompt 中，不做任何改写。

```
锚点词1：[发型/发色] — 如"深棕色及肩短发，刘海微斜"
锚点词2：[瞳色/眼型] — 如"琥珀色杏眼，眼角微微上挑"
锚点词3：[标志性特征] — 如"左耳戴一枚银色小耳钉"
```

#### 可变 vs 不可变

生成不同场景的角色 prompt 时：

- **复制（锁死）**：锚点词1+2+3、脸型描述、身高体型
- **重写（可变）**：表情、动作、服装（但保留服装中的不可变元素如配饰）、光影色调

#### Seed 策略（Midjourney / 支持seed的平台）

- 首次生成角色时记录 seed 值
- 同角色后续生成使用相同 seed → 面部特征一致性显著提升
- 如果角色外观因剧情需要改变（如剪发），使用新 seed 并更新角色视觉档案

#### 参考图策略（跨平台）

- 在角色视觉档案中保存1张"基准图"——最能代表该角色标准外观的生成图
- 在支持图片参考的平台（MJ `--cref`、即梦图生图）中使用基准图作为参考
- 在仅支持文字的平台中使用锚点词保证一致性

### 6.5 内容合规自动检查

#### 「音」的 prompt 合规扫描步骤

每生成一个平台的 prompt 后，自动执行：

**步骤1：通用红线扫描**
- [ ] prompt中是否包含政治敏感词汇？（检查清单：国家领导人/政治事件/领土争议相关词）
- [ ] prompt中是否包含色情/性暗示词汇？
- [ ] prompt中是否包含极端暴力/血腥/恐怖描述？
- [ ] prompt中是否指名了真实公众人物/明星？
- [ ] prompt中是否包含知名IP受版权保护的角色名？

**步骤2：平台特有敏感词扫描**
- [ ] 豆包/即梦：是否包含"武器""战斗""打架""血腥""恐怖""鬼"等词？（→ 替换为"竞技""对决""运动""较量"等）
- [ ] 国内平台：是否包含可能被误判的词汇？（如"革命""抗议""占领"→ 在非政治语境中可能是安全的，但需评估风险）
- [ ] Midjourney：是否包含 "gore""blood""weapon""nsfw" 等词？

**步骤3：上下文安全评估**
- [ ] 画面描述整体是否可能被AI安全系统误判？（如角色持剑的仙侠场景→在prompt中加"仙侠/武侠风格"明确类型语境）
- [ ] 情感表达是否可能被误读？（如"悲伤""痛苦"等情绪词在角色表情描写中正常，但需确保不在暴力语境中）

**步骤4：修复与标记**
- 发现违规 → 自动替换为安全等价描述 → 在 prompt 末尾加注释标注替换内容
- 无法安全替换 → 标记 ⚠️ 并提示用户人工判断

#### 各平台通用敏感词替换表

| 原词（可能触发审核） | 安全替换词 |
|---------------------|-----------|
| 战斗/打架 | 对决/较量/对抗 |
| 武器/剑/刀 | 道具/器械（或在前面加"仙侠/武侠"风格限定词） |
| 血/流血 | 红色痕迹/伤痕（或避免提及） |
| 恐怖/可怕 | 阴森/诡异的氛围 |
| 鬼/幽灵 | 灵体/虚影/幻象 |
| 死亡/杀死 | 消逝/落幕/倒下 |
| 枪/子弹 | 远程武器（或避免提及，仅用画面暗示） |
