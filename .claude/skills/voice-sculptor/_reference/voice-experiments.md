# 声音实验区

> **核心信念**：角色的声音不是在档案里"设计"出来的，是在写对话时"听"到的。
> 在写正式章节之前，先用这些实验找到每个人的声音。

## 使用原则

- 每个实验 300-800 字，不超过 15 分钟
- 写之前不看角色档案——写了再对照，检查是否一致
- 写得烂是完全正常的。实验的目的是"听见声音"，不是"写出好文章"
- 找到一个角色的声音后，把关键发现回写到 `novel/voice-bible.md`

## 实验类型

### A. 独白实验（找男主叙事声音）

**目的**：找到主角那个独一无二的"讲故事的方式"。

从以下选题中挑一个，用第一人称写 500 字：
1. 男主描述昨天发生的一件让他尴尬的事
2. 男主描述他最喜欢的一个地方（为什么喜欢/在这里待着是什么感觉）
3. 男主在深夜失眠，脑子里在想什么
4. 男主收到一条让他心跳加速的消息，描述他从看到消息到回复的全过程
5. 男主描述他第一次见到某个重要的人时的印象（可能和现在完全不同）

**检查**：写完读一遍。这个声音——是"自嘲吐槽型"还是"干涩观察型"？他的句子有多长？他用不用比喻？他停顿在哪里？

### B. 对话实验（找角色之间的化学反应）

**目的**：在不确定角色关系的情况下，用纯对话测试两个人的声音能否"碰撞"。

选题：
1. A 和 B 在争论一件完全无关紧要的事（午饭吃什么 / 周末去哪 / 哪部动漫最好看）
2. A 做了一件让 B 尴尬的事，B 试图讨回公道
3. A 不小心说出了不该说的话，沉默三秒后两人都假装没听到
4. 两人在深夜独处，有一搭没一搭地聊天，话题从日常滑向私人

**规则**：
- 只写对话和简短的动作标记（像剧本），不写旁白
- 写完遮住名字——能分得出谁是谁吗？如果分不出，说明两个人的说话方式太像了

### C. 场景切片实验（找场景的情感质感）

**目的**：在正式写一个场景之前，先用 300 字"速写"它，捕捉关键的情感瞬间。

选题：
1. 写一个"两个人都不说话但都在想很多"的 60 秒
2. 写一个"表面在说 A 但实际在说 B"的对话片段
3. 写一个"喜剧场景突然变得真诚"的转折瞬间
4. 写第一章的第一个 200 字（开幕——读者看到的第一个画面）

**检查**：这个场景的情感质感对吗？读者会感受到你想要的氛围吗？

### D. 文本挖掘实验（从已有文本提取声音模式）

**目的**：不写新东西——分析已经存在的文本，提取角色声音模式写入 novel/voice-bible.md。适用于：已有章节草稿、角色对话片段、参考作品中的角色对话、角色笔记中的台词。

**输入源**（任选其一）：
1. 已写好的章节草稿或片段（如 `novel/_drafts/` 中的内容）
2. 角色档案中的台词/对话笔记（如 `novel/characters/` 中的文字）
3. 参考作品中某个角色的对话段落（如 `reference/` 中的素材）
4. 灵感日志中的台词片段（`novel/inspiration-log.md`——横切工件，grep `^- \[` 提取带 `#台词` 标签的 bullet）

**挖掘流程**（Agent 执行）：

```
1. 调用 voice-sculptor Agent（Agent tool, subagent_type="voice-sculptor", Mode: mine）：
   ├── 读取源文本 → 提取该角色所有的对话 + 内心独白（如有）
   ├── 遮住名字——确认能分得清不同说话者
   ├── 结构化分析（5 个维度，每个维度给出文本证据）：
   │   ├── 声音类型：自嘲吐槽/干涩观察/热血燃向/冷静写实/诗意文学/其他
   │   │   └── 证据：从文本中摘 1-2 句最能说明类型的原文
   │   ├── 句长分布：统计角色对话的句长（<15 字/% | 15-30 字/% | >30 字/%）
   │   │   └── 证据：最短的一句 / 最长的一句
   │   ├── 锚点词：3-5 个"只有此人会说/此人用得特别多"的词汇/句式
   │   │   └── 证据：每个锚点词在文本中出现 ≥2 次，标注出处行
   │   ├── 对话节奏：快/慢/跳跃/抢话/习惯沉默/说话带刺/绕弯子/直奔主题
   │   │   └── 证据：一段能体现节奏特征的连续对话（3-4 轮）
   │   └── OOC 红线：从此人的对话模式中推断 1-2 种"此人绝对不会说的话"
   │       └── 证据：反推——文本中哪段对话暗示了这条红线？
   ├── 对照角色档案——挖掘结果与档案描述是否一致？不一致处标注
   ├── 写入 voice-bible.md（创建或补充角色子节，标注声音样本来源）
   └── 追加实验记录到本文档实验记录表
```

**输出格式**：
```
### 文本挖掘：[角色名] — 源：[文本来源描述]
[角色名]的对话从源文本中共提取 X 轮 / Y 字

**挖掘结果**：
- 声音类型：[…]（证据："[原文引用]"）
- 句长分布：短句 X% / 中句 Y% / 长句 Z%
- 锚点词：[词1] ×N次, [词2] ×N次, …
- 对话节奏：[…]（证据：[连续对话片段]）
- 绝对不会说：[…]（反推依据：[…]）
- 与角色档案一致？✅ / ⚠️ 不一致：[…]
→ 已写入 novel/voice-bible.md §[角色名]
```

**与生成式实验的区别**：

| | A/B/C 生成式 | D 挖掘式 |
|------|------------|---------|
| 输入 | 无（从零写） | 已有文本 |
| 产出性质 | 探索性——"这个声音对吗？" | 确认性——"这个声音是什么？" |
| 时机 | 角色尚无任何文本时 | 已有草稿/笔记/参考素材时 |
| 风险 | 写偏了——声音不自然 | 过拟合——把一次性的写法当成规律 |
| 后续 | 用 D 挖掘验证 A/B/C 的发现 | 用 A/B/C 生成测试 D 提取的规律 |

## 从实验到正式章节

当某个角色的声音在实验中稳定下来后：
1. 把发现的规律写入 `novel/voice-bible.md`
2. 把实验中最自然的对话片段标记为"声音样本"
3. 对照角色档案——档案中的描述是否需要根据实验发现来修正？

## Skill 调用协议

> 本实验区由 plan-chapter（pre-flight-check）和 qing-novelist 协同调用，不独立触发。实验执行通过 Agent 完成，产物写入 `novel/voice-bible.md`。

### 调用链

```
plan-chapter pre-flight-check C6（声音就绪检查）
  → 检测到角色声音缺失
  → 用户选择"跑实验"
  → pre-flight-check C7 询问模式：
      ├── 生成式（A/B/C）——无已有文本，从零写
      │   └── 调用 voice-sculptor Agent（Mode: generate）
      │       → Agent 写 300-500 字实验文本 → 自分析提取声音特征 → 写入 voice-bible.md
      └── 挖掘式（D）——有已有文本，分析提取
          └── 调用 voice-sculptor Agent（Mode: mine）
              → Agent 结构化分析 → 写入 voice-bible.md + 标注来源 + 不一致回注
  → 进入阶段 1

plan-chapter 阶段 3（启发式交谈）
  → 五更 Skill 内维度 11（角色声音建立）
  → pre-flight-check C6/C7 未覆盖的角色仍需建立声音时触发
  → 同样支持生成/挖掘双模式（Agent 调用同上）
  → voice-sculptor 写入 voice-bible.md（草稿优先）→ 继续其他维度
```

### 模式选择指南

| 情况 | 推荐模式 | 理由 |
|------|---------|------|
| 角色完全空白，无任何文本 | 生成式 A/B/C | 没有文本可挖掘，只能从零写 |
| 已有章节草稿或对话片段 | 挖掘式 D | 分析已有文本比凭空写更准确 |
| 已有角色档案中的台词笔记 | 挖掘式 D + 生成式 B | 先挖掘确认基线 → 再用对话实验测试 |
| 两个角色已有各自的文本但未同场 | 挖掘式 D（各自）+ 生成式 B（碰撞） | 各自声音确认后再测化学反应 |
| 参考作品中有对标角色 | 挖掘式 D | 从参考文本提取模式，不复制，找差异 |

### 触发条件

| 条件 | 触发点 | 行为 |
|------|--------|------|
| 出场角色的 novel/voice-bible.md 条目缺少声音样本 | pre-flight-check C6 | 提醒 + 可选跑实验（生成/挖掘） |
| 出场角色的 novel/voice-bible.md 条目缺少锚点词（<3 个） | pre-flight-check C6 | 提醒 + 可选跑实验（生成/挖掘） |
| 两个角色首次同场但对话化学反应未知 | qing-novelist 维度 11 | 推荐 B 类对话实验（生成式） |
| 新角色首次出场，尚无档案 | qing-novelist 维度 11 | 先建档案，再跑 A 类独白实验（生成式） |
| 已有草稿/对话片段/参考素材中有该角色文本 | qing-novelist 维度 11 | 推荐 D 类挖掘实验（挖掘式） |
| 用户上传了新的参考素材或写了角色笔记 | 维度 11 或独立调用 | 可选跑 D 挖掘——从新素材提取声音 |

### 实验产物写入规范

执行实验后，必须在 `novel/voice-bible.md` 中写入：

**生成式 A/B/C**：
- **角色尚无条目** → 创建二.角色名子节（用 novel/voice-bible.md §二模板）
- **已有条目缺声音样本** → 填入"声音样本"区
- **已有完整条目** → 用实验发现修正锚点词/潜台词模式区

**挖掘式 D**（额外要求）：
- 上述 3 条同上
- 此外必须在 novel/voice-bible.md 角色子节中标注声音样本的**来源**（如"源自 Ch1 草稿对话"或"源自 inspiration-log.md 台词"）
- 挖掘中发现的"与角色档案不一致"项 → 同步标注在角色档案对应字段旁（如 `⚠️ 声音挖掘发现实际对话倾向与档案描述不一致：[具体差异]`）

同时在本文档"实验记录"表追加一行。
