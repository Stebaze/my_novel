# 启发式交谈方法论（外移参考）

> 12 维交谈框架 + 5 阶段写作行为切换 + 决策支持 + 灵感捕获的详细方法论。被 `qing-novelist` Skill 在交谈时按需读取。

---

## 一、核心原则：用内容呈现选择

永远不要求用户在抽象标签间选择。为每个选项生成一段实际可用的内容——场景开头、角色独白、对话片段——让用户直接"感受到"选项的质地。

```
❌ 标签式："叙事声音偏向自嘲型还是观察型？"
✅ 内容式：基于用户已确认的背景，写两种叙事声音的同一场景开头，
   让用户读完后判断"哪个更像脑子里那个人的声音"。
```

### D8 风格参考详细处理

**触发场景**：
- 写作流：作者说"对标 XX 风格"或"用 XX 的技法指导我"
- 改编流：作者指定对标作者
- 维度 8 默认询问

**双模式处理**：

```
用户指定对标作者 →
  ├── profiles/authors/{作者}.md 存在 →
  │     询问模式：
  │     ├── A: 对标模式 — 叙述声音/语言风格/情节设计全部对齐参考作者
  │     │     示例："如果是{参考作者}，这个场景可能会用叙述者冷吐槽来处理情感落差"
  │     └── B: 指导模式 — 保持自己的声音，借鉴参考作者技法修补薄弱点
  │           执行流程（见下）
  └── 不存在 → 弹提示"是否先建立 XX 的作者档案？"
        ├── 是 → 切到 qing-novelist 分析模式（7 维提取）
        └── 否 → 跳过，标注"⚠️ 无档案，凭印象对标"
```

**B 指导模式执行流程**：

```
1. 诊断薄弱点：
   - 基于当前章节场景清单/情感目标/场景类型（战斗/日常/情感/过渡）
   - 薄弱点来源：author-voice.md 已知风格局限 + 用户自评 + 历史章节的琉璃反馈

2. 匹配技法：
   - 读参考作者档案，聚焦可迁移维度：§二 情节设计 / §三 关系设计 / §四 常用技法 / §六 篇幅结构
   - 不读取：§一 叙述者声音 / §七 语言指纹（由 author-voice.md 保护）
   - 为每个薄弱点匹配 1-2 个参考作者的具体技法

3. 呈现建议（内容呈现原则）：
   - 不要问"你想用风月的战斗节奏吗"——展示具体操作
   - 示例："你的原稿战斗通常在 3-4 段后出结果，有时读者会觉得拖。
            风月处理类似场景的做法是'一招一果'——攻击→击中→效果→对手反应，四步不超过 200 字。
            比如你这场[主角]对傀儡的战斗，可以试试：..."

4. 用户选择 → 采纳的技法标注"指导模式-已采纳"写入方向卡

5. 墨生成简报时：叙述声音锚定 author-voice.md，技法执行参照写入简报的技法提示层
   琉璃额外检查：借来的技法有没有越界替换叙述者声音？

6. 章末：用户采纳的技法写入 style-guide.md §特殊叙事技法，标注来源 + 采纳原因
```

### D1 高潮章子流程

当 D1 判断本章类型 = 高潮章时，叠加执行：

```
1. 查 thread-map：
   - 本章是否在卷级高潮节拍地图中？
   - 是否有支线高潮在本章？
   - 本章在五阶段中的位置？（开端/发展/高潮/结局/余震）

2. 桥段库匹配：
   - 加载 framework/templates/climax-patterns/_index.md
   - 有预设（卷级高潮地图已指定）→ 加载该桥段模板
   - 无预设 → 基于当前故事上下文推荐 2-3 个候选桥段（用内容呈现）

3. 5 阶段节拍映射：
   - 将选定桥段模板的 5 阶段映射到本章场景
   - 至少留一个"读完本章立刻想知道"的问题

4. 支线高潮协调：
   - 检查节拍是否冲突/互补

5. 节奏强化（参考 cn-webnovel-guide §4.3 + bridge pattern 事件密度建议）：
   - 高潮阶段事件密度：4-6 件/千字
   - 断章方式：答案前一刻/关系临界点/情绪反转/认知翻转
```

## 三、按写作阶段切换行为

### 阶段零：从零开始

触发：outline.md 空白，无角色档案。

1. 不问"你的设定是什么"——问"你脑子里最早出现的画面/台词/人物是哪个？"
2. 从碎片出发，立刻生成 100 字内容——让用户看到"这个碎片变成文字是什么样的"
3. 用户看到内容后自然产生"不对，应该是这样"的反应——这就是下一步方向
4. 追问"为什么被它吸引"——通往核心主题

退出条件：L1 核心主题和终点画面已确定。

### 阶段一：概念成型

触发：L1 已确定，大纲和角色待展开。

1. 用"如果…会怎样"探索可能性——每个"如果"附带 50 字场景片段
2. 每确定一个设定，追问"代价是什么"
3. 用角色声音实验帮用户"听到"角色：让角色在第一人称独白中介绍自己

退出条件：L2 分卷规划已确定，主要角色档案已建立基础字段。

### 阶段二：大纲构建

触发：L2 确定，需规划篇章和章节。

1. 用五级大纲体系从 L2 推导 L3
2. 每规划一个篇章，写 100 字"预告片段落"
3. 用关系里程碑排布
4. 问"这篇里哪个场景是读者会想截图的"，写 50 字画面

退出条件：L3 篇章规划已确定，关系里程碑已排布。

### 阶段三：逐章写作

触发：大纲就绪，开始写章节。详见 qing-novelist SKILL.md §Flow Step 1-6。

### 阶段四：审阅修订

触发：章节初稿完成。

1. 帮用户判断需要什么深度的审阅
2. 帮用户把"感觉不对但说不出来"翻译成具体修改方向
3. 深度审阅 → 建议调用 `chapter-review` Skill

## 五、决策支持

当用户犹豫不决时，读取 `framework/guides/decision-exploration.md`：

| 用户的问题 | 方法 | 内容呈现方式 |
|-----------|------|------------|
| "不知道 A 和 B 选哪个" | 后果涟漪 + 双写采样 | 为 A 和 B 各写 100 字场景开头 |
| "不知道角色会怎么做" | 角色完整性检查 | 用角色第一人称写 3-5 句内心独白 |
| "感觉不对但说不出来" | 消失的场景 | 生成"现在没有但你可能想看到的那个场景"的 100 字 |
| "大局上走到哪了" | 全书态势扫描 | 呈现弧光位置 + 伏笔状态 + 支线健康度 |

关键：永远不说"你应该选 A"。用内容让用户感受到每个选项的质感。

## 六、灵感捕获

对话中出现以下信号时**提示作者用 spark**（不自动写入——记录职责收敛到 spark Skill，避免交谈中途文件副作用）：

| 信号 | 行动 |
|------|------|
| 用户说了一句好台词 | → 提示"这句台词值得 `/spark <台词> #台词` 记一下吗？" |
| 用户描述了一个精彩场景画面 | → 提示"这个画面值得 `/spark <描述> #名场面` 记一下吗？" |
| 用户突然"对了！" | → 等用户说完，提示"要 `/spark <内容>` 记下来吗？" |
| 用户说"先不管这个，以后再说" | → 提示"要 `/spark <内容>` 记到灵感日志备用吗？" |

**原则**：qing-novelist 只负责"识别 + 提示"，写入动作由作者执行 `/spark` 触发。作者愿意就记，不愿意就放过——不打断交谈节奏。
