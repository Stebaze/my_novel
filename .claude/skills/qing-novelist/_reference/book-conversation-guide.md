# 书级 grilling 方法论（外移参考）

> 8 维书级交谈框架 + L1→L2→L3 逐层收敛 + 三门禁协作 + 文件缺失处理。被 `qing-novelist` Skill（mode=book）在书级 grilling 时按需读取。
>
> **对称关系**：本文件是 `_reference/qing-conversation-guide.md`（章节级 12 维）的书级对应物——章节级解决"这一章写什么"，书级解决"整本书定型成什么"。

## 一、核心原则：内容呈现选择（书级版）

永远不要求作者在抽象标签间选择结构。为每个结构决策生成一段 50-100 字的具体陈述——分卷的核心问题、主角弧光的起终点、终点画面的镜头——让作者直接"读到"那本书的质地。

```
❌ 标签式："第一卷的主题是成长还是代价？"
✅ 内容式：基于 Premise 段已选定的核心主题，写两版第一卷核心问题陈述——
   "第一卷的核心问题：主角能不能接受自己不再是普通人——卷末以他主动选择留下作答"
   "第一卷的核心问题：主角配不配拥有这种力量——卷末以他被力量反噬失去重要的人作答"
   让作者读完判断"哪本是我想要的"。
```

**与章节级内容呈现的差异**：章节级调 `sensory-writer` 生成场景片段（具体到对白/动作）；书级**不调 sensory-writer**——结构陈述句是抽象层，由五更直接生成。唯一例外是 B8 终点画面，可写简短画面陈述（镜头式，非完整场景）。

## 二、8 维书级 grilling 框架

| # | 维度 | 何时激活 | 关键追问方向 | 产出落点（outline.md 字段） |
|---|------|---------|-------------|---------------------------|
| B1 | 主题深度 | L1 阶段（始，第一维） | Premise 的表层主题之下，真正想探讨的命题是什么？主题的反命题是什么（不写什么）？主题在全书如何递进（不是重复）？ | L1「核心主题」+「一句话概括」 |
| B2 | 主角弧光起终 | L1 阶段（B1 后） | 主角起点是什么状态（信念/能力/关系）？终点是什么状态？弧光是平移（换环境不换人）还是翻转（换人）？起终的张力来自哪里？ | L1「主角起点状态 → 终点状态」 |
| B3 | 分卷核心问题 | L2 阶段（每卷一问） | 每卷在主线中不可替代的功能是什么？每卷读者读完感受到什么（情感目标）vs 推进什么（剧情目标）？卷与卷之间是递进/对比/反转？ | L2 每卷「卷主题」+「情感目标」+「剧情目标」 |
| B4 | 篇章高潮分布 | L3 阶段（每篇一图） | 每篇 5-7 章的迷你电影，核心问题是什么？高潮点落在第几章？章位表的开端/发展/高潮/结局/余震怎么排？篇末钩子是什么？ | L3 每篇「核心问题」+「关键事件链」+ 章位表 +「结尾钩子」 |
| B5 | 关系里程碑 | L2（卷级）+ L3（篇级） | 主要关系线（情感/敌对/师徒/...）每卷推进到什么阶段？每篇有什么关系里程碑？关系变化是因果驱动还是事件驱动？ | L2「角色弧光阶段」+ L3「关系里程碑」 |
| B6 | 不可违背规则 | L1 阶段（B1/B2/B8 后） | 连载膨胀的刹车在哪？3-5 条不可违背的规则是什么（世界观规则/叙事规则/道德底线）？规则之间有没有冲突？违反规则的代价是什么？ | L1「不可违背的规则」（3-5 条带编号） |
| B7 | 核心隐喻 | L1 阶段（B6 后） | 全书有没有一个反复出现的意象/隐喻？它在 L1→L3 如何变形？隐喻是装饰还是结构（推动情节的隐喻 vs 渲染氛围的隐喻）？ | L1「核心隐喻/意象」 |
| B8 | 终点画面 | L1 阶段（与 B1/B2 同期，可先可后） | 最后一卷最后一章，读者合上书看到什么画面？感受到什么？这个画面是不是 B1 主题 + B2 弧光的视觉化？ | L1「终点画面」 |

### B1 主题深度——详细协议

**追问序列**（grilling 范式，单一焦点持续深挖）：

```
1. Premise 段已选定的核心主题（来自 idea-explorer mode=book + 作者选定）→ 五更复述确认
2. 追问 1："这个主题的反命题是什么——你这本书坚决不写什么？"
   → 反命题界定了主题的边界，避免主题膨胀
3. 追问 2："如果读者读完只记住一句话，你希望是哪句？"
   → 这句话是一句话概括的草稿
4. 追问 3："这个主题在第一卷怎么入场？在最后一卷怎么收束？中间有没有翻转？"
   → 主题递进曲线（不是平面重复）
5. 产出 50-100 字主题陈述句（含主题 + 反命题边界 + 递进方向）→ 作者确认 → 交 outline-tingle 写入 L1
```

### B6 不可违背规则——详细协议

**为什么放在 B1/B2/B8 之后**：规则是为主题和弧光服务的刹车——先知道车往哪开（B1/B2/B8），再设护栏（B6）。

```
1. 扫描已定的 B1 主题 + B2 弧光 + B8 终点画面 → 识别"会膨胀的方向"
2. 追问："这个方向上，什么是一定不能发生的？发生就崩盘的？"
3. 每条规则带编号（便于门禁 C9 / plan-chapter 精确引用）：
   1. [世界观规则] 例："主角的能力有明确代价，每次使用消耗 X"
   2. [叙事规则] 例："不写主角视角外的关键信息（限制 POV）"
   3. [道德底线] 例："主角不杀无辜者（即使有理由）"
4. 规则数量 3-5 条（少于 3 护栏不足，多于 5 难以遵守）
5. 检查规则间冲突 → 产出 3-5 条带编号规则 → 作者确认 → 交 outline-tingle 写入 L1
```

### B8 终点画面——详细协议

**与 B1/B2 的关系**：终点画面是 B1 主题 + B2 弧光的视觉化——三者必须自洽。可先定终点画面反推主题，也可先定主题顺推终点画面。

```
1. 追问："最后一卷最后一章，最后一个镜头——读者看到什么？"
2. 不要求完整场景（书级阶段无具体场景），但要求"镜头式陈述"：
   "主角站在 X 地，手里拿着 Y，远处 Z——读者感到 W"
3. 校验：这个画面是否视觉化了 B1 主题？是否对应 B2 弧光终点？
   ├── 是 → 产出 50-100 字终点画面陈述 → 作者确认 → 写入 L1
   └── 否 → 反推调整 B1 或 B2（grilling 回溯）
```

## 三、L1→L2→L3 逐层收敛流程

```
L1 阶段（宪法层，不可暂缓）：
  ├── B1 主题深度 → L1 核心主题 + 一句话概括
  ├── B2 主角弧光起终 → L1 主角起点状态→终点状态
  ├── B8 终点画面 → L1 终点画面（与 B1/B2 交叉校验）
  ├── B6 不可违背规则 → L1 不可违背的规则（3-5 条带编号）
  ├── B7 核心隐喻 → L1 核心隐喻/意象
  └── 【L1 末门禁】B1/B2/B6/B7/B8 全部填实 → l1_ready=true → 进 L2
      任一含 `（待定）` → l1_ready=false → outline-tingle 拦截（不可暂缓）

L2 阶段（分卷层）：
  ├── 扫描 outline.md L1「计划卷数/总字数」→ 确定卷数
  ├── 逐卷激活 B3（分卷核心问题）→ 每卷 卷主题/情感目标/剧情目标/大高潮/卷末状态/本卷新角色
  └── B5 关系里程碑（卷级）→ L2 角色弧光阶段

L3 阶段（篇章层，每卷内）：
  ├── 逐卷扫描 L2 → 每卷分若干篇章（每篇 5-7 章）
  ├── 逐篇激活 B4（篇章高潮分布）→ 篇章功能/核心问题/关键事件链/角色聚焦/关系里程碑/情感曲线/结尾钩子 + 章位表
  └── B5 关系里程碑（篇级）→ L3 关系里程碑

【Session 2 末门禁】L1-L3 均实质填充（无 `（待定）`）→ handoff_ready=true
  → outline-tingle 推进 frontmatter workflow_position: outline-tingle-step2-done
  → 写 project-config.md（最小集）+ init-draft
```

**每维产出的结构陈述句**：50-100 字，含具体内容（不是标签）。交 outline-tingle 写入 outline.md 对应字段前过 `ping-critic(editor-consult)` 自检一致性。

**grilling 回溯**：L2/L3 阶段若发现与 L1 冲突，可回溯调整 L1——但 L1 调整必须重新过 L1 末门禁。

## 四、与章节级 12 维的区分（宪法层 vs 执行层）

| 维度 | 书级 8 维（mode=book） | 章节级 12 维（mode=chapter） |
|------|----------------------|---------------------------|
| 层级 | 宪法层（一次定型，长期不变） | 执行层（每章重新激活） |
| 修改成本 | 高——修改需记录到 notes.md 决策表 + 同步所有下级大纲 | 低——每章方向卡独立 |
| 激活频率 | 一次（outline-tingle Session 2） | 每章一次（plan-chapter 阶段 3） |
| 焦点 | 整本书的结构定型 | 单章的方向选择 |
| 内容呈现载体 | 50-100 字结构陈述句 | 50-100 字场景片段（调 sensory-writer） |
| 产出落点 | outline.md L1/L2/L3 字段 | _briefs/chapter-{N}-direction.md |
| 门禁 | L1 末门禁（不可暂缓） | pre-flight C0-C8（章节级） |

**互补关系**：书级 8 维定宪法（L1-L3），章节级 12 维在宪法下执行（L4-L5）。书级 grilling 完成后，章节级 grilling 在 plan-chapter 阶段 3 接力——章节级 D0 全书态势扫描会读取书级 B1-B8 的产出作为上下文。

**不互替**：不能用章节级 12 维代替书级 8 维（会陷入"每章都对但整本书散"）；也不能用书级 8 维代替章节级 12 维（会陷入"结构清晰但每章不知道写什么"）。

## 五、三门禁协作

门禁由 `outline-tingle` 编排，本 Skill（mode=book）仅返回字段填实状态：

| 门禁 | 位置 | 责任方 | 本 Skill 返回字段 | 阻断级别 |
|------|------|--------|------------------|---------|
| 门禁 1 | Session 1 末（主题选定） | idea-explorer mode=book + 作者 | 不涉及（本 Skill 未启动） | 可暂缓 |
| 门禁 2 | L1 末（B1/B2/B6/B7/B8 填实） | 本 Skill mode=book | `l1_ready: bool` | **不可暂缓** |
| 门禁 3 | Session 2 末（L1-L3 全填实） | 本 Skill mode=book | `handoff_ready: bool` | 可暂缓（但 outline-tingle 不推进状态机） |

**门禁 2 不可暂缓的理由**：L1 是宪法层——L2/L3 全部依赖 L1。L1 未定而进 L2 会产生"分卷不知道在分什么"的悬空结构。作者要求暂缓 L1 时，outline-tingle 提示"建议先回 idea-explorer mode=book 重新发散主题"。

## 六、文件缺失处理（强制降级不静默）

对称 `_reference/qing-conversation-guide.md` §四。mode=book 的文件依赖较少（书级阶段无需角色档案/声音圣经/线程地图——那些是章节级输入）。

| 缺失文件 | 补齐来源 | 补齐失败时降级处理 |
|---------|---------|-----------------|
| 草稿 `outline.md` | `file-manager` ensure-draft ← novel/ ← framework/templates/outline.md | 🚫 硬阻断——mode=book 的核心输入是 outline.md，缺失无法 grilling。提示"请先跑 /outline-tingle Session 1 或 file-manager ensure-novel 初始化" |
| `outline.md` frontmatter `workflow_position` 缺失/异常 | （无补齐——状态机字段） | ⚠️ 软阻断——标注"outline.md 状态机异常（期望 `outline-tingle-step1-done`）"，提示"建议跑 /outline-tingle 校准"。可放行进 grilling 但 handoff_ready 强制 false |
| `outline.md` Premise 段含 `（待定）` | （无补齐——由 idea-explorer mode=book 产） | 🚫 硬阻断——Premise 未定无法 grilling 主题。提示"请先跑 /outline-tingle Session 1（调 idea-explorer mode=book）选定主题" |
| `novel/project-config.md` | `file-manager` ensure-novel | ⚠️ 标注"无项目配置"，按原创模式（original）继续；adaptation 模式判定降级为 original |
| `framework/guides/narrative-engineering.md` | 框架文件 | ⚠️ 五级大纲体系方法论缺失，L1→L2→L3 收敛降级为常识推理 |
| 草稿 `inspiration-log.md` | `file-manager` ensure-draft | ⚠️ 标注"无灵感日志"，grilling 中产生的好点子直接交 outline-tingle 写入 outline.md（不暂存） |

**与章节级文件缺失处理的差异**：章节级依赖 author-voice.md / voice-bible.md / thread-map.md / character-arcs.md / _character-state.md（执行层输入）；书级**不依赖**这些——书级 grilling 时这些文件可能尚不存在（outline-tingle 是项目最早阶段，角色档案还未建立）。书级只依赖 outline.md + project-config.md。
