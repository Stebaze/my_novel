# 头脑风暴方法

> 被 `idea-explorer` Skill 在发散生成阶段读取。提供 7 种结构化头脑风暴方法的完整执行步骤和方法选择指南。

---

## 核心原则

1. **数量创造质量**：前 3 个想法往往是显而易见的。第 5-8 个才开始有趣。每个方法至少产出 3 个方向，坚持推够数量。
2. **用具体内容表达可能性**：每个方向必须有一段 50-100 字的场景片段或对话片段让作者"读到感觉"。调用 `sensory-writer` Skill 生成——不产出抽象标签。
3. **不做决定**：头脑风暴阶段不评价、不筛选、不预判好坏。所有想法先产出再说。收敛和选择是五更（`qing-novelist`）的工作。

---

## 方法 A：如果爆炸

**思维类型**：发散性组合
**适用场景**：完全不知道写什么 / 知道事件但想知道其他可能性

**核心逻辑**：将当前场景/角色/设定的 N 个元素拆开，对每个元素做"如果不是 X，而是 Y 会怎样"的替换，重构原场景。

**步骤**：

1. **列出当前"事实"**：
   - 本章必须发生的事（3-5 个硬约束）
   - 角色的当前状态（2-3 个关键状态——位置/情绪/身体状态/关系）
   - 已有设定的边界（2-3 条不可违反的规则）

2. **对每个硬约束生成变体**：对每个约束提 2-3 个"如果不是 X，而是 Y"的假设。至少生成 3 个不同的"如果"方向。

3. **为每个方向写场景片段**：一句话摘要 + 50 字场景开头（调 `sensory-writer`）。示例格式："如果不是 A 打败了 B，而是 B 在最后一刻选择了放过——B 松手的那一瞬，A 看到的是怜悯还是算计？"

4. **检查与事实约束的关系**：
   - 完全符合约束 → 可行
   - 轻微调整约束仍可行 → 需要调整
   - 违反不可违反的规则 → 不可行（记录为灵感日志·开放问题）

5. **追加组合**：将上一步中两个有趣的"如果"叠加——"如果不是 X + 如果同时 Y → 会发生什么？"——产出第 4 个方向（往往最有趣）。

**思维触发词**（卡壳时用）：
- "如果不是[角色A]，而是[角色B]在这个位置？"
- "如果不是今天，而是十年前？"
- "如果不是这里，而是完全不同的地点？"
- "如果不是战斗，而是谈判？不是成功，而是失败？"
- "如果不是 TA 主动，而是被动卷入？"

**每次至少产出**：3 个方向（含 1 个叠加方向）

## 方法 B：约束游戏

**思维类型**：限制性创造
**适用场景**：知道事件但场景太常规需要变形 / 需要增加场景张力和新鲜感

**核心逻辑**：添加 1-2 条额外的约束条件（时间/空间/信息/能力限制），迫使场景在受限条件下展开——限制反而激发创意。

**步骤**：

1. **列出当前计划场景的核心动作**（1-2 句）。

2. **选择约束类型**（随机或按需）：
   - 时间约束："必须在 X 时间内完成"（沙漏/倒计时/截止时间）
   - 信息约束：某个角色缺失关键信息（误解/误认/不知道某事已经发生）
   - 能力约束：核心能力暂时失效/被限制/代价过大不能再用
   - 空间约束：场景限定在狭小/公开/不可移动的空间
   - 社交约束：有第三方在场/不能暴露身份/必须维持特定社交角色
   - 随机物品约束：场景中多了一件（或少了一件）关键物品

3. **生成场景片段**：为每个选择的约束类型生成 1-2 个具体约束，并写 50 字场景片段展示"在约束 X 条件下——角色的每一步会怎么变？"

4. **追加"双约束"**：叠加两个约束（注意不要相互抵消或矛盾）。"如果[时间约束 A] + [信息约束 B]同时存在——哪种冲突最有趣？"

5. **标识约束是否可接受**：不违反已有设定 → 可行；违反 → 标注"设定冲突"记入灵感日志备用。

**思维触发词**（卡壳时用）：
- "如果 TA 必须在 10 秒内做决定？"
- "如果 TA 不知道对方已经知道了什么？"
- "如果 TA 的能力现在不能用了？"
- "如果还有别人在看着？TA 不能说真话？"

**每次至少产出**：3 个方向

## 方法 C：视角切换

**思维类型**：同理心跳跃
**适用场景**：角色行为可预测/太单一 / 想让读者看到隐藏维度 / 需要多角度叙事

**核心逻辑**：把当前场景从主角视角切换到另一个角色的视角来重看。不同角色的认知、情感和动机把同一事件折射出不同面貌。

**步骤**：

1. **列出当前场景的核心事件**（1-2 句 + 主角的行动）。

2. **列出场景中/附近的每个角色**（包括路人和反派），选择 2-3 个非主角角色。

3. **对每个选中角色写"如果在 TA 眼中"**：
   - TA 在这个场景中看到了什么？（和主角看到的不同吗？）
   - TA 此刻想要什么？（不一定和主角一致）
   - 主角的行为在 TA 眼中意味着什么？（完全不同的解读）
   - 写 50-80 字该角色的内心独白或第一人称感受（调 `sensory-writer`）

4. **识别最有张力的视角差异**："哪个角色的认知与主角的认知差距最大？这个差距能成为场景的核心张力吗？"

5. **如多 POV 章节**：考虑是否让该角色持有本章的一个 POV 片段。

**思维触发词**（卡壳时用）：
- "反派的视角——TA 在这个场景中看到了什么主角不知道的事？"
- "旁观者（路人/配角）的视角——TA 怎么解读主角的行为？"
- "不在场的角色——如果 TA 后来听说了这件事，会怎么反应？"
- "物的视角——如果场景中的某物（信物/武器/建筑物）'目睹'了这一幕？"

**每次至少产出**：2-3 个视角方向

## 方法 D：情感核心探索

**思维类型**：由内向外
**适用场景**：知道事件但不知道怎么展开细节 / 需要情感深度 / 想让场景更有感染力

**核心逻辑**：不从事件开始，从"我想让读者在这一章结束时感受到什么情绪"开始——回溯推导出能产生该情绪的场景设计。

**步骤**：

1. **列出情感目标选项**：由思源提议（基于当前故事上下文）或用"情感色轮"让作者选。用内容呈现——"读完这章，你希望读者是（A）心里一紧，（B）想翻到下一章，（C）合上书深呼吸？"

2. **选定 1-2 个核心情感**。

3. **逆向设计场景**：对选定的情感 E → "什么场景能让读者强烈感受到 E？" → 写 2 个候选场景（每个 50-100 字片段，调 `sensory-writer`），展示场景如何引出该情感的机制。

4. **为选定的场景确定情感节拍线**：起点情绪 → 触发事件 → 转折 → 顶点情绪 → 余韵。"从[起点]到[E]的路径——哪个转折点让 E 的感觉最强烈？"

5. **技法匹配**（调 `technique-selector`）："哪种技法最适合呈现这种情感节拍？" → 展示推荐技法 + 在该场景中的使用示例。

**思维触发词**（卡壳时用）：
- "你想让读者在章末合上时心里堵一下，还是松了一口气？"
- "如果这章只能让读者记住一种情绪——是什么？"
- "哪种情绪是前几章还没给过读者的？"

**每次至少产出**：2 个候选场景

## 方法 E：类比移植

**思维类型**：异质同构
**适用场景**：整体感觉不新鲜 / 想要寻找结构性的新方向 / 卡在想不出不同的叙事结构

**核心逻辑**：把当前情境映射到完全不同领域（历史/自然/游戏/体育/职场/其他作品）的结构，提取其核心模式后移植回故事世界。

**步骤**：

1. **界定核心结构**（不依赖故事内容）："抛开故事设定，当前情境最抽象的结构是什么？"示例："强者被迫向弱者求助" / "两个信任的人信息不对等" / "在资源耗尽前必须到达某地"

2. **选择映射领域**：历史事件（真实历史中的类似结构）、游戏机制（回合制/资源管理/信息不对称）、体育竞技（淘汰赛/逆风局/加时赛）、职场政治（站队/谈判/背锅）、自然法则（共生/寄生/捕食/演化）、其他作品的结构（不含角色和设定，只提取节奏和关系模式）。

3. **映射移植**：领域 X 中的核心结构是什么？→"把它翻译回你的故事世界会变成什么样？" → 写 50 字场景片段展示移植结果（调 `sensory-writer`）。

4. **识别"不适应"的部分**："移植过程中有哪些地方觉得别扭？别扭的地方往往有两种可能——①这个领域不适合，②别扭本身可以成为新的故事张力。"如果是②则深入展开。

5. **确认差异化**："现在的版本和原来的思路最大的不同是什么？这个不同是你想要的吗？"

**思维触发词**（卡壳时用）：
- "这不像是小说情节——倒像是[某个其他领域]的什么情况？"
- "如果这不是战斗，而是一盘围棋/一次谈判/一场手术——节奏会怎么变？"
- "历史上有没有类似结构的事件？"

**每次至少产出**：2 个移植方向 + 至少 1 个场景片段

## 方法 F：逆向思维

**思维类型**：反转
**适用场景**：角色行为僵化/太可预测 / 想打破套路 / 读者能猜到接下来会发生什么

**核心逻辑**：列出读者（和作者）期望的"合理的"走向，然后逐一反向——找出每一步的相反面，看哪些相反面产生了有趣的新方向。

**步骤**：

1. **列出期待走向**：读者最可能预期本章如何发展？（列出 3-4 个"明显"的路径）"如果你告诉一个看了前文的读者'下一章是这样的'——他们的默认预期是什么？"

2. **对每个预期做逆向**：预期 A → "反过来——[反向A1] 或者更温和的反向——[反向A2]"。写 50 字场景片段展示"如果不是[预期]，而是[反向]，场面会变成什么样？"

3. **预期对抗性格检查**：对每个方向问——"这个逆向是否服务于角色性格？"→ 服务于性格 → "那这个角色比读者以为的更深"；违背性格 → "角色不会主动做 X，但如果外力逼迫 TA 不得不做——冲突来自内部"。

4. **三层逆向叠加**（可选）：预期 A → 反向 A1 → "如果反向 A1 本身再被反转，变成 A2？"第一层反转让读者意外，第二层反转让读者重新思考。

5. **最后确认**："基于角色性格和已有的伏笔，这些逆向方向中哪些有支撑？逆向不是乱来——好的逆向是'读者完全想不到，但发生后觉得这很合理'。"

**思维触发词**（卡壳时用）：
- "读者最期待什么？——那就别写那个。"
- "这个场景里，角色最不可能做的事是什么？——如果 TA 做了呢？"
- "所有人都觉得会成功的事——如果失败了呢？（或反之）"
- "最温和的解决方式 → 最激烈的解决方式，哪一端更靠近故事的真实？"

**每次至少产出**：3 个逆向方向

---

## 方法 G：缩放实验

**思维类型**：量变到质变
**适用场景**：整体感觉太平/太重复 / 事件知道但节奏不对 / 想找更极致的表达

**核心逻辑**：对场景中的某个核心元素做极端化放大或缩小处理，观察量变是否带来质变，从而找到新的叙事角度。

**步骤**：

1. **选择核心元素**：场景中的某个关键因素——冲突强度/时间尺度/参与人数/资源数量/代价程度/信息密度/情感强度。

2. **放大实验（×5, ×10）**：把核心元素放大到极端 —— 写 50 字场景展示放大后的场面。

3. **缩小实验（÷5, ÷10）**：把核心元素缩小到极端 —— 写 50 字"极简版本"和"极致放大版本"对比。

4. **找到"质变点"**：放大到哪个程度时，场景的性质变了？缩小到哪个程度时，场景的核心反而更清晰了？"放大的版本保留了场景的本质还是改变了它？——改变的地方往往是新方向的入口。"

5. **选择平衡点**：作者选择一个介于两个极端之间的平衡点。"如果你想要[放大版]的冲击力，但保留[原版]的细腻——平衡点在哪？"

**思维触发词**（卡壳时用）：
- "把这个冲突放大 10 倍——角色会怎么反应？"
- "如果这一章只写 300 字——哪个瞬间不能删？"
- "如果这个场景的时间拉长到一整天？压缩到 3 分钟？"
- "增加一个人/减少一个人——剧本怎么变？"

**每次至少产出**：2 个极端方向 + 至少 1 个平衡方向

---

## 方法 H：叙事永动机

**思维类型**：随机注入式发散
**适用场景**：场景太平淡 / 不知道角色接下来做什么 / 所有角色都在"等剧情发生"

**核心逻辑**：当叙事陷入停滞，往往不是因为缺剧情，而是因为角色缺欲望。此方法通过随机注入"角色欲望"和"外部事件"打破僵局——让角色从"被动等待"切换到"主动驱动"。

**步骤**：

1. **选定目标角色**：当前场景中戏份最少或行为最被动的角色
2. **随机注入欲望**：从欲望库中抽取一个欲望种子，分配强度（轻微/中等/明显/强烈）
   - 欲望库（可根据小说世界观裁剪）：
     - 身体驱动：想独处 / 想离开这里 / 想吃东西 / 身体不适需要休息 / 想舒展一下
     - 情感驱动：想被注意 / 想被忽略 / 想报复某人 / 想证明自己 / 想被安慰 / 想道歉
     - 认知驱动：想弄清楚某事 / 想确认某个猜测 / 想推翻某个假设 / 想比较两人的反应
     - 社交驱动：想拉近关系 / 想保持距离 / 想测试对方 / 想引起冲突 / 想和解 / 想引起嫉妒
     - 行动驱动：想做点什么/什么都行 / 想完成手头的事 / 想停下来 / 想破坏什么
3. **随机注入事件**（概率：约 40% 触发）：
   - 环境类：天气突变 / 噪音干扰 / 物品损坏/掉落 / 有人误入 / 设备故障
   - 信息类：新消息到达 / 旧信息被重新解读 / 误会发生 / 偷听到不该听的
   - 因果类：之前的某个决定现在产生了后果 / 某个承诺到了兑现/无法兑现的时候
   - 感官类：突然闻到什么 / 突然注意到之前忽略的东西 / 身体感觉突然变化
4. **生成"如果"方向**：
   - "如果 [角色A] 突然产生了 [欲望]，强度 [X]，会发生什么？"
   - "+ 如果此时 [随机事件] 发生呢？"
5. **写 80-120 字的场景片段**展示这个方向的感觉（调用 `sensory-writer` Skill）
6. **换角色重复 2-3 轮**，每轮换不同角色，产出多个互不重复的方向

**约束**：
- 欲望必须翻译为场景特定的、角色身体化的具体渴望——不是抽象标签，是这一刻角色真正在"馋"什么
- 事件必须因果合理——不是天降陨石，而是"一直存在但刚刚被注意到的东西"
- 欲望和事件的标签词（如"寻求刺激""误会发生"）只用于内部推理，不暴露给作者

**思维触发词**（卡壳时用）：
- "[角色] 现在最想要什么——不是因为剧情需要，是因为 ta 是 ta？"
- "如果这个场景的平静被一件小事打破——最自然的小事是什么？"
- "谁在这个场景里最沉默？ta 在想什么？ta 想要什么？"
- "上一章的角色决策现在会产生什么涟漪？"

**每次至少产出**：2-3 个"欲望注入"方向（各附场景片段）

## 方法选择指南

作者描述困境 → 问自己：作者卡在哪种空白？

```
├── "完全不知道写什么"——情节方向空白
│   → 主：方法 A（如果爆炸）
│   → 辅：方法 B（约束游戏）
│
├── "知道事件但展开不了"——场景设计枯竭
│   → 主：方法 D（情感核心探索）
│   → 辅：方法 G（缩放实验）
│
├── "角色太可预测"——角色行为僵化
│   → 主：方法 C（视角切换）
│   → 辅：方法 F（逆向思维）
│
├── "场景太平淡/角色被动等剧情"——叙事停滞
│   → 主：方法 H（叙事永动机）
│   → 辅：方法 B（约束游戏——添加外部约束逼迫角色行动）
│
├── "整体太平/太重复"——新鲜感缺失
│   → 主：方法 E（类比移植）
│   → 辅：方法 A（如果爆炸——对核心设定做变体）
│
├── "就是感觉不对，但说不出来"
│   → 主：方法 E（类比移植——从结构找差异）
│   → 辅：方法 C（视角切换——从他人视角找盲区）
│
└── 仍然卡住 / 以上都不适用
    → 方法 F（逆向思维）+ 从以下随机选一：新增一个约束跑方法 B / 随机选一个角色跑方法 C / 跑方法 H 注入随机欲望
```

**不重复原则**：同一个方法在同一章头脑风暴中最多跑 2 轮深入。2 轮后仍未找到满意方向 → 换方法。
