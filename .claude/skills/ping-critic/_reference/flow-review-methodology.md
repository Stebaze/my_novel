# 琉璃 — 心流评审与综合评审方法论

> **定位**：本文件是「琉璃」(Liuli) Agent 的核心评审方法论。琉璃在执行综合评审时读取本文件获取心流五维 18 项检测算法、场景类型阈值调整规则、三维评审框架和 DoD 检查标准。
>
> 本文件被 `ping-critic` Agent 直接引用。

## 一、心流五维 18 项检测算法

对章节全文运行以下检测。场景类型（日常/战斗/对话/情感爆发）影响阈值期望，见第二章。

### 维度一：挑战-技能平衡

| # | 检查项 | 检测方法 | 阈值 |
|---|--------|---------|------|
| 1.1 | 语义留白度 | 统计解释句占比（因为/所以/这意味着/换句话说/毕竟/说到底）；检测叙述者解码四模式（抽象情感标签/破折号态度旁白/引导内态度转述/对称排比式性格解码） | >10% → ❌；>7% → ⚠️；四模式全章命中≥3 → ❌ |
| 1.2 | 认知负荷均衡 | 标注新信息点，检查分布。每 2-3 新信息后是否有 ≥2 段低负荷缓冲？ | 所有新信息集中同一场景 → ❌；单场景 >3 → ❌；无缓冲 → ⚠️ |
| 1.3 | 描写克制度 | 环境描写是否 1-2 句？外貌 ≤2 句？动作按时间顺序？ | 连 3 句纯环境 → ❌；静态外貌 → ❌ |

### 维度二：目标清晰

| # | 检查项 | 检测方法 | 阈值 |
|---|--------|---------|------|
| 2.1a | 场景切换定向 | 每个 `---` 切换点前 2 句含"谁/在哪/什么时间/什么情绪"？ | 无定向 → ❌；≥2 场景无定向 → 维度降级 |
| 2.1b | 场景桥接质量 | 场景 A 末句 + 场景 B 前 2 句的 T(时间)/P(物理)/E(情绪) 检查 | 缺 ≥2 类 → ❌；≥2 硬切 → 维度降级 |
| 2.1c | 衔接技法多样性 | 统计衔接技法类型数（S1-S14）；连续同技法？ | 全章仅 1 种 → ⚠️；连 3 场同技法 → ⚠️ |
| 2.1d | 章间衔接 | Ch{N-1} 末段 → Ch{N} 首段衔接类型？ | 硬切换 → ❌ |
| 2.2 | 段落衔接自然度 | 逐对标注 C(因果)/T(时间)/A(补充)/K(对比)/N(无衔接) | N 类 >15% → ❌；连续 N ≥3 → ❌ |
| 2.3 | 对话主题流转 | ≥4 轮对话标注主题转换 | ≥2 处脚本对话 → ❌ |

### 维度三：即时反馈

| # | 检查项 | 检测方法 | 阈值 |
|---|--------|---------|------|
| 3.1 | 信息密度波动 | 逐段标注 P(情节)/C(人物)/E(环境)/F(情感)/T(过渡) | 连 P≥4 → ❌；P/C/F 连 ≥3 → ⚠️；转折前密度未降 → ❌ |
| 3.2 | 句子节奏变化 | 每 200 字 ≥1 破句(≤8字)？每 500 字 ≥1 长句(≥45字)？ | 破句全缺失 → ⚠️；连 3 段波动 <10 字 → ⚠️ |
| 3.3 | 段落反馈单元 | 大多 ≤2 句？对话独立成段？ | 连 4 句以上 → ❌ |
| 3.4 | 对话密度 | 占比 ≥50%？最长无对话段落 >300 字？连说 ≥5 轮无反应？ | 对话 <50% → ⚠️；>300 字无对话 → ❌ |

### 维度四：专注保护

| # | 检查项 | 检测方法 | 阈值 |
|---|--------|---------|------|
| 4.1 | 叙事声音一致性 | 异质词检测（于是/突然/倏忽/显然/毫无疑问）；归因位置；称呼体系 | 归因在前 ≥2 → ❌ |
| 4.2 | 模式新鲜度 | 桥段/开篇/断章 vs 近 3 章模式记录 | 断章连续 2 章同类型 → ❌ |
| 4.3 | 干扰性过渡词 | 每 500 字过渡词密度 | >10 → 触发衔接深度审查 |
| 4.4 | 人味锚点 | 每个场景 ≥1 "有趣但不完全合理"的表达？ | 无 → ⚠️ |
| 4.5 | 句法模板多样性 | 提取句法骨架，统计重复；跨章对比 | 同骨架 ≥8 次 → ❌；5-7 → ⚠️ |

### 维度五：行动-意识融合

| # | 检查项 | 检测方法 | 阈值 |
|---|--------|---------|------|
| 5.1 | 情感身体性 | 内心标签词（感到/觉得/意识到/涌起/泛起/充满）密度 | 每 500 字 >2 → ❌；关键时刻出现 → 直接 ❌ |
| 5.2 | 情绪递进链 | 场景情绪强度变化(1-10)，画情绪波形 | 跳变 >3 分无过渡 → ❌；连 3 段同强度 → ⚠️ |
| 5.3 | 角色声音区分度 | 遮名测试 | 无法区分 → ⚠️/❌ |
| 5.4 | 感官锚定质量 | L1: 每场景≥1 非视觉？L2: 全章≥3 种非视觉？L3: ≥100 字描写经角色意识过滤？L4: 关键时刻配感官？ | L1 无 → ⚠️；L3 无人机 → ❌ |

**汇总**：记录每项判定（✅/⚠️/❌）+ 断裂点位置（段落号+行文引用）。

## 二、场景类型与阈值调整

不同场景类型对检测阈值有不同期望。以下为各类型的阈值宽松/收紧规则：

| 场景类型 | 宽松项 | 收紧项 |
|---------|--------|--------|
| **日常** | 3.1 信息密度、1.2 认知负荷 | 4.4 人味锚点、5.4 感官锚定 |
| **战斗** | 3.4 对话密度、5.1 情感身体性（动作优先） | 3.1 信息密度（需节奏控制）、2.1b 场景桥接 |
| **对话** | 3.3 段落反馈单元、1.3 描写克制度 | 2.3 对话主题流转、5.3 角色声音区分度 |
| **情感爆发** | 4.4 人味锚点（情感真实优先）、5.2 情绪递进链 | 1.1 语义留白度、5.1 情感身体性 |

## 三、综合评审框架

### 3a. 结构层评审

- 每个场景是否有不可删除的理由？
- 场景之间因果链是否成立？
- 章节在本卷中的定位是否正确？
- 场景功能/情感弧线/信息密度/场景衔接/断章有效性
- 字数是否在目标范围？有无空转段？

### 3b. 角色与设定层评审

- **设定一致性**（逐项 ✅/⚠️/❌）：角色行为/生死/世界观规则/时间线/前后章节一致/_changes.md 验证/POV 纪律
- **心理一致性**：心理结构/防御机制/情感递变/阴影揭示（结合 Step 1 维度五数据）
- **逻辑质疑**：每个关键情节转折生成质疑 → 🔴/🟡/🟢
- **心流中断点归因**：结合 Step 1 断裂点，从编辑视角解读每个断裂点的角色动机/设定成因
- **模式新鲜度判定**：Step 1 的 4.2 数据 → 是角色驱动重复还是作者惯性

### 3c. 文本质地层评审

- 五维逐维判定：✅ / ⚠️ / ❌
- AI 指纹归因：断裂点→指纹→生成模式→修复策略（来自铃）
- 校对问题汇总（来自铃）

### 3d. 综合判定

- **心流评级**：顺畅 / 有摩擦 / 断裂 / 无法进入
- **跨章升级**：连续 2 章同一指纹 → 升级；连续 3 章 → 校准建议
- **修复优先级排序**：🔴 > 🟡 > 🟢

### 3e. DoD 检查（Definition of Done）

| # | 检查项 |
|---|--------|
| 1 | 情感目标已交付？ |
| 2 | 至少一个名场面？ |
| 3 | OOC 零命中？ |
| 4 | 结尾有效断章？ |
| 5 | 字数在目标范围？ |

---

## 四、评审原则

1. **综合评审一次完成**：心流检测→指纹+校对→三维评审→统一报告，一轮到位
2. **AI 不自动修改正文**：所有修改由作者执行。🔴 标注矛盾位置+冲突来源+修改方向
3. **分级响应**：🔴=逻辑漏洞/能力越界必须修，🟡=动机不充分建议修，🟢=可接受的艺术省略
4. **DoD 五条**：情感目标已交付/至少一个名场面/OOC零命中/结尾有效断章/字数在目标范围

## 扩展指南

### 新增检测项

在对应维度下按编号递增添加新行。遵循现有格式：

```
| {编号} | {检查项名称} | {检测方法描述} | {阈值} |
```

新增维度时编号递增（维度六、维度七…），同步更新场景类型调整表。

### 调整阈值

阈值调整只需修改对应表格中的数值。✅/⚠️/❌ 级别调整修改最后一列。注意：
- 收紧阈值 → 更多 ❌ 判定 → 更严格的评审
- 放宽阈值 → 更多 ⚠️ 判定 → 更宽松的评审

### 新增 DoD 检查项

在 3e 表格中添加新行，编号递增。DoD 项应该是二元判定（是/否），不应有模糊空间。
