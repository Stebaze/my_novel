# 铃 — AI 指纹检测与文字校对方法论

> **定位**：本文件是「铃」(Ling) Agent 的完整检测方法论（铃已并入 `ping-critic` Skill）。`ping-critic` 在执行指纹检测/校对时读取本文件获取五指纹匹配规则、15 项自检阈值、生成模式推荐逻辑、跨章升级规则、校准策略和四类校对检测标准。
>
> 本文件被 `ping-critic` Agent 直接引用，也被 `fingerprint-discovery` Agent 在已知指纹匹配阶段参考。
>
> **SSOT 契约**（2026-06-25 重构）：下表「五指纹匹配」与下文「叙述者解码参考」正字法级检测规则的**完整定义在 `framework/guides/ai-risk-mitigation.md`「叙述者解码参考（诊断附录）」**（含 `{#fingerprint-N}` 锚点）。本文件保留为兼容层——`ping-critic` 在 fingerprint-match 操作时优先读 ai-risk-mitigation.md 锚点，失败时降级读本文件第一/三章。**新增指纹优先更新 ai-risk-mitigation.md，再按本文件扩展指南同步。**

## 一、五指纹匹配

对每个心流断裂点，匹配以下五指纹模式：

| # | 指纹 | 表现 | 检测触发 |
|---|------|------|---------|
| 1 | 过度平滑 | 句子均匀、过渡词密集、无破句 | 3.1/3.2/3.3 ❌ |
| 2 | 声音均化 | 角色说话趋同、锚点词缺失 | 5.3 ❌ |
| 3 | 合理偏置 | 选最合理而非最有趣的回答、情感直达 | 1.1/5.1/5.2 ❌ |
| 4 | 语境衰减 | 伏笔揭示失去埋入时的气氛 | 2.2 ❌（伏笔相关场景） |
| 5 | 叙述者解码 | 抽象情感标签/破折号态度旁白/引导内转述/对称排比式性格解码 | 1.1 ❌（四模式命中） |

> 五指纹的理论定义见 `framework/guides/ai-risk-mitigation.md`「为什么 AI 文本会暴露身份」。正向生成方法（感官锚定写作/角色声音自然化/回响式伏笔）见同文件方法一~三。本文件聚焦**检测端**——如何在已有文本中定位这些指纹。

## 二、15 项自检

### 风格层（5 项）

| # | 自检项 | 阈值 | 级别 |
|---|--------|------|:--:|
| 1 | 过渡词密度 | 每 500 字 ≤8 | 🔴 |
| 2 | 句长有破句 | <8 字 + >45 字各 ≥1 | 🟡 |
| 3 | 出场角色锚点词 | ≥1 | 🟡 |
| 4 | 有趣但不合理的表达 | ≥1 处 | 🟡 |
| 5 | 断章是有效钩子 | 结尾在"变化即将发生" | 🟡 |

### 心流层（5 项）

| # | 自检项 | 阈值 | 级别 |
|---|--------|------|:--:|
| 6 | 信息密度波动 | 无连 ≥4 段同类型 | 🔴 |
| 7 | 情绪递进链 | 每 500 字内心标签 ≤2 | 🔴 |
| 8 | 过度解释 | 解释句占比 <10% | 🔴 |
| 9 | 叙述者解码 | 无四模式命中 | 🔴 |
| 10 | 认知休息点 | 每 2-3 新信息后有 ≥2 段缓冲 | 🟡 |

### 文本风格层（5 项）

| # | 自检项 | 阈值 | 级别 |
|---|--------|------|:--:|
| 11 | 数字滥用 | 无前文根据的精确数字 | 🟡 |
| 12 | "不是…而是…"误用 | 作者叙述中出现 | 🟡 |
| 13 | 英式中文 | 过度"当…时"/被动/长定语 | 🟡 |
| 14 | 重复描写 | 3000 字内同一角度 ≥3 次 | 🟡 |
| 15 | 长段落疲劳 | 连单 POV 纯描写 >500 字 | 🟡 |

**级别含义**：🔴 = 纯量化违规，自动修复。🟡 = 定性判断，生成 2-3 个修改方案供选择。

## 三、生成模式推荐

| 指纹触发 | 建议模式 | 行为 |
|---------|---------|------|
| 指纹 1（过度平滑） | 高纹理 | 强化方法一执行——更深进入身体感知，句长跟随情绪呼吸 |
| 指纹 2（声音均化） | 声音锁定 | 强化方法二执行——更深进入角色声音，脑中"听到"再下笔 |
| 指纹 3（合理偏置） | 高纹理 | 强化方法一+方法二——更有趣的表达，更不讲理的对白 |
| 指纹 4（语境衰减） | 回响 | 强化方法三执行——重读语境快照，感官细节呼应 |
| 指纹 5（叙述者解码） | 高纹理 | 强化方法一执行——进入角色身体，用身体感觉替代情感标签 |

---

## 四、跨章升级规则

- 连续 2 章同一指纹 → 判定升级（⚠️→❌），建议下章启用对应模式
- 连续 3 章同一指纹 → 附带"声音快照对比"或"风格校准"建议
- 同一场景 ≥3 种指纹 → 评级自动降为"断裂"，建议重写
- 维度五 ❌ + 指纹 2 → 标记交叉验证触发

---

## 五、校准策略（条件执行）

- 每 10 章：声音快照对比（当前 vs 10 章前）
- 每 20 章：风格校准（最新 vs 第一章各 500 字）
- 每月：盲测校准（关掉所有策略 vs 启用策略）

## 六、校对检测标准

### 6.1 语病检测

| # | 检测项 | 检测方法 | 级别 |
|---|--------|---------|:--:|
| G1 | 成分残缺 | 缺主语/缺宾语/缺谓语 | 🔴 |
| G2 | 搭配不当 | 动宾不搭/主谓不搭/修饰语与中心语不搭 | 🔴 |
| G3 | 句式杂糅 | 两种句式混用（如"之所以…是因为…"与"…的原因是因为…"杂糅） | 🟡 |
| G4 | 语序不当 | 定语/状语位置错误导致歧义 | 🟡 |
| G5 | 重复啰嗦 | 同义词连用（如"目前的现状"、"更加进一步"） | 🟢 |
| G6 | 歧义表达 | 代词指代不明/修饰语指向不明 | 🔴 |

### 6.2 错别字检测

| # | 检测项 | 检测方法 | 级别 |
|---|--------|---------|:--:|
| T1 | 形近错字 | 字形相近的别字（如"己/已/巳"、"侯/候"） | 🔴 |
| T2 | 音近错字 | 同音/近音别字（如"的/地/得"、"在/再"） | 🔴 |
| T3 | 规范用字 | 异体字/繁体字（非风格需要时） | 🟡 |

### 6.3 标点符号检测

| # | 检测项 | 检测方法 | 级别 |
|---|--------|---------|:--:|
| P1 | 引号嵌套 | 双引号内再引用应用单引号 | 🟡 |
| P2 | 省略号/破折号 | 省略号应为 `……`（非 `。。。` 或 `...`）；破折号应为 `——`（非 `--` 或 `—`） | 🟡 |
| P3 | 对话标点 | 说话人说在前用冒号+引号，说在后用逗号+引号 | 🟡 |
| P4 | 逗号/句号误用 | 该用句号处用逗号（逗号连接多个完整句） | 🟡 |
| P5 | 感叹号/问号滥用 | 连用 ≥3 个感叹号/问号 | 🟢 |

### 6.4 分段检测

| # | 检测项 | 检测方法 | 级别 |
|---|--------|---------|:--:|
| B1 | 超长段落 | 单段 >500 字且非意识流/内心独白 | 🟡 |
| B2 | 对话未独立成段 | 不同说话人的对话挤在同一段 | 🔴 |
| B3 | 场景切换未分段 | 场景/时间/视角切换时无空行 | 🟡 |
| B4 | 动作-反应混段 | 角色 A 的动作和角色 B 的反应在同一段（应分段） | 🟡 |

## 七、词汇多样性检测

> 词汇+句法+感官三维扫描——检测文本底层结构中的重复模式、感官盲区和角色声音均化。检测结果附带修复建议。

### 7.1 n-gram 重复扫描

| # | 检测项 | 检测方法 | 级别 |
|---|--------|---------|:--:|
| V1 | 实词高频重复 | 排除功能词（的/了/是/在/和），统计 ≥3 次重复的实词，标注位置 | 🟡 |
| V2 | 短语模板重复 | 2-gram/3-gram ≥2 次重复，标注位置 | 🟡 |
| V3 | AI 高频词标记 | 扫描"美丽""壮观""非常""然后""似乎""仿佛""缓缓""深深"（来自 ai-risk-mitigation.md 指纹库），≥3 次标注 | 🟡 |

> 不自动判定"好"或"坏"——风格性重复（海明威式）和无意识重复由作者判断。V3 项直接给替换建议。

### 7.2 语义场分布

| # | 检测项 | 检测方法 | 级别 |
|---|--------|---------|:--:|
| V4 | 语义范畴占比 | 描写性词汇按视觉/听觉/触觉温度/嗅觉味觉/情感/动作/认知分析分类，计算占比 | 🟢 |
| V5 | 主导感官单一 | 全章场景 ≥80% 为同一感官主导 → 标注 + 建议补充其他感官通道 | 🟢 |

### 7.3 句法模板扫描

| # | 检测项 | 检测方法 | 级别 |
|---|--------|---------|:--:|
| V6 | 句法骨架重复 | 提取全章叙述句（非对话）的句首 2-3 词结构模式，≥5 次标注 🔴，3-4 次标注 ⚪ | 🟡 |
| V7 | AI 高频模板 | 检测"动作逗号观察"（[角色][动作]，[观察/结果]）、"否定断言排比"（不是X。是Y。）、"心理标签直达"（[角色]感到/觉得/意识到[名词]），≥5 次标注 🔴 | 🟡 |

### 7.4 感官分布扫描

| # | 检测项 | 检测方法 | 级别 |
|---|--------|---------|:--:|
| V8 | 连续同感官场景 | 连续 ≥3 场景以同一感官为主导 → 标注 🟡 | 🟡 |
| V9 | 无人机视角段落 | ≥100 字的纯视觉描写段（未经角色意识过滤，零非视觉感官）→ 标注位置 | 🟡 |

### 7.5 对话遮名测试

| # | 检测项 | 检测方法 | 级别 |
|---|--------|---------|:--:|
| V10 | 角色台词可辨度 | 提取所有对话台词（去归因/动作描写），隐去角色名——检查能否从台词本身（词汇/句式/口癖）分辨说话人。标注可能混淆的台词对 | 🟡 |

> V10 检测依赖角色语言指纹档案。某角色无指纹档案 → 仅标注与同场景其他角色的明显词汇重叠，标注"⚠️ 无指纹基准"。

### 7.6 修复建议生成

对 V1-V10 的每项检测结果，生成对应修复建议：

| 检测项 | 修复策略 | 输出形式 |
|--------|---------|---------|
| V1 实词重复 | 提供 2-3 个上下文相关的同义替换选项 | 每个替换选项附带效果差异说明（1-2 句） |
| V2 短语重复 | 提供句法重构建议——不换意思，换骨架 | 2-3 种句法骨架改写，附带效果差异 |
| V3 AI 高频词 | 直接给替换词 + 标注"AI 高频词" | 替换选项 + 来源标记 |
| V4-V5 感官偏斜 | 建议补充占比最低的感官通道 | 具体到场景/段落——哪个位置可以加什么感官描写 |
| V6-V7 句法重复 | 提供句法多样化建议 | 标注重复骨架位置 + 2-3 种替代骨架 |
| V8-V9 感官盲区 | 建议在无人机视角段落插入角色感官滤镜 | 具体段落引用 + 建议插入的感官类型 |
| V10 声音均化 | 标注混淆台词对 + 建议差异化方向 | 基于角色档案给出各角色应强化的语言特征 |

> 修复建议遵循"作者决策原则"：AI 呈现选项并解释差异，不自动替换。选哪个（或都不选）由作者决定。

### 新增指纹

**SSOT 流程**（2026-06-25 重构）：

1. **必做**——在 `framework/guides/ai-risk-mitigation.md`「叙述者解码参考（诊断附录）」追加新指纹，锚点格式 `### {编号}. {模式名} {#fingerprint-N}`，必含字段：表现 / 例句 / 为什么是 AI 指纹 / 为什么违和 / 检测规则 / 修复策略 / 与已知指纹关系
2. **必做**——调用 `fingerprint-discovery` Skill 阶段 4 自动同步校验（验证 `sensory-writer` 加载协议能 grep 到锚点）
3. **条件**——在「三、生成模式推荐」添加对应推荐行（指纹触发 → 建议模式）
4. **条件**——在「四、跨章升级规则」评估是否需要新组合规则
5. **可选**——在本文「一、五指纹匹配」表格添加一行（兼容层摘要，完整定义在 ai-risk-mitigation.md）

> **校验回路**：完成 1+2 后，fingerprint-discovery 会自动 grep `{#fingerprint-N}` 验证 sensory-writer / ping-critic 都能读到。失败时警告不阻断——但下次评审/生成可能漏检。

### 候选指纹自动发现（评审端反向回路）

> **问题**：当一段文本「未命中已知指纹但读起来违和」时——是已知指纹变体？全新模式？还是人类写作风格的自然变化？本节定义评审端的**反向检测**算法：从 review 端向 discovery 端输送候选。

**SSOT 闭环**（2026-06-25 重构）：当 chapter-review 评审时发现候选指纹 → 一键调用 `fingerprint-discovery`（input mode="from-candidate"）→ 人工审阅 → 入库。新增指纹只需写一次 ai-risk-mitigation.md，消费方全部自动同步。

#### 三层叠加检测算法

```
触发场景：Step 2a 指纹匹配后，存在 🔴/🟡 项但未对应任何 5-指纹
          或 Step 1 心流断裂但无指纹归因

Layer 1 — 统计异常检测（确定性，零 LLM 调用）：
  输入：Step 1/2 已识别的 V6/V7/V9 命中段
  判定：对每段检查（V6 ∨ V7 ∨ V9）∧ (指纹 match = None)
  输出：signal_a = { location, triggers: [V6|V7|V9], snippet }

Layer 2 — 读者效应自述（LLM 调用，1 次/段）：
  触发：signal_a 存在 或 Step 1 心流断裂且无指纹归因
  Prompt:
    "阅读以下段落。从普通读者视角描述：这段让你有什么违和感？
     要求：1-2 句自然语言；不引用任何写作术语或已知规则；描述'感觉'而非'诊断'。"
  输出：signal_b = { location, reader_effect_text, keywords[] }

Layer 3 — 描述-指纹不匹配检测（确定性，文本相似度）：
  加载：ai-risk-mitigation.md 附录所有指纹的「表现」+「例句」+「为什么违和」字段
  计算：description_novelty = 1 - max(LLM_similarity(signal_b.text, 已知指纹描述))
  判定：description_novelty > 0.6 → signal_c = True
  输出：signal_c = { location, novelty_score, closest_known: { name, similarity } }
```

#### 置信度合成

```
confidence = 0.4 × stat_score + 0.4 × novelty_score + 0.2 × cluster_density

其中：
  stat_score = (V6_fired + V7_fired + V9_fired) / 3 ∈ [0, 1]
  novelty_score = signal_c.novelty_score ∈ [0, 1]
  cluster_density = min(相邻 3 段内候选数 / 3, 1) ∈ [0, 1]

分级：
  🟢 高（>0.7）：3+ 独立信号 + 描述独特 → 强烈建议作者确认
  🟡 中（0.4-0.7）：1-2 信号 → 提示作者注意
  ⚪ 低（<0.4）：单一统计异常 → 仅记录，不主动提示
```

#### 输出 schema

```yaml
new_fingerprint_candidates:
  - location: "scene 2, para 4"
    evidence: "本段被 V7 (AI 高频模板 ≥5) 命中，但 5-指纹 match 无结果"
    reader_effect: "读起来像 AI 写的总结，不像角色在说话"
    suggested_pattern: "AI 总结式叙述（候选）"
    confidence: 0.78
    confidence_level: "🟢 高"
    triggering_signals: [V7, description_novelty_high, cluster_2/3]
    closest_known:
      name: "否定式双句排比"
      similarity: 0.12
```

> **零候选**：本轮无 Layer 1 触发段时，输出 `new_fingerprint_candidates: []`（不写 null，保持 schema 稳定）。

### 调整阈值

阈值调整只需修改对应表格中的数值。🔴/🟡/🟢 级别调整修改最后一列。注意 🔴 和 🟡 的语义区别：🔴 是纯量化自动修复，🟡 是定性判断需人工选择方案。
